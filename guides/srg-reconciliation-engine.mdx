## 🔧 THE PROCESSES / TOOLS — the per-client flow

The system is **harvest-once (bulk) → resolve-per-client (deterministic) → LLM reconcile → human gate →
publish**. Boxes 1–5 are bulk prep; Boxes 6–7 are the per-client loop.

**Phase A — Bulk harvest (run once across the book; the 5 harvest tables):**
`harvest-fub-facts.mjs` (brain, Haiku) · `harvest-mls-history.mjs` + `resolve-mls-rooftops.mjs` (MLS
rooftop, via Camp David Matrix scraper) · `restamp-mls-hlid.mts` + `link-hlid-on-mls.mjs` (HLID stamp) ·
`harvest-loop-full.mjs` + `dotloop-auth.mjs` (dotloop `.loop`, Haiku Tier-3 only) · `scrape-deed-homes.mjs`
(lost-buy source) · `dedupe-past-clients.mjs` (one card / person).

**Phase B — Per-client resolution (Box 6, "reconcile, don't rebuild"):**
1. `triangulate-resolver.mjs` → `createResolver(c).assemble(id)` — **Stage 1, deterministic, $0, no LLM.**
   JOINs the 5 harvest tables + roster + identity/bucket + Adam's GT table, unions transactions (dedup
   rooftop+side), runs the agent waterfall, tiers GREEN/YELLOW/RED. Reads ONLY virgin sources (never its
   own prior output).
2. `triangulate-stage2.mjs` → **Stage 2 Haiku one-shot** — reconciles conflicts + writes the reactivation
   dossier (2 paragraphs + facts matrix) + confirms tier. Writes STAGING (`hl_srg_resolver_stage2`).
3. `complete-closings.mjs` → materialize each SISU/lost closing as its own child row (buy/our-sell/lost-sell),
   `--write` gated, scoped to given ids.
4. `link-contacts-fub.mjs` → deterministically resolve each household member's `fub_id`/`fub_url` onto
   `dossier_json.contacts`, `--write` gated.
5. `persist-reconcile.mjs` → stage the resolver output into `hl_srg_pch_staging` (the "Reconciled — &lt;date&gt;"
   audit tab).

**Phase C — Human gate + publish (Box 7):**
