## Missing ‚Äî Needs an API ‚ùå

These exist in the UI but have no API endpoint. Every one of these is a gap
that blocks Myli and any integration partner.

### Priority 1 ‚Äî Myli-Critical

| UI Action | Missing Endpoint | DB Table |
|---|---|---|
| Add a note to a lead | `POST /api/leads/{id}/notes` | `hl_notes` |
| List notes for a lead | `GET /api/leads/{id}/notes` | `hl_notes` |
| Get message history for a lead | `GET /api/leads/{id}/messages` | `hl_messages` |
| List appointments for a lead | `GET /api/leads/{id}/appointments` | `hl_appointments` |

### Priority 2 ‚Äî Settings Missing API

| UI Feature | Missing Endpoint | Notes |
|---|---|---|
| Appointment Types CRUD | `/api/appointment-types` | Sidebar shows this page but no route exists |
| Appointment Outcomes CRUD | `/api/appointment-outcomes` | Same |
| Business Hours | `/api/business-hours` | Sidebar item, no route |
| Forms / Websites | `/api/forms` | Sidebar item, no route |
| Outbound Webhooks | `/api/webhooks` | Sidebar "Webhooks" item ‚Äî no management API |

### Priority 3 ‚Äî Nice to Have

| Feature | Missing Endpoint |
|---|---|
| Lead activity feed | `GET /api/leads/{id}/activity` |
| Stage transition history | `GET /api/leads/{id}/transitions` |
| Conversations / thread view | `GET /api/conversations` |
| Notification preferences | API exists (`/api/notification-preferences`) but not documented |
| Profile update | API exists (`/api/profile`) but not documented |
