## The goal

An agent opens a lead in **TC view** and clicks **Add Transaction**. A native intake funnel — the Cognito-equivalent,
but in-app — **auto-fills from the lead** (name, phone, address, agent) **and the AI-created buyer-agency loop**,
captures the transaction (contract, EMD, inspections, financing, commission) + the **key dates**, and on save it
**creates/links the loop, sets the anchor dates, records the deal, and projects the dated stepper.** Buyers or
sellers. One flow. We own the data.

## The problem it fixes (current state)

Today the TC surfaces are fragmented — three unlinked pieces:

| Piece | Where it is now | Gap |
|---|---|---|
| **Create Loop** | `TcCreateLoopPrompt` — shows in TC view ONLY when the lead has *no* `hl_transactions` row | Hidden the moment any row exists (even a deal); no one can find "create new loop" |
| **Add Deal** | Deals tab → `DealMilestoneModal` (Pending / Closed / Lost) → a separate `hl_transactions` row | Creates a *deal*, never a *loop*; no intake, no dates |
| **Move to Under Contract** | opens the deal sheet, creates a deal row, commits the stage | Does **not** create/link a loop or write any milestone dates |

**Loops and deals are separate, unlinked rows** in `hl_transactions` produced by two independent flows. Nothing
joins them, nothing captures the intake, and the stepper has no dates.

## The target — one "Add Transaction" flow

**One entry point** replaces the hidden Create-Loop prompt: an **Add Transaction** button on the TC Dashboard band
(and the band's empty state). It — and moving a lead to **Under Contract** — open the **same native intake funnel**:

<Steps>
<Step title="Pre-fill">
From the lead (`hl_leads`: name, phone, address, agent) + the AI-created loop if one exists (side, price, participants).
</Step>
<Step title="Capture">
The intake (Contract · EMD · HOA · Inspections · Financing · Commission — the [Cognito schema](/guides/cognito-forms-api))
+ the two **anchor key dates**: **Effective / Under Contract** and **Closing**.
</Step>
<Step title="Create / link the loop">
`createLoop` (if none for that side) seeds the 14/16 milestones + tasks from the side templates.
</Step>
<Step title="Set dates → project the stepper">
Write the anchor dates to the loop → the **date engine** projects every other milestone/task date (`anchor ± offset`)
→ the dated stepper renders.
</Step>
<Step title="Record + link the deal">
The deal (commission / close) is recorded and **linked to the loop** — deal + loop become ONE transaction, not two orphan rows.
</Step>
</Steps>

Same funnel for buyers and sellers; same funnel whether triggered by the button or the Under-Contract stage move.

## Where the data lives (own it)

The intake is stored natively in `hl_tc_*` using the A-Frame-style **Collection → Group → Field** model
(config-not-code), seeded from the [Cognito schema](/guides/cognito-forms-api). Cognito's New Listing / Status Change
/ Pending Transaction forms map the same way as future native forms. The [key dates](/guides/cognito-forms-api)
(`ContractDetails.EffectiveDate` + `ClosingDate`) are the anchors the stepper runs on.
