## Error Format

All errors return JSON with `ok: false` and an `error` string:

```json
{ "ok": false, "error": "not_found" }
```

## HTTP Status Codes

| Code | Meaning | When |
|---|---|---|
| 200 | OK | Success |
| 201 | Created | Resource created |
| 400 | Bad Request | Missing required field, invalid JSON, invalid value |
| 401 | Unauthorized | No session or invalid token |
| 404 | Not Found | Resource doesn't exist or was deleted |
| 409 | Conflict | Duplicate (email already exists, name already taken) |
| 500 | Server Error | Unexpected failure — check server logs |

## Common Error Strings

| error | Meaning |
|---|---|
| `unauthorized` | No valid session or token |
| `not_found` | Resource not found |
| `invalid_json` | Request body is not valid JSON |
| `no_fields` | PATCH body has no updatable fields |
| `has_leads` | Stage delete blocked — reassign leads first |
| `name_taken` | Stage/tag name already exists in this pipeline |
| `A lead with that email already exists.` | Email uniqueness violation |

## Handling Errors (Myli Pattern)

```typescript
const res = await fetch('/api/leads', { method: 'POST', body: JSON.stringify(body) });
const json = await res.json();

if (!json.ok) {
  console.error(`CRM error ${res.status}: ${json.error}`);
  return;
}

console.log('Lead created:', json.id);
```
