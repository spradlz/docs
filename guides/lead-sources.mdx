A **lead source** is where a lead came from. Every lead carries its source (`{{lead.source}}`), and
each source sets the **defaults** a new lead inherits. Sources are the first link in the pipeline:
**Lead Sources → [Lead Flow](/guides/lead-flow) → [Workflows](/guides/workflows)**.

Managed at **Settings → Pipeline & Workflow → Lead Sources**.

## Attribution: Platform › Lead Source › Tags

Every lead carries **three levels** of where it came from:

| Level | It is | Examples |
|---|---|---|
| **Platform** | The system it arrived through (a whole integration). | `HouseLoop.ai` · `Follow Up Boss` (later: Sierra Interactive, other CRMs). |
| **Lead Source** | The finer origin **within** a platform. | Follow Up Boss → `Zillow Preferred`; HouseLoop.ai → `houseloop.ai Organic`; later → `houseloop.ai/ppc`. |
| **Tags** | The page / campaign it came from. | any tag. |

Platform is the top level (the tab axis on Trigger Events + the scope on workflows). Lead Source sits **under** a
platform. Today each platform has one source; new sources (PPC, other portals) slot under their platform later.

## The sources

| Source | Type | How leads arrive |
| --- | --- | --- |
| **HouseLoop.ai** | web | The website fires a registration/activity event → the CRM creates (or matches) the lead. The default source for everything on-site. |
| **SRG-Follow Up Boss** | import (api) | An inbound Follow Up Boss property inquiry is imported as a clean HouseLoop lead (`fub_person_id` linked), tagged Zillow + SRG FUB Import. See [Follow Up Boss → HouseLoop](/lead-sources/fub-import-and-context). |
| **Manual** | manual | A user adds the lead by hand in the CRM. |

## Per-source defaults

Each source's **Edit** modal sets the defaults a new lead from that source inherits:

| Default | Meaning |
| --- | --- |
| **AI owner** | The AI agent assigned to work the lead (e.g. Myli). |
| **Human owner** | The human the lead is assigned to — or "assign on handoff" so the AI works it until a human steps in. |
| **Start stage** | The pipeline stage a new lead lands in (e.g. New Lead). |
| **Tag** | A tag stamped on every lead from this source. |

These are defaults, not behavior — what actually *happens* to the lead (the emails, texts, stage moves)
is the **workflow** bound to each of the source's events in [Lead Flow](/guides/lead-flow).

## Sources feed Lead Flow

Every source has its own set of **doors** (events) in Lead Flow. A door is live and visible once it has
an `hl_lead_source_events` row for that source; binding a workflow to it is what makes outreach fire.

For example, the **SRG-Follow Up Boss** source has these doors seated (each maps to the FUB webhook
trigger that fires it):

| FUB trigger | Door (event) |
| --- | --- |
| Person created | `general_registration` |
| Property inquiry / tour | `fub_tour_request_registration` |
| Inquiry / question | `contact_request` |
| Seller | `seller_get_started` |
| Saved search | `saved_search` |

A seated door with **no workflow bound** still logs the lead's registration/activity row (so the
lead-detail hero card renders) but runs **no outreach** until a workflow is assigned in Lead Flow.

## Adding a new source

Add the source (name + type + defaults). Existing workflows + `{{lead.source}}` cover it — there's no
need to rebuild flows per source. Bind workflows to the new source's doors in Lead Flow when ready.
