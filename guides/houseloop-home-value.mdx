## What this is

Every home on houseloop.ai has a **HouseLoop Value** and a **HouseLoop Home Value History** — our equivalent of
Zillow's Zestimate + "Zestimate history." The value is a single current number; the history is a **monthly series**
of that value going back as far as the data allows, rendered on the home page as either a **graph** or a **table**.

One home = one **HLID** = one value-history series.

## HouseLoop Value (the current number)

The value lives on the website `Listing` row (centerbeam DB), computed at read time in `lib/listings/get.ts`:

| Field | Meaning |
|---|---|
| `zestimate` (+`zestimateLow/High`) | raw Zillow Zestimate (via OpenWeb Ninja), cached for homes that have it |
| `hlEstimate` (+`hlEstimateLow/High`) | **the HouseLoop Value** = `zestimate × 1.005` |
| `hlEstimateAt` · `hlEstimateSource` | when it was computed · provenance |

The value resolves through tiers (active = list price × 1.005; sold/past-client = cached Zestimate; everyone else =
last sold price × the ZHVI area appreciation index). The current number is always live from these tiers.

## HouseLoop Home Value History (the monthly series)

### Where it lives — `hl_value_history`

A dedicated table in the website DB, **one row per HLID per month**, append-only:

| Column | Meaning |
|---|---|
| `hlid` | the home's HLID (permanent id) |
| `month` | first-of-month bucket (e.g. `2012-01-01`) |
| `value` (+`value_low`/`value_high`) | the HouseLoop value that month |
| `source` | `backseed` (historical) · `current` (the latest month) |
| `created_at` · `updated_at` | audit |

Primary key `(hlid, month)` — every monthly write is idempotent (re-runs never duplicate a month). It's a real
relational table, not a JSON blob, so the series is queryable, immutable per point, and cheap to append at scale.

### How it's built — back-seed to the earliest anchor

The series is generated by **one algorithm** (shared with the live graph): start at the home's **oldest real sale**
(which can predate 2012), otherwise floor at **2012** using the Zillow Home Value Index (ZHVI, `hl_market_index`),
then climb monthly to today's HouseLoop value. The percentage shown spans the whole line ("+X% since {year}").

The back-seed is **free** — pure index math, no Zillow API call.

### How it stays current — monthly append

A registered Inngest function, **`value-history-persist`**, owns both jobs:

- **Back-seed** (first time a home is seen): writes the full computed monthly series.
- **Monthly append** (every run): writes the current calendar month's point from the live HouseLoop value.

It runs on a **monthly cron (27th)** — after the ZHVI ingest (24th) — and can be triggered manually with the
event `value-history/persist.run` (`{ limit?, hlids?[] }`) for back-seeds and ad-hoc runs. Per
[Run the pipeline through Inngest](/guides/listing-intake-pipeline), it is a registered function, not a one-off script.

**Target:** the current **past-client homes** (`hl_past_client_homes`). The worklist is derived from that table each
run, so new past clients are covered automatically as they're added.

### How it renders

`getListing(hlid)` reads `hl_value_history` as the source once a home is seeded (falling back to the computed line
for any home not yet seeded), and exposes it as `houseLoopValue.history`. On the home page, the **HouseLoop Value**
hero shows the value range plus a **HouseLoop Home Value History** section with a **Graph ↔ Table** toggle:

- **Graph** — an interactive line + scrubber.
- **Table** — the Zillow-style monthly table: Date | HouseLoop Value, newest first, with "+X% since {year}".
