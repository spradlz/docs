There is **one** home for HouseLoop brand assets. Don't re-draw the logo in code (for a
fixed-color context) and don't copy these files anywhere — **reference the canonical URL.**

- **Canonical URL namespace:** `https://houseloop.ai/brand/<name>`
- **Machine-readable manifest:** [`https://houseloop.ai/brand/manifest.json`](https://houseloop.ai/brand/manifest.json)
- **Authed tool endpoint (inside the CRM):** `GET /api/brand/assets`
- **Repo source:** `apps/houseloop-web/public/brand/`

Brand color **`#2D6DFF`**. Wordmark: "House" `#0A1F44`, "Loop" `#2D6DFF`.

## For AI workflows — how to GET a logo

When a workflow needs to put the logo on an email, page, or asset:

```bash
# 1. Read the manifest
curl https://houseloop.ai/brand/manifest.json
# 2. Pick the asset by key (logo_full for an email header, og_image for a share card)
# 3. Embed its url — e.g. in an email:
#    <img src="https://houseloop.ai/brand/logo-full.png" alt="HouseLoop" width="170">
```

Inside the CRM (TypeScript), import from the one module — never hardcode a path:

```ts
import { BRAND, BRAND_ASSETS } from '@/lib/brand/assets';
// BRAND.logoFull, BRAND.logoMark, BRAND.ogImage
// BRAND_ASSETS.logo_full.url, .width, .height, .use
```

## Raster (this SSOT) vs the in-app vector mark

These hosted PNGs are the source of truth for **fixed-color** contexts: emails, OG,
marketing, docs, AI content, and logo *displays* (e.g. CRM Business Settings).

A separate **recolorable SVG mark** still exists for UI chrome that must adapt to its
background — e.g. the website header renders a **white** logo on the blue bar (a PNG can't
recolor). That vector mark mirrors the canonical design (roof + loop). Don't invent a new
one; if you need a fixed-color logo, use the assets above.
