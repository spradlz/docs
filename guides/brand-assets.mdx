There is **one** brand kit for HouseLoop. Every asset is a hosted PNG at `houseloop.ai/brand/`.
Don't re-draw logos in code, don't hardcode brand colors, don't copy files — **reference the URL.**

- **Canonical URL namespace:** `https://houseloop.ai/brand/<name>`
- **Machine-readable manifest:** [`https://houseloop.ai/brand/manifest.json`](https://houseloop.ai/brand/manifest.json)
- **Repo source:** `apps/houseloop-web/public/brand/`
- **Canonical HTML source for lockups:** `apps/houseloop-web/public/brand/lockup/canonical-source.html` (bars) + `lockup/_render-vertical.html` (vertical stacks)

## Rules

- **Lockups are images.** Use the hosted PNG. Never re-render a lockup in HTML/CSS/JSX.
- **Lockups are TRANSPARENT.** Every composed lockup PNG has a transparent background. A lockup
  rendered on a baked-in white crop is a broken asset — re-render and replace it, never ship it.
- **Email signatures are images in docs.** The actual signature is HTML stored in the DB, but Mintlify can't render it — use the hosted PNG screenshot.
- **HouseLoop logo** = `<img src="https://houseloop.ai/brand/logo-full.png">` — never CSS text.
- **eXp logo** = `<img src="https://houseloop.ai/brand/partner/exp-realty.png">` — never text.
- **eXp on dark bg** = use `exp-realty-white.png`. **HouseLoop on dark bg** = use `email/email-logo-white@2x.png`.
- **Email signatures** = max-width 480px, agent info LEFT | vertical line | lockup RIGHT (Lockup D by URL). Never stretch to fill screen width.
- **Canonical lockup HTML** (for email signatures that need inline HTML) lives at `apps/houseloop-web/public/brand/lockup/canonical-source.html`; the vertical stacks regenerate from `lockup/_render-vertical.html` + `_render-vertical.mjs` (Playwright, transparent screenshot).
- **Enforcement:** `tools/houseloop-drift-check.sh` CHECK 6 blocks inline logo redraws and hardcoded brand hex.

## For AI workflows

```bash
curl https://houseloop.ai/brand/manifest.json
# Pick asset by key, embed its URL:
# <img src="https://houseloop.ai/brand/logo-full.png" alt="HouseLoop" width="180">
# Lockups live under the lockup_* keys, e.g. lockup_vertical_spradling.
```

Inside the CRM (TypeScript):
```ts
import { BRAND, BRAND_ASSETS } from '@/lib/brand/assets';
```
