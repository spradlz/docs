The lead detail page (`/leads/[id]`) is where an operator works a single lead. It is the hub for a set of
surfaces that each have their own deep-dive doc — this page explains how they fit together, and links out to
the detail.

## The three-lane shell

The page is a fixed three-column grid; each column scrolls independently.

| Lane | What it holds |
|---|---|
| **Left** | The lead + the human agent (and the agent's AI clone). Identity header, contact, agent/collaborators, tags, the **AI Mode** toggle. |
| **Center** (the "iPad") | The navy activity panel — hero KPI cards, the Buyer/Seller stage stepper, the activity streams, and the dock. **This panel never changes shape** — see below. |
| **Right** | **Myli**, the assigned AI agent — status, context, brain map. |

The left and right lanes are built from the same shared primitives so they stack pixel-identically — see
[Lead Detail — Parity Lanes](/guides/lead-detail-parity-lanes) and
[Lead Detail — AI Agent](/guides/lead-detail-ai-agent).

## The center iPad never changes

The center panel always renders the **active baseline**, regardless of the lead's mode:

**Hero KPI cards → Buyer/Seller stage stepper → activity streams → the dock.**

The dock tabs are always: **All · Texts · Calls · Emails · Notes · Appts · Tasks · Website · Deals**. A deal is
added on the **Deals** tab; the stage stepper reads the lead's Buyer/Seller pipeline. Nothing about this panel
mutates when you switch modes — modes render a band *above* the lanes instead (next section).

<Note>
Deals are always **added on the iPad Deals tab.** The TC Dashboard band renders the transaction, but it is not
where you create one.
</Note>

## Modes render a band above the lanes

Two modes drop a full-width **dashboard band above the three lanes** — both the same fixed height, different
content — while the iPad below stays identical. Modes are driven by the URL `view` param (no new routes):

| `?view=` | Band shown | Toggle |
|---|---|---|
| `agent` (default) | none — just the iPad | — |
| `tc` | **TC Dashboard band** | the left-lane **Agent ⟷ TC** switch |
| `referral` | **Referral Dashboard band** | the left-lane **Active ⟷ Referral** switch |

The Referral view is a *view*, not a status flag — it can be opened on an active client as well as a past client.

## AI Mode (per lead, per surface)

Each lead has an **AI Mode** toggle in the left lane. When on, inbound texts/emails for the lead are dispatched
to the assigned AI agent (Myli by default) — the human owner is still notified; AI Mode adds the AI loop, it does
not suppress the human. The AI cockpit (Active Triggers → Current Workflow → Current Action) on the right lane is
gated on this flag. See [Lead Detail — AI Agent](/guides/lead-detail-ai-agent).

## Deep-linking

Every surface is addressable via URL params so a notification or another screen can open the lead exactly where
it should land:

```
/leads/[id]?view=<agent|tc|referral>&loop=<loopId>&tab=<Tab>&subtab=<SubTab>&focus=<id>
```

Full param reference: [CRM Deep-Linking](/guides/lead-detail-routing).
