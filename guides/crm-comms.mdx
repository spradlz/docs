## The one-writer law

Every person-to-person message — SMS, iMessage, inbound/outbound — is written to **`hl_messages`** through
**ONE function, `logMessage()`** (`lib/messages/log-message.ts`). Nothing else inserts into `hl_messages`.
That function also mirrors the message into the canonical activity feed (`hl_lead_activity`) with the right
`kind` (`text_sent` / `text_received` / `email_sent` / `email_received` / `call_logged`). `direction` is
locked to `'inbound' | 'outbound'` by a CHECK constraint (mig 0351) — the old stray `'in'/'out'` rows were
normalized and can't drift again.

<Warning>**SSOT scope (audit correction 2026-07-06):** `hl_messages` is the one table for **texts +
iMessage**. Email and calls have their own tables (`hl_email_send_log`, `hl_calls`) and are NOT mirrored
into `hl_messages` today — the "one table for ALL comms" claim was aspirational. The one-writer law holds
per-channel; a cross-channel union is a future step, not a current guarantee.</Warning>

## Inbound routing — by DESTINATION identity

A text/email/iMessage lands AT one of our owned identities (Myli's number/email/BB line, or an agent's).
**Whoever's identity was contacted is who answers** — routing is channel-agnostic and keyed to the
destination address, not the lead's number (`lib/comms/resolve-inbound-owner.ts`). This is the reverse of
`resolveAgentSendIdentity` (lead → who sends).

## Calls — the stack

- **Inbound voice** hits `/api/phone/webhooks/voice`. Order: (1) caller-based forwarding **rules**
  (`hl_phone_forward_rules`, Settings → Phone) win for any caller; (2) else the **FUB front-router** hands
  non-HouseLoop callers to FUB; (3) else the per-dialer default forward. A dead-end **records a voicemail**
  (`<Record>` → `/api/phone/webhooks/recording`), which seats the `hl_calls` row + fires
  `phone_call_received`.
- **`hl_calls.lead_id` is nullable** (mig 0351): an unknown-caller call still seats (was silently
  insert-failing before, losing the call + recording).
- **Twilio webhooks are signature-verified** (`lib/phone/twilio-verify.ts`, X-Twilio-Signature HMAC-SHA1) —
  a forged POST is 403'd before any write or dial.
- Idempotency: both the voice seat (NOT EXISTS on call_sid) and the recording UPDATE (recording_url IS NULL)
  guard against Twilio's retry double-fires.

## Send identity

Outbound is sender-configured, never hardcoded: `resolveAgentSendIdentity(leadId)` picks the acting
identity (Myli default → the agent's clone if configured → the human owner). System/brand sends are
agent-agnostic (HouseLoop Team). Numbers: Myli iMessage/SMS on the one number via BlueBubbles; Twilio for
voice + SMS on the dialer lines.

## Where it's managed
Settings → Phone (dialers + forwarding rules) · Settings → Voice & Recording · Settings → iMessage.
Run Log surfaces every call-analyze / sms-dispatch run.
