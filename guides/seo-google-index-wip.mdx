## The build

### 1. Submit script — `tools/seo/gsc-submit-sitemap.mjs` (✅ built + tested)
Mints an SA access token (the SA acts **as itself** — JWT `iss`=SA email, **no `sub`** impersonation, scope
`https://www.googleapis.com/auth/webmasters`, RS256-signed — mirrors `lib/google/dwd.ts` minus the DWD
subject), then `PUT`s `https://houseloop.ai/sitemap-index.xml` to
`webmasters/v3/sites/sc-domain:houseloop.ai/sitemaps/<encoded>` — expects **204**. `--status` reads current
state without submitting. Tested 2026-06-27 → token mint OK, returns 403 (SA lacks property access).

### 2. The one human step (blocker)
The SA mints a valid write token but isn't a GSC owner. An **existing owner** must add it (one-time):
**GSC → pick the `houseloop.ai` Domain property → Settings → Users and permissions → Add user →
`houseloop-master@houseloop-496200.iam.gserviceaccount.com` → Owner.** Then re-run the submit script → 204.

### 3. Weekly Inngest monitor (building)
Per `INNGEST_OR_IT_DOESNT_EXIST`: a registered weekly fn that re-submits the sitemap + pulls the
coverage / URL-inspection report (indexed vs excluded + reasons), stores a row, and Telegram-pings Zach.
Registered in `apps/houseloop-web/app/api/inngest/route.ts`, captured in the Run Log.
