## What's live

### 1. Submit script — `tools/seo/gsc-submit-sitemap.mjs` (✅ DONE)
Mints an SA access token (the SA acts **as itself** — JWT `iss`=SA email, **no `sub`**, scope
`https://www.googleapis.com/auth/webmasters`, RS256-signed), then `PUT`s the sitemap to
`webmasters/v3/sites/sc-domain:houseloop.ai/sitemaps/<encoded>`. **Returned 204 on 2026-06-27** — sitemap
registered, `lastDownloaded=2026-06-27`, 0 warnings / 0 errors, Google processing. `--status` reads without
submitting. (Earlier 403 → fixed by adding the SA as a GSC Owner.)

### 2. Weekly monitor — `lib/inngest/functions/seo-index-monitor.ts` (✅ built, ⏳ verify-run)
Registered Inngest fn `seo-index-monitor` (cron `0 14 * * 1` = Mon 14:00 UTC ≈ 8am CT, + manual event
`seo-index-monitor/manual.run`). Each run: re-submits the sitemap, pulls GSC's index data (sitemap child
counts + a sample live `/home` page's crawl/index verdict + 28-day traffic), and Telegram-pings Zach the
climb. Auto-captured in the CRM Run Log (capture-fn subscribes to `function.finished`). Auth via the new
`mintSAAccessToken()` in `lib/google/dwd.ts`. **After the Railway deploy:** `curl -X PUT
https://houseloop.ai/api/inngest` to re-sync, then Invoke `seo-index-monitor/manual.run` to verify the first run.
