## The one idea

**A lead is not a lookup — it's a relationship we convert, deliberately, from the first text to the
signed showing to the follow-up that never quits.** The old way answers the *what* ("yes, 3pm works").
We answer the what **and** win the *who*. This section is that arc, in order.

Each chapter owns exactly one stage. The boundaries are sharp on purpose (rule 2 below): if a fact could
live in two chapters, the boundary is wrong.

## Where a lead enters

- **A Follow Up Boss inquiry / tour request** (the front door we tune most). The import grounds the first
  touch on the *real* inquiry — the home, the requested time, the lead's own words — never a synthetic seed.
  The conversation is driven by the [Connecting With People](/guides/connecting-with-people) method as the
  clone of the human agent. Entry mechanics live in [Routing & the Care Team](/guides/care-team-routing).
- **A website signal** (registration, a liked/saved home, a saved search) grounds on the pixel history.
- Either way, the lead lands on **one active workflow** and one staffed owner, then flows down the arc.

## SPEC vs LIVE — read the tags

Every claim in these chapters is tagged **LIVE** (shipped, with a `file:function` citation) or **SPEC**
(target design, not yet wired). Never read a design goal as a shipped feature — that mislabel is exactly
what rotted the old docs. When a wire ships, its tag flips. Showing Coordination in particular is mostly
**SPEC** today with a "built vs. wire" ledger at the bottom of that chapter.

## The six rules that keep this section from drifting

This section replaced three overlapping, drifting docs. It stays un-drifted only if every edit obeys these
(they are this section's operating contract — a superset of
[Docs Section Discipline](/mintlify-how-to-guide)):

1. **One fact, one home.** A fact lives in exactly ONE chapter. Everyone else **links** — never restates.
   The tools live in the AI Toolbox; the stages live in Routing; the beats live in Connecting With People.
   If you're explaining the same thing twice, one copy becomes a link.
2. **Disjoint chapters.** Each chapter owns one stage with a sharp boundary. A paragraph that could live in
   two chapters means the boundary is wrong — sharpen it.
3. **The Overview is a map.** This page links down and never copies, so the map can't diverge from the
   territory.
4. **Cite the code.** Anything LIVE names its `file:function` or table, so drift is checkable against the
   code — the table is the truth.
5. **Tag SPEC vs LIVE** on every claim. Design never masquerades as shipped. Flip the tag when it ships.
6. **The consolidation gate.** Before writing a new fact, name the ONE chapter it belongs to and edit that
   chapter — never spawn a sibling. Three docs on one topic is the bug this section exists to fix.
