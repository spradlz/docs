HouseLoop runs its own social presence under the **"Spradling Group powered by HouseLoop"** strategy — fully separate from the Spradling team brand and from PlacedAgent. Every channel is owned by HouseLoop's own accounts so it can be managed (and one day sold/transferred) as a standalone product.

All three channels share one handle: **@houseloopai**.

## The "standalone product" model

A Facebook Page and an Instagram Business account are **assets, not logins** — you never "log into Facebook as HouseLoop." What makes them separable is the **Business Portfolio** that owns them, plus a **System User token** that the bot uses to post with no human in the loop.

- **Business Portfolio "HouseLoop"** — `985346324392840`. The container that owns the Page + IG. This is the transferable unit.
- **Human admin** — Zach's personal Facebook profile. Meta requires a real person at the root; this is by design and unavoidable. Day-to-day posting never uses it.
- **Instagram** is the one channel with its own discrete login (`admin@houseloop.ai`), because IG accounts can be created standalone and then linked to the Page.

<Note>
The **Graph Page ID** (`1124401867423716`) differs from the **Studio/asset ID** (`61590429638312`) Meta shows in some URLs. Always use the Graph ID for API calls. Likewise the IG **user ID** for publishing (`17841424760298572`) differs from the IG **business-asset ID** (`1070475209492397`).
</Note>

### Publishing to Instagram

Two-step container flow on the IG user ID:

```bash
# 1) create a media container
POST https://graph.facebook.com/v21.0/17841424760298572/media
  image_url=<public image URL>&caption=<text>&access_token=$HOUSELOOP_INSTAGRAM_ACCESS_TOKEN

# 2) publish it
POST https://graph.facebook.com/v21.0/17841424760298572/media_publish
  creation_id=<id from step 1>&access_token=$HOUSELOOP_INSTAGRAM_ACCESS_TOKEN
```

### Verify the token

```bash
GET https://graph.facebook.com/v21.0/me/accounts?access_token=$HOUSELOOP_INSTAGRAM_ACCESS_TOKEN
GET https://graph.facebook.com/v21.0/1124401867423716?fields=name,instagram_business_account&access_token=$HOUSELOOP_INSTAGRAM_ACCESS_TOKEN
```

The second call returns the linked `@houseloopai` account — proof the token can reach the IG profile.

## Gotchas (these ate real time — read before touching the bot)

- **"Add Instagram account" only *connects an existing* IG.** You must create `@houseloopai` first (desktop `instagram.com/accounts/emailsignup` in an incognito window, since the IG app shortcuts already-logged-in accounts). Then connect it.
- The desktop Business-Suite connect can throw **"Invalid redirect URI"** (a flaky Meta OIDC bug for new accounts). The reliable path is the **Instagram app → Accounts Center → Add Facebook account** → pick the HouseLoop Page.
- **Each brand's Instagram has its own login** — not the personal Facebook.
- The System User needs the **app assigned as an asset** (Assign assets → Apps → Manage app) or the generate-token wizard shows "no permissions available."
- **Facebook feed posting** needs the "Pages" use case added to the app — only Instagram scopes are offered under the Instagram use case the app was created with.
