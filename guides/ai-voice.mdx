# ═══ 01 · THE STACK (one call, three senses) ═══

Myli is built on **Pipecat** (~11.9k★, Daily.co) — the industry-standard voice framework that solves the hard
audio problems (barge-in, turn-taking, VAD, low-latency streaming, the Twilio transport) as defaults. We bring
only the logic: her persona and the CRM wrap-up. The engine is a **cascade** (separate STT → LLM → TTS), which
is what the whole industry runs for telephony — raw speech-to-speech is banned here after Gemini crashed on
8 kHz phone audio (see HISTORY).

The signal flows top → bottom:

| Layer | Choice | Billed by | Notes |
|---|---|---|---|
| **Transport** | Twilio Media Streams → `<Connect><Stream>` | per-minute | Live audio WebSocket to the bot; no SIP. |
| **Ears (STT)** | **Deepgram** (`nova`) | per audio-minute | Top-tier; unchanged. `DEEPGRAM_API_KEY`. |
| **Brain (LLM)** | **Claude Haiku 4.5** (`claude-haiku-4-5-20251001`) | per token | Funded, fast (= low phone latency), one-provider. `ANTHROPIC_API_KEY`. |
| **Mouth (TTS)** | **Cartesia Sonic-3** (voice *Skylar*, `db6b0ed5…`) | per character | ~40 ms TTFA latency leader + wins blind naturalism. `CARTESIA_API_KEY`. |
| **Seam** | CRM `persona` + `wrapup` | — | Persona in at call start; the one write out at hangup. |

**The runtime files** (`apps/myli-voice-agent/`): `pipecat_bot.py` (the bot) · `requirements-pipecat.txt`
(the deps — Pipecat 0.0.108 on Python 3.12) · `.venv-pipecat/` (the local venv). Every model/voice value is
env-overridable (`MYLI_LLM_MODEL`, `MYLI_CARTESIA_VOICE`, `MYLI_CARTESIA_MODEL`) so we retune without a code change.

# ═══ 03 · HER PERSONA (config, not code) ═══

Composed at `GET /api/phone/myli/persona` — `hl_role.persona` (who she is everywhere) + `hl_role.voice_persona`
(phone behavior) + the `agent.*` / `team.*` merge tokens, with the current Central date/time prepended. Edited
in the CRM (**My Team → AI Receptionist** role + the AI Voice persona layer), never in the bot. It is
**goal-driven, not scripted** (a scripted version sounded robotic and was reverted):

- **Identity** — Myli, Spradling Group's front desk. Warm, upbeat, casual, quick — never a stiff IVR.
- **One hard rule** — one question per turn. She listens; she doesn't monologue.
- **Triage gate** — buyer / seller / something else, first move (fixes the "sell → your current home" drift).
- **Past-agent line** — names the departed agent the caller was reaching (reactivation gold).
- **Goal** — capture who + what, take a callback time, hand off clean. Never fakes being human. Never live-transfers.
- **On hangup** — writes the lead + transcript + a callback task automatically.

# ═══ 04 · THE CRM SEAM (Pipecat, all on prod) ═══

The CRM voice webhook (`app/api/phone/webhooks/voice/route.ts`) is the front door. For an `answer_mode='myli'`
line it returns:

```
<Connect><Stream url="wss://{pipecat_ws_host}/ws" /></Connect>
```

bridging Twilio Media Streams to the bot's WebSocket. The host lives in
`hl_settings.phone_provider.pipecat_ws_host` (env `PIPECAT_WS_HOST` wins), so it repoints with **no redeploy**.
If `<Connect>` ends (bot down / no host) the call falls through to the same `<Record>` voicemail — **instant
rollback is clearing the host**. The bot then talks to two endpoints only:

| Route | Method | One line |
|---|---|---|
| `/api/phone/myli/persona` | GET | The composed receptionist persona (cold-receptionist gating is server-side). |
| `/api/phone/myli/wrapup` | POST | **THE ONE WRITE SEAM** — every post-call CRM effect. |

**The wrap-up receipts (one POST, every effect):** spam → silent flag (no lead, no ping) · find-or-**create**
the lead (door law: source `phone_call`, assignment/stage from the source defaults, never hardcoded) · Myli
joins the care team · `hl_calls` gets transcript/summary/role/callback · a callback task on the assigned agent ·
`logActivity` `call_logged` (the All-feed + AI Follow-Up row) · `notifyLeadEvent` → the assigned agent · **the
cost writes (below)**.

**The answer gate — 3 layers, all must pass:** (1) org master `hl_settings('ai_conversations').voice_enabled`
(fail-closed) · (2) per-line `hl_phone_numbers.answer_mode='myli'` · (3) role capability `voice_inbound`.

# ═══ 05 · THE COST WIRE — nothing unattributed ═══

<Check>
**A live Myli call now lands on `/ai-cost` under its own `ai_voice` lane** (wired 2026-07-17). Before today the
voice call never fired the cost choke point, so voice minutes were invisible on the sheet. Fixed.
</Check>

One voice call = **three vendors on three billing models**, so it can't be a single token count. The bot meters
the real usage during the call and the wrap-up bills it once:

- **The bot** (`pipecat_bot.py`, `MyliUsageObserver`) sums Pipecat's `MetricsFrame`s — LLM prompt/completion
  tokens + TTS characters — and reports call wall-clock seconds, in the wrap-up payload's `usage` object.
- **The wrap-up route** computes the blended cost from real usage: Haiku tokens priced from `MODEL_PRICING`
  ($1/$5 per M) + **Deepgram** ≈ $0.0058/audio-min + **Cartesia** ≈ $22/M chars + **Twilio** ≈ $0.0125/min.
- It writes cost **two ways, no double-count** (different surfaces): (a) `stampActivityTokens` on the `hl_calls`
  row = the per-lead activity view; (b) `logAiCost({category:'ai_voice', costUsd})` = the ONE choke point that
  fires `houseloop/ai.call` → `hl_run_log` → the global `/ai-cost` page. `cost_source='metered'` when the bot
  sent real usage (`'estimate'` for the legacy fallback).
- `ai_voice` is registered once in the shared categorizer (`lib/ai-cost/ai-category.ts` `KNOWN_AI_CATEGORIES`
  + `lib/ai/log-ai-cost.ts` `AiCostCategory`), so the AI Cost page and the Run Log agree — the Run Log renders
  its pill from `r.ai_cat` via the same categorizer (Benji, 2026-07-17). See [AI Cost](/guides/ai-cost).

<Warning>
The cost numbers are **wired but unverified** until the first live call produces real metrics — the per-minute /
per-char rates are approximate public list. Confirm the first `ai_voice` row against the vendor bills, then
tighten the rate constants in the wrap-up route if needed.
</Warning>
