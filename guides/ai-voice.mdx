## The research verdict (3 parallel agents, 2026-07-11 — why Retell)

- **NOW: Retell AI.** 580–620ms measured voice-to-voice on real phone calls, Twilio number import, inbound
  webhook = caller-lookup-before-she-speaks, `call_analyzed` = our wrapup POST, recordings + transcripts API,
  ~7–10¢/min, SOC2. On 8kHz phone audio the tuned cascade FEELS more human than raw speech-to-speech.
- **AT SCALE (v2 graduation path): OpenAI gpt-realtime on LiveKit + SIP trunk** — sub-300ms true S2S; budget
  for SIP scar tissue + weak multi-turn tool calling. LiveKit project rebuilt under admin@houseloop.ai
  (`p_mk0zneefhs2`) and parked for this. Retell can also flip to OpenAI Realtime as a model choice without
  re-platforming.
- **NEVER: Gemini Live on phone audio** — industry-wide unresolved `1011` mid-call-drop cluster
  (livekit/agents #2274 #1679 #5096, google adk #3918, python-genai #985). Our crashes were not our bug alone.
- Fallback #2: Vapi ($0.18–0.22/min real). Wildcard: Phonely (~350ms, ~5¢, thin track record).

# ═══ THE CRM SEAM (all deployed on prod) ═══

Six routes under `apps/houseloop-crm/app/api/phone/myli/`:

| Route | Method | One line |
|---|---|---|
| `/api/phone/myli/persona` | GET | The composed prompt (role persona + AI-Voice doctrine + overrides, merge-resolved) + `voice_enabled` + `tools_enabled` + `voice_config`. |
| `/api/phone/myli/caller` | GET | Last-10 phone lookup → the lead, for caller context. |
| `/api/phone/myli/wrapup` | POST | **THE ONE WRITE SEAM** — every post-call CRM effect (receipts below). |
| `/api/phone/myli/retell-inbound` | POST | Retell's ringing-time webhook → returns `dynamic_variables` (caller context) before Myli speaks. |
| `/api/phone/myli/retell-webhook` | POST | Retell post-call events: `call_started` seats `hl_calls` (runtime='retell'); `call_analyzed` → forwards to wrapup. |
| `/api/phone/myli/tool` | POST | Mid-call tool executor (`log_callback` · `take_message`). |

**The wrap-up seam receipts — one POST, every CRM effect:**

- **Spam** → silent flag; no lead, no notification (Zach's law — spam never pings a human).
- **Find-or-CREATE the lead** — door law: source `phone_call` (mig 0382); assignment + stage come from the
  **source defaults** (today: Zach + New Lead) 🔒 — **never hardcoded**.
- **Myli joins the care team** on the lead.
- **`hl_calls` update** — transcript, summary, role, callback window (params CAST `::text` — the 42P18 lesson).
- **Callback task** — worded "call back FROM `{line}`" per the `callback_line_policy` 🔒 (default: the dialed line).
- **`logActivity` kind `call_logged`, actorRole `ai_agent`** — this IS the AI Follow-Up feed row + the
  lead-detail All-feed row, automatically (ALL_FEED law: the writer calls logActivity, the feed follows).
- **`notifyLeadEvent` → the ASSIGNED AGENT** 🔒 — never a hardcoded person.
- **Cost stamp** — duration estimate ~$0.045/min, `cost_source='estimate'`, stamped via `stampActivityTokens`
  onto the activity row (exact Retell per-call cost replaces the estimate — build queue #5).

**Auth:** `/api/phone/myli/` is in the middleware `PUBLIC_PREFIXES`; each route self-authenticates against env
`HL_INTERNAL_SECRET` OR the DB secret `hl_settings('myli_voice').secret` via the `x-myli-voice-secret` header.
⚠️ Railway's edge 403s the `Python-urllib` user agent — any client must send a real User-Agent.
