# ‚ïê‚ïê‚ïê ‚öÝÔ∏è NEED TO DO NEXT ‚Äî MAP AI COST FOR AI VOICE ‚ïê‚ïê‚ïê

<Warning>
**Voice cost is 100% invisible today, and it's the hardest cost to track in the whole platform because it's
THREE separate vendors on THREE different billing models ‚Äî not one token count.** When the voice stack is
finalized (see status below ‚Äî we're still testing), the first work is instrumenting all three under ONE
dedicated **`ai_voice`** cost category. See [AI Cost](/guides/ai-cost) for the logging spine this plugs into.

**The three costs of one voice call (the gotchas):**

| Leg | Vendor (today) | Billed by | The gotcha |
|---|---|---|---|
| **Ears (STT)** | **Deepgram** (`nova`) | **per audio-minute** | No token count exists ‚Äî cost = call-recording **duration √ó per-minute rate**. Not in `MODEL_PRICING` (that table is per-token). Same shape as the CRM Deepgram STT hole (call/voicemail transcription). |
| **Brain (LLM)** | **OpenAI gpt-4o-mini** ‚öÝÔ∏è | **per token** | This IS token-based ‚Äî but the bot runs **OpenAI, not Claude** (docs long claimed Haiku; the code says gpt-4o-mini). A one-provider violation to resolve when we lock the stack. Turn-by-turn, so tokens accrue across the whole call. |
| **Mouth (TTS)** | **ElevenLabs** | **per character** | Cost = characters spoken √ó per-char rate. Another non-token model. |

**The requirements when we build it:**
1. A dedicated **`ai_voice`** category ‚Äî voice must NOT land in `follow_up` (that's Myli's TEXT conversations;
   Zach's confusion: "AI Follow-Up says 429 calls" = text, not voice). Add `ai_voice` to `KNOWN_AI_CATEGORIES`
   + the Run Log pills + the cost-page.
2. **Emit at the wrap-up seam** ‚Äî the bot already posts a wrap-up (transcript + summary + callback task) on
   hangup. That is the ONE place that knows the call's duration (Deepgram), the turn tokens (LLM), and the
   characters spoken (ElevenLabs). Emit a single aggregated `houseloop/ai.call` `{category:'ai_voice'}` row
   there ‚Äî OR three sub-rows if we want per-leg breakdown.
3. **Per-minute / per-character pricing** must live in the categorizer's price map alongside the per-token
   models (a small extension ‚Äî the emitter takes a pre-computed `costUsd`, so voice can pass its own math).

**Blocked on:** finalizing the voice stack. We do NOT instrument a bot we might replace ‚Äî the moment the
architecture is locked (Pipecat cascade vs. something else, and Claude vs. gpt-4o-mini brain), this becomes
the first task. Everything below is the current state that feeds that decision.
</Warning>
