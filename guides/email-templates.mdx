HouseLoop uses an AI-assisted email template system. Templates are full production HTML
with inline styles and merge field placeholders — the exact HTML that gets sent to leads
via Resend. They're stored in the CRM database and previewed in an iframe that renders
them exactly as they'd appear in a real inbox.

## How templates are created

Templates are **not** composed in a WYSIWYG editor. The workflow is:

1. **Describe** — tell the AI agent (Lee) what email you want in plain English.
2. **Generate** — Lee builds the full production HTML (logo, listing cards, CTA button, unsubscribe footer).
3. **Save** — the HTML is saved to `hl_templates` in the CRM database.
4. **Preview** — open the template in the CRM, click "Preview Email" to see it rendered in an iframe.
5. **Iterate** — request changes, Lee updates the HTML, preview again.
6. **Lock in** — when approved, the template becomes the single source of truth for that email type.

## Template anatomy

Every HouseLoop email follows a consistent 6-section structure:

| Section | Description |
| --- | --- |
| **Logo header** | White background, HouseLoop logo PNG (`/houseloop-logo.png`) |
| **Body content** | Template-specific (greeting, listing cards, market data, etc.) |
| **CTA button** | Brand blue (`#2D6DFF`) button linking to the relevant page |
| **Manage link** | "Manage your preferences at houseloop.ai/updates" |
| **Promo band** | Dark navy band: "Never miss your HouseLoop alerts" |
| **Footer** | Unsubscribe link + Privacy Policy (CAN-SPAM required) |

## Merge fields

Templates use `{{namespace.field}}` placeholders that get resolved with real data at send
time. The canonical registry is `apps/houseloop-crm/lib/merge-fields.ts`.

| Group | Fields |
| --- | --- |
| **CONTACT** | `first_name`, `last_name`, `full_name`, `email`, `phone` |
| **LISTING** | `address`, `price`, `beds`, `baths`, `sqft` |
| **AGENT** | `full_name`, `email`, `phone` |
| **MARKET** | `area_name`, `avg_price` |
| **SEARCH** | `name`, `match_count`, `url` |

Custom fields from `hl_custom_fields` automatically join via `{{contact.<key>}}`.

Dynamic blocks like `{{listing_cards}}` are resolved at send time by the specific
automation function — they're template-specific, not in the merge field registry.

## Preview system

The CRM template editor at `/settings/email-templates/[id]` has a **"Preview Email"**
button that opens the `EmailPreviewModal`:

- **Preview** — full HTML rendered in an iframe, exactly as it looks in an inbox
- **HTML** — raw source code view
- **Desktop** — 600px viewport width
- **Mobile** — 375px viewport width

The preview resolves merge fields with sample data (Jordan Avery, Overland Park, $425K)
and shows which merge fields are used in the template.

For full HTML templates (`body_source = 'html'`), the editor shows a read-only code
textarea. Editing happens through the AI agent, not the WYSIWYG.

## Template picker in campaigns

The campaign composer at `/settings/email-campaigns` includes a **"Use Template"** button
that lists all available templates. Selecting one pre-fills the campaign's subject line
and email body.

## Database schema

Templates live in `hl_templates` (Railway houseloop-crm database):

| Column | Type | Purpose |
| --- | --- | --- |
| `id` | uuid | Primary key |
| `kind` | text | Always `'email'` |
| `name` | text | Display name |
| `key` | text | Immutable slug for code references |
| `category` | text | Grouping (listing_alert, open_house, etc.) |
| `subject` | text | Subject line with merge fields |
| `body` | text | Full production HTML |
| `body_source` | text | `'html'` (production) or `'manual'` (WYSIWYG) |
| `visibility` | text | `'system'`, `'team'`, `'private'` |

## Logo

All emails use the HouseLoop logo from `/houseloop-logo.png` — a single PNG file that
exists in both `apps/houseloop-web/public/` and `apps/houseloop-crm/public/`. This is the
single source of truth for the logo across all email templates and documentation.
