Every CRM notification writes one row to **`hl_notification`**, the topbar **bell** reads it, and a
**dispatcher** delivers it over the recipient's enabled channels (in-app / SMS / email). Every email
renders in **ONE canonical branded shell** (HouseLoop logo тЖТ white card тЖТ lockup тЖТ footer) with a
**centered, Follow-Up-Boss-style body** (contact card + real message + assigned agent + button). Every
text is a clean emoji-per-line message with a `тАФ HouseLoop` sign-off. Add a notification by passing an
`event_key` тАФ no schema change.

## How it flows

```
event source                    spine                        surfaces
stage change / inbound text тФАтЦ╢ createNotification() тФАтЦ╢ hl_notification row
lead likes/tours/registers                              тФВ
AI (Myli) sends a text / logs a call                    тФЬтФАтЦ╢ bell (unread count + dropdown)
                                                        тФЬтФАтЦ╢ lead-detail dock badges
                                                        тФФтФАтЦ╢ dispatch.ts тЖТ SMS / branded email (per prefs)
```

## Architecture тАФ where everything lives

| **The branded email SHELL** (logo + footer) | `lib/email/notification-shell.ts` тЖТ `notificationShell()` |
| **The centered BODY** (contact card + message + agent + CTA + settings) | `lib/email/notification-content.ts` тЖТ `renderNotificationInner()` |
| Event тЖТ template mapping | `lib/notifications/template-registry.ts` (`NOTIFICATION_TEMPLATES`) |

| Merge fields | `lib/merge-fields.ts` (preview) + `lib/email/template-merge.ts` (send) |

**Email templates are HEADLINE-ONLY.** The per-event `hl_templates` body is just the centered headline
(e.g. `тЭдя╕П {{contact.full_name}} liked a home`). `dispatch.ts` wraps it in `renderNotificationInner()`
(the shared chrome: contact card + message + assigned agent + Open-lead button + settings link) then in
`notificationShell()` (logo + footer). So editing a template controls the *message*; the brand chrome is
always correct. Never hand-roll a `<div>` email.

## The email body (centered, FUB-style)

```
              [ HouseLoop logo ]
      ЁЯПа You've received a new lead named James Ross from Zillow Preferred
              тХнтФАтФАтФАтФАтФАтХо   ЁЯУЮ (785) 330-6800
              тФВ  JR тФВ   тЬЙя╕П james.ross@email.com
              тХ░тФАтФАтФАтФАтФАтХп   ЁЯПа 6213 E 109th Ter тЖТ   (address тЖТ /home link)
        "I am interested in 6213 E 109th TerтАж Requested Tour Mon Jun 22 2pm"
                   Assigned Agent ┬╖ Ana Spradling
              [  Open lead in HouseLoop CRM  ]
                 Change notification settings
              [ HouseLoop | eXp ┬╖ footer ]
```

- **Contact card** тАФ avatar (initials) left, phone / email / address right. Address links to `/home`.
- **Inbound message** тАФ **source-agnostic** `resolveInboundMessage()` in `dispatch.ts`: FUB
  (`fub.events.message` via the `hl_leads.fub_person_id` bridge) **or** HouseLoop (latest inbound
  `hl_messages.body`). Only shown when present.
- **Assigned Agent** тАФ the lead's owner (`hl_leads.assigned_user_id` тЖТ `hl_users.name`). Admin-level.
- Team/system alerts (no lead) = just the centered headline + settings link.

## The AI (Myli) format

AI-agent notifications use **тЬи** (not ЁЯдЦ) and lead with the AI agent's name:

```
        тЬи Myli sent James Ross a text
   "Hi James! Thanks for your interest in 2116 NW Harbor Place. Free Tuesday at 2pm for a tour?"
                       Myli
             [ Open lead in HouseLoop CRM ]
```

- Headline `тЬи Myli {action}` тАФ `dispatch.ts` composes `Myli ` + the lowercased action summary.
- **Action payload block** (`resolveAiDetail()` in `dispatch.ts`): the real sent text (`hl_messages`
  outbound, `actor_role` myli/ai_agent), the call date ┬╖ length + **View transcript** link
  (`hl_calls`), or the notification body for facts / homes / stages / appointments.
- Agent line shows just **Myli** (no "AI Agent" label).

## Data model

- **`hl_notification`** тАФ append log (`recipient_user_id`, `event_key`, `title`, `body`, `lead_id`,
  `entity_type`/`entity_id`, `payload`, `channels_sent`, `seen_at`, `read_at`). Migration `0045`.
- **Prefs** тАФ `hl_users.data.notification_prefs` (`{ eventKey: { email, sms } }`) + `notification_channels`.
  Defaults in `app/api/notification-preferences/route.ts` **and** `dispatch.ts` `PREF_DEFAULTS` (keep in sync).
- **Templates** тАФ `hl_templates`, keys `notify_<event>` (email) / `notify_<event>_sms` (text).

## Sign-off harness

`scripts/notification-matrix.ts` renders **every** notification's email + text with sample data through
the SAME primitives as `dispatch.ts`, and sends both to a target for sign-off:

```
npx tsx --env-file=.env.local scripts/notification-matrix.ts [email] [phone]
MATRIX_ONLY=ai_calls,ai_messages ...   # a subset
MATRIX_EMAIL_ONLY=1 ...                # skip SMS
```
