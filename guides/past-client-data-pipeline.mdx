This is the canonical developer reference for the **Spradling Realty Group past-client data project** — one of the largest data lifts in HouseLoop. It powers the CRM seller-report dashboard, every past-client home's `/home` page, **the Follow Up Boss past-client dossier**, and the **CRM Audit Report** (`Seller Report → Audit Report`). The internal strategy lives in `GT_HOUSELOOP_SG_PAST_CLIENTS_PRD.md` §12–§13; this page is the **end-to-end process a developer needs to operate it.**

### THE ARCHITECTURE LAW (Zach LOCKED 2026-06-23) — the resolver owns the output, the transport is dumb

> **The S6 LLM resolver OWNS the entire output.** Every byte that takes judgment is written by the resolver: the household, the **labeled contacts**, the **labeled transactions** (BOUGHT vs SOLD), the **disposition**, the **DNC screen**, the **tags**, AND the human **NOTE**.
>
> **The transport (`dossier-to-fub.mjs`) is DUMB** — it writes the resolver's finished block to FUB + the SSOT and NEVER re-derives anything. **If the output is wrong, fix the RESOLVER prompt/schema, re-run, re-transport.**

Born from a day of regressions where hand-rolled JS (`roleOf`, `lastNameOf`, contact regex, current-home heuristic) broke what the resolver had right and burned ~$50. Never again: no judgment in the transport.

## THE LIVE-DEED DISPOSITION — county deed is primary truth

For every home the client bought, the resolver re-derives the disposition **live** (never a stored flag):

1. **County deed** (`verifyOwner()` in `county-owner-verify.mjs`; Johnson/Wyandotte KS + Jackson/Clay/Cass MO) is **primary truth.** `owner_match` (client on the deed) = they own it; `owner_differs` (a stranger owns it) = they sold/moved.
2. **Repliers** gives the clean sold date/price/office; **OpenWeb Ninja** fills the listing/buyer **agent names** Repliers lacks.
3. **DEED-LAG RULE (the Nations case):** county deed recording trails a closing by weeks, so a deed still showing the client does **not** mean they didn't sell. A **recent, real** MLS sale (arm's-length: negotiated price and/or a buyer agent) **BEATS** a lagging deed → `sold=true`, `deed_lag=true`, note appends "(deed not yet recorded)."
4. **ARBORIDGE ARTIFACT GUARD:** a "sale" priced **exactly** at list with **no buyer agent** is a non-arm's-length artifact (refi/relo/quitclaim), NOT a sale → the deed (owner_match) wins, the home stays owned. (Chari still owns 3216 SW Arboridge.)
5. For a sold home, fetch the **full MLS listing** (`repliersSoldCheck(mls)`) for the real `raw.CloseDate` — the address-history endpoint returns the contract date, not the close.

**⚠️ PARCEL LAW (the Sycamore trap):** Repliers' `addressKey` ignores ZIP, county, and the N/S/E/W directional → it MERGES different houses sharing street#+name. A re-sale counts only if the PARCEL matches: zip + county + neighborhood + legal description + coordinates within ~150m.

## CHAIN OF CUSTODY — the agent on every deal

Every `transactions[]` line carries `agent` = the **Spradling Group agent who repped US on THAT specific deal** (per-deal, not per-client — an Ana-handoff client shows Ana on the early deals and the handoff agent later). Source: SISU `agent` of the matching closing, else the closing's `agent`; fallback `"SRG"`. It renders as `— <agent>` on every transaction line in the FUB note and as the **Agent (chain of custody)** column in the Audit Report.

## 🚫 AGENT DO-NOT-CONTACT SCREEN (Step 0) — never reactivate our own agents

We never market to our own SG agents or their immediate family. Before anything else the resolver:

- Loads the **agent roster** = distinct names in the SISU `agent` field (the agents who repped our deals).
- `checkAgent()`: **exact full-name match** → the person IS an agent → **Agent DNC** (auto). **Surname-only match** → POSSIBLE agent-family → DNC **only** with a confirming signal in the FUB notes/deals: an "Agent COI" note, an agent spouse/co-owner on the deed, or a deal where that agent repped the OTHER side. A surname coincidence with **no** signal stays a normal client.
- On DNC: `note_markdown` becomes a short **"DO NOT CONTACT — Agent / Agent Family (reason)"** card (no reactivation dossier), `tags=["HouseLoop Resolved","SRG Closed - Agent DNC"]`, and the CRM **bucket flips to `agent-dnc`**.

Proven: **April Atkinson** (exact agent match; the resolver even flagged her $0-GCI "closing" as an internal agent record) and **James Duston** (surname `Duston` + the COI note + the `DUSTON JAMES & STEPHANIE` deed + SISU lead-source `agent coi` → husband of agent Stephanie Duston).

## THE CARD — the Follow Up Boss note format (LOCKED)

The `note_markdown` is EXACTLY these sections, nothing else (no free-form reasoning or data-quality musings — those go in `card_errors`, internal):

```
HouseLoop · Spradling Group Past Client — <Bucket>
Resolved <MM/DD/YYYY>

CONTACTS
• <Name> — <phone> · <email>

TRANSACTIONS WITH US
• <date> BOUGHT/SOLD <address>  $<price>  — <agent>

CURRENT
• Current home: <addr> — county-deed confirmed (<owner>).   (or: Current residence: not yet verified)
• Still owns: <addr> (investment) · <addr> (investment)
• SELLER LEAD — <addr> sold <date> for $<price>, listed by <agent> (<broker>) — we lost this listing.  (only when deed-proven)

DOSSIER
<2–4 sentence "know them before you call" brief: who they are, our history + which agent, where they stand
 now, and the one actionable angle. Grounded only in the facts — never invented.>
```

A **DNC** card replaces all of the above with the short suppression note.

## The Audit Report (`Seller Report → Audit Report`)

A full-width CRM page (`apps/houseloop-crm/components/seller-report/AuditReport.tsx`, fed by `/api/settings/seller-report/audit` reading `resolver_block` live). It's the "gold" audit layout, intentionally separate from the tokenized CRM table (an ESLint exception), with the fonts/buttons/header tokenized. A sticky jump bar: **All · Transaction History · Resolution Matrix · Follow-Up Boss Notes**.
- **Transaction History** — every deal + the Agent (chain of custody) column + the live deed verdict.
- **Resolution Matrix** — per client: bucket, status, current residence, per-home deed verdicts, lost listings (with agent), contacts, tags.
- **Follow-Up Boss Notes** — every note verbatim.
- **Each client name links to their Follow Up Boss profile** (`https://spradlinggroup.followupboss.com/2/people/view/<fub_id>`).

## The resolver's output block — `resolver_block` jsonb on `hl_srg_past_clients`

`contacts[]` · `transactions[]` (with `agent`) · `disposition` (current_residence, still_owns, lost_listings) · `dispositions[]` (per-home live deed verdict, for the Audit Report) · `dnc` (`{is_dnc, reason}`) · `tags[]` · `note_markdown`. `resolution_status` gates the transport: `resolved` → written to FUB; `needs_review` → held; `complete` → transported. ⚠️ Write to **`resolver_block`**, never the pre-existing `dossier` TEXT column (Myli's case-file). Migration `0243` declares the columns.

### 🔖 FUB TAG TAXONOMY (Zach LOCKED 2026-06-23)

| Tag | On which leads |
|---|---|
| `HouseLoop Resolved` | **every** resolved client |
| `SRG Closed - Ana Only` / `Ana Handoff` / `Team` / `Agent DNC` | the `bucket` |
| `SRG Closed - Lost Listing` | a home re-sold by another agent (seller-engagement lead) |

Tags are **ATOMIC** — never bake combinations into one tag; FUB smart lists AND them (e.g. `Ana Only` + `Lost Listing` = "Ana's seller leads").

## The one rule that governs everything: resolve first, enrich second

A past-client home is never enriched, slugged, or surfaced until it has been resolved to a client and signed off. Per-home lifecycle: `(1) RESOLVE → (2) SIGN-OFF → (3) BACKFILL (Ninja → master) → (4) CANONICAL SLUG → (5) SELLER REPORT`.
