The `/leads` table is the operator's command center. Every column is a real,
DB-backed metric — **nothing is hardcoded** — and every metric comes from a
**single aggregation** in `lib/leads-query.ts`, not per-cell fetches. Columns are
toggleable via the **Columns** panel; the system columns are on by default.

## The one source of truth

All per-lead metrics are computed in one `SELECT` (`BASE_SELECT` in
`lib/leads-query.ts`) over the CRM tables plus the first-party
[pixel](/guides/pixel-tracking) events:

Column order (left → right): **Phone Status · Email Status · Registered ·
Requests · Visits · Listings · Alerts · Agent Action · Tasks**.

| Column | Source |
|---|---|
| **Lead Information** (name · phone · location) | `hl_leads` + `hl_lead_addresses` (primary) |
| **Phone Status** (calls / texts) | `hl_calls` count + texts from the comms SSOT `hl_messages` (`channel IN ('sms','imessage')`) — the old two-source `hl_bb_events` + `hl_inbound_messages` merge was replaced by the ONE-table law (verified in `lib/leads-query.ts`) |
| **Email Status** (emails / last) | `hl_messages` channel = `email` |
| **Registered** (when + source) | `hl_leads.created_at` + `source` |
| **Requests** (tour/showing requests) | `hl_pixel_event` request-type events (`tour_request` / `contact_request`), split by type — count + last request time (verified in `lib/leads-query.ts`; `hl_notification` powers only the UNREAD badges below) |
| **Visits** (count + last) | `hl_pixel_event` `page_view`, de-duped by distinct day |
| **Listings** (views / saved / $) | `hl_pixel_event` `property_view` / net `save_home` / `property_payload.price` |
| **Alerts** (e-Alerts) | `hl_pixel_event` `saved_search` |
| **Agent Action** (what · when · who) | most-recent of outbound message / note / task / appointment, via `LEFT JOIN LATERAL` |
| **Tasks / Activities** | `hl_tasks` (open count / next due) |

When a metric has no data source yet (market reports, alert unsubscribes), the
cell renders an **honest 0** — it is never faked.

## Unread alerts (mission control)

The **red icon-chip badge means UNREAD for the current user** — not the total
(the total lives in the label underneath). Unread is computed per-user in the
same aggregation against `hl_notification` (`read_at IS NULL`, scoped to the
session user via the `userP` bind):

| Badge | Event keys | Status |
|---|---|---|
| **Texts** unread | `new_message` | Live |
| **Requests** unread | `tour_request` / `saved_search` / `contact_form` | Live |
| **Calls** unread | — | Pending Gate 5 (Twilio voice ingest) |
| **Emails** unread | — | Pending Gate 5 (inbound-email ingest) |

**Clear-on-open:** opening a lead `POST`s `/api/notifications/lead-badges`
with `category: 'all'`, marking that lead's notifications read so its badges
clear. Calls/Emails light up once inbound call-logging + email ingestion land.

## Clickable metric icons

Every metric chip is a link to that lead's detail with the matching iPad panel
pre-opened: `/leads/[id]?tab=<Tab>` (Calls→Calls, Texts→Texts, Email→Emails,
Requests→Requests, Listings/Visits→Homes, Registered/Agent Action→All). The
page validates `?tab` server-side against the `ActivityTab` union.

## Agent Action — what & who

Agent Action reads the most recent real action against the lead through a
`LATERAL` union over outbound messages, notes, tasks, and appointments. It shows
the verb + relative time + **who** did it. The actor is derived honestly from
`hl_messages.actor_role`, `hl_notes.source`, and `hl_tasks/appointments.created_by`
— so a system/automation action reads "HouseLoop System" and a human action reads
the agent's name. Never fabricated.

## Layout system

- **Icon chips** — every column uses one shared round icon chip sized by the
  `--hl-stat-chip-size` token (the "cannon"), so all icons match and line up.
- **Proportional growth** — `ColumnDef.grow` (fr weight) lets columns expand at
  different rates as the table widens (Agent Action grows far faster than Visits)
  instead of every column growing equally.
- **Full width** — the page opts into `<PageShell wide>` (`.hl-page-inner--wide`)
  to lift the global 1400px content cap so the table fills the browser.
- **Truncation** — every metric value clips with an ellipsis when its column is
  too narrow; no cell can ever bleed into its neighbor.

## Relative time

All relative timestamps use minutes / hours / days only — never "about", never
months or years (a 400-day-old lead reads "400 days ago"). Times are Central.
