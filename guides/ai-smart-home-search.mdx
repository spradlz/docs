**AI Smart Home Search** lets a visitor type what they want in plain language
("homes with a white kitchen and an in-ground pool in Overland Park") instead of
clicking filters. Entry points: the **"AI Search"** item in the top nav, the
gradient **"AI Search"** button beside the hero search bar, and the **"Search
smarter with AI"** row in the search dropdown — all open one chat modal.

## How it works (hybrid: our chat UX, Repliers' search engine)

1. The modal sends the running conversation to `POST /api/search/nlp`.
2. **gpt-4o-mini** (server-side — the key never reaches the browser) runs the
   *conversation*: a friendly reply + deciding when there's enough to search (`ready`).
3. When ready, the search is powered by **Repliers NLP** (`POST https://api.repliers.io/nlp`):
   - **Structured filters** — "pool" → `swimmingPool=not:null`, "3 bed under 600k" →
     `minBeds=3&maxPrice=600000`, city, etc.
   - **AI image search** — descriptive/visual features ("white kitchen") are matched
     against the listing **PHOTOS** (semantic image embedding), not the text.
   - We run the returned query, map each `mlsNumber` to our `hl_heartlandmls_<n>` hlid,
     and hydrate from our cache (**Core-Heartland 8-county clamp applied for free** —
     non-Heartland listings drop out).
4. **Matched-photo lead** — on image searches Repliers re-orders each listing's photos
   so the matched shot (the pool, the white kitchen) is first; we lead the card with it.
5. The reply comes back with the match count and a **View N homes** button that
   deep-links into the gallery (matches pinned + badged "AI Match" via `aimatch=`).

<Note>
**Repliers NLP is LIVE and primary** (enabled in the Repliers developer portal under
the API key's "Enable NLP" toggle + an OpenAI key on the account). It bills that
OpenAI key per search. If Repliers returns nothing, the route **falls back** to our
own matcher — a **gpt-4o** re-rank (reads listing descriptions; rejects "white quartz
countertops" ≠ white kitchen, "pool table"/"community pool" ≠ private pool) plus a
graceful partial-match fallback ("couldn't find both X and Y — here are N with a Y").
</Note>

## Private vs HOA amenities

Repliers separates the home's OWN features from the neighborhood's:
`details.swimmingPool` ("In Ground") = a **private** pool; `condominium.amenities`
containing `"Pool"` = an **HOA/community** pool. A home can have both. (Pool matching
trusts the structured field — the pool image-embedding is too flat to filter on.)

## Endpoint

`POST https://houseloop.ai/api/search/nlp`

**Request**

```json
{
  "messages": [{ "role": "user", "content": "homes with a pool in Overland Park" }],
  "conversationId": "optional — reuse to continue a conversation"
}
```

**Response (ready to search)**

```json
{
  "ok": true,
  "ready": true,
  "reply": "I found 24 homes matching homes with a pool in Overland Park — plus 6 more nearby you might like.",
  "count": 30,
  "matchCount": 24,
  "aiMatchHlids": ["hl_heartlandmls_2619637", "..."],
  "url": "/listings?city=Overland Park&aimatch=hl_heartlandmls_2619637,...",
  "listings": [{ "hlid": "...", "photo": "https://cdn.repliers.io/<matched photo>", "...": "..." }],
  "conversationId": "ff71a4f0-…"
}
```

`GET /api/search/nlp/history` returns the visitor's recent searches (History dropdown).

## Conversation storage

Every turn persists (best-effort) to `hl_ai_search_conversation` (`sessionId` = the
`hl_session` cookie; `userId` set/backfilled on sign-in; `messages`, `lastFilters`,
`lastUrl`, `lastCount`). Anonymous threads link to the account on the next signed-in turn.
