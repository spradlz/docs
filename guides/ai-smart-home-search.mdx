**AI Smart Home Search** lets a visitor type what they want in plain language
("homes with a white kitchen and an in-ground pool in Overland Park", "what's the
most expensive home in Overland Park") instead of clicking filters. Entry points:
the **"AI Search"** item in the top nav, the gradient **"AI Search"** button beside
the hero search bar, and the **"Search smarter with AI"** row in the search dropdown
— all open one chat modal.

## How it works (our chat UX, a layered search engine)

1. The modal sends the running conversation to `POST /api/search/nlp`.
2. **Claude Haiku 4.5** (server-side — the key never reaches the browser) runs the
   *conversation*: a friendly reply + structured filter extraction + deciding when
   there's enough to search (`ready`). Descriptive phrases are resolved to canonical
   registry features (grounded to the buyer's words); a deterministic layer applies
   home-type / single-story defaults.
3. When `ready`, the search resolves through a **fixed layer order** — the first
   layer that can handle the query wins:

<Steps>
  <Step title="Superlative answer">
    If the query carries a sort ("most expensive / cheapest / biggest / most beds"),
    we **answer the question** instead of dumping a list. See [Superlative answers](#superlative-answers).
  </Step>
  <Step title="Vision-tag match (PRIMARY for tagged metros)">
    Descriptive features match our **pre-computed AI vision catalog** of each
    listing's photos — instant SQL, photo-verified, exact. See [Vision Catalog](#vision-catalog).
    If the metro has no vision tags yet, this layer is skipped.
  </Step>
  <Step title="Repliers NLP (untagged fallback)">
    `POST https://api.repliers.io/nlp` extracts true structured filters + runs
    Repliers' own AI image search. Used where we haven't vision-tagged yet.
  </Step>
  <Step title="gpt-4o re-rank (last backstop)">
    Reads listing descriptions and rejects false positives ("white quartz
    countertops" ≠ white kitchen, "pool table" / "community pool" ≠ private pool).
  </Step>
</Steps>

4. The reply comes back with the match count, a `listings` array (preview cards),
   and a **View N homes** URL that deep-links into the gallery (matches pinned +
   badged "AI Match" via `aimatch=`).

<a id="vision-catalog" />
## Vision Catalog (the primary matcher)

At ingest, **gpt-4o-mini** (vision, low detail, up to 16 photos) reads each listing's
photos ONCE against a controlled matrix of **50+ buyer-searchable features** and caches
the result in the `Listing.visionTags` JSON column. Descriptive search then becomes
**instant SQL** — no per-query LLM, no Repliers round-trip.

<a id="superlative-answers" />
## Superlative answers

When the query carries a sort, the engine **answers the question** (Zach's rule: "if I
ask a question it should answer it — not 24 matching + 455 more nearby"): singular →
leads with ONE home + a short factual narration; plural → the sorted list. `url = /listings?sort=<sortKey>`.

## Private vs HOA amenities (vision-verified)

A home's OWN pool is separated from the neighborhood's. **Pool is vision-verified:**
`visionTags.pool_type = "private"` only when the model SEES a pool in the home's own yard;
community/HOA pools are tagged `"community"` and never match a private-pool query. The
`private_pool` feature also triggers the single-family privacy rule (community-pool condos
never leak into a "homes with a pool" search).

## Endpoints

### `POST /api/search/nlp`
Runs one conversational turn (the AI modal + Myli). Returns a `reply` and, when `ready`, `count` / `matchCount` /
`aiMatchHlids` / `url` / `listings[]` / `filters` / `conversationId`.

### `POST /api/search/classify`
Routes the traditional search bar: deterministic filter extraction (Haiku) → `{ intent, params }`; the bar applies
the same defaults + shows removable chips. `params` carries `city/style/story/minBeds/feature/detail.*` etc.

### `GET /api/detail-search-facets`
The human Details modal's checkbox values + counts. **F1 (in progress):** counts move from Repliers aggregates onto
our DB so they match the result counts.
