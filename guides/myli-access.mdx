## Overview

Every CRM action Myli takes goes through this REST API. No direct DB queries.
No UI automation. Pure API calls â€” same as any integration partner would use.

## What Myli Can Do

| Action | Endpoint |
|---|---|
| Find a lead by name/email/phone | `GET /api/leads/search?q=Jane` |
| Get all leads in a pipeline stage | `GET /api/leads?criteria=[...]` |
| Create a new lead | `POST /api/leads` |
| Move a lead to a new stage | `PATCH /api/leads/{id}` |
| Create a task for a lead | `POST /api/tasks` |
| Complete a task | `PATCH /api/tasks/{id}` with `{ status: "done" }` |
| Create a calendar event | `POST /api/calendar/events` |
| Send an email template | `POST /api/email/send-template` |
| Get all pipeline stages | `GET /api/stages` |
| Get all lookup data at once | `GET /api/options` |

## Myli's Auth Pattern

Myli uses a **service session** â€” a long-lived cookie set at startup.
In production, set the `HL_MYLI_TOKEN` env var and the API will accept it
as a `Bearer` token in lieu of a user cookie.

```bash
# Example: Myli moves a lead to "Appointment Set"
curl -X PATCH https://crm.houseloop.ai/api/leads/f40c91f1-... \
  -H "Authorization: Bearer $HL_MYLI_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"stage_id": "7c8cba84-..."}'
```

## Common Patterns

### Find a lead and move their stage

```bash
# 1. Search for the lead
LEAD=$(curl "https://crm.houseloop.ai/api/leads/search?q=Jane+Smith" | jq '.leads[0].id')

# 2. Get pipeline stages to find the target stage ID
STAGES=$(curl "https://crm.houseloop.ai/api/stages")

# 3. Move the lead
curl -X PATCH "https://crm.houseloop.ai/api/leads/$LEAD" \
  -H "Content-Type: application/json" \
  -d "{\"stage_id\": \"$TARGET_STAGE_ID\"}"
```

### Create a follow-up task

```bash
curl -X POST https://crm.houseloop.ai/api/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Call back â€” expressed interest in 3br listings",
    "lead_id": "f40c91f1-...",
    "due_at": "2026-05-20T10:00:00Z"
  }'
```

### Load all dropdown data in one call

```bash
# Returns stages, sources, tags, users â€” everything needed for filter panels
curl https://crm.houseloop.ai/api/options
```

## Filter Criteria Format

The `GET /api/leads` endpoint accepts a `criteria` query param (JSON-encoded array of conditions):

```json
[
  { "fieldKey": "stage", "operator": "include", "itemIds": ["fc1972f1-..."] },
  { "fieldKey": "assigned_to", "operator": "include", "itemIds": ["user-uuid"] }
]
```

**Available fieldKeys:** `name`, `email`, `phone`, `stage`, `source`, `assigned_to`, `tag`, `lead_score`, `last_communication`, `created_at`, `open_tasks`, `overdue_tasks`

**Available operators:** `contains`, `not_contains`, `is_empty`, `is_not_empty`, `include`, `exclude`, `gt`, `lt`, `gte`, `lte`, `within_last`, `older_than`

## Home Search â€” the website search Myli runs for buyers

Everything above is the **CRM REST API** (leads, tasks, stages). To search **listings** for a buyer â€” "single-family homes in Overland Park with a white kitchen and a pool" â€” Myli uses a **separate** path: the **`houseloop-search` MCP server** (`tools/mcp/houseloop-search-server.mjs`), which calls the live **houseloop.ai** website API. This is the **same engine** end-buyers use on the site; full internals are in [AI Home Search](/guides/ai-smart-home-search) (Website Integration).

| MCP tool | Website endpoint | What it does |
|---|---|---|
| **`houseloop_nlp_search`** | `POST /api/search/nlp` | **The AI Home Search.** Natural language â†’ matched listings + tiered counts (For Sale / Coming Soon / Nearby) + a deep link. The way Myli "describes a home" for a buyer. |
| `houseloop_classify_search` | `POST /api/search/classify` | The search-bar parser â€” free text â†’ removable filter pills. |
| `houseloop_search_listings` | `GET /api/listings/search` | Structured search by the shared query envelope â†’ Listing objects (price, address, beds/baths/sqft, photos, status) + engagement. |
| `houseloop_count` | `GET /api/listings/count` | Live count for a query (the number behind any pill/filter). |
| `houseloop_map_search` | `GET /api/listings/map` | Map pins/clusters for a viewport. |
| `houseloop_get_listing` | `GET /api/listings/{hlid}` | One home's full detail. |
| `houseloop_autocomplete` | `GET /api/listings/autocomplete` | Location/feature typeahead. |
| `houseloop_place_boundary` | place-boundary | A city/neighborhood polygon for map scoping. |

Base URL: `HOUSELOOP_API_BASE` (default `https://houseloop.ai/api`). Owner: Lee.

<Note>
**Same library, every surface.** `houseloop_nlp_search` runs the exact engine the website's AI modal + search bar run â€” it reads the canonical `FEATURE_REGISTRY` and the live detail-facet values, so a filter Myli can ask for is a filter a buyer can click, and vice versa. When the [AI Home Search](/guides/ai-smart-home-search) roadmap lands its matching layer, Myli inherits the wider filter reach automatically â€” no change here. See that page for what's shipped vs. in flight.
</Note>
