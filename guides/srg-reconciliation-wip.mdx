## 🛑 THE LAW (locked 2026-06-27 10:57 AM EDT — read this first, it overrides everything below)

**`hl_srg_past_client_homes` (the CHILD) is THE single source of truth. EVERY resolved fact is WRITTEN there as a
column. The Audit Report, the FUB note, and the `/home` page all RENDER from the child. Nothing is "resolved" until
it is written into the child table.** The parent `hl_srg_past_clients` holds the person + `bucket` + a `resolver_block`
that is a **DERIVED render of the child** — never the place truth lives.

<Warning>
**The drift this corrects (the 2-day mistake).** A parallel "cheap deterministic resolver" was writing a rendered JSON
blob (`resolver_block`) onto the PARENT while the CHILD master sat EMPTY (`determined_agent` = NULL book-wide). The
truth lived in a buggy render. Symptoms: "SRG"/"Past Agent Referrals" placeholder agents shown as real (the dotloop
CONTRACT step was never called), false lost listings (homes WE sold flagged "lost" — Bazalgette), conflated buyer/listing
agents (Metman), duplicate house rows. **Fix: write the resolved truth INTO the child; render everything FROM it.**
</Warning>

## The data model (two tables, one SSOT)

| Layer | Table | Grain | Holds |
|---|---|---|---|
| PARENT — person | `hl_srg_past_clients` | 1 / person (~1,834) | name · contact · `bucket` · `website_hlid` · `resolver_block` (**derived render**) |
| **CHILD — house = THE SSOT** | `hl_srg_past_client_homes` | **1 / house (~2,256)** | the ONE reconciled row — all facts flat + `*_source` provenance. **`determined_agent` + `determined_agent_role` + `determined_agent_source`, disposition, `sold_not_by_us` MUST be populated here.** |

**5 sources feed the child; the app never reads them directly:** `hl_srg_mls_deals` (MLS, retired as SSOT) ·
`sisu_transactions` (SISU agent/dates/status) · `client_reconciliation_notes` (Zach's recon) ·
`hl_srg_dotloop_contracts` (signed-contract licensee) · `fub.*` (the person: notes/comms/DNC signal → parent).

## The agent waterfall (LOCKED — "no agent = not resolved"; never SRG/placeholder/MLS-listing)

1. **SISU `agent`** (client first+last + amount). Zach → Ana. Not `closings.agent` (MLS-polluted).
2. **No real SISU agent → dotloop CONTRACT** → the signed **Licensee Assisting Seller/Buyer** = final truth.
   `dotloop-contract-agent.mjs → contractAgent(loopId, profileId)`. The participant list lies on departed-agent loops;
   the real agent survives only in the signed PDF. **`Past Agent Referrals` and `SRG` are PLACEHOLDERS that TRIGGER this
   step — never the displayed agent.** Proven: Earl → Jessica Rich, Burton → Jeremy David.
3. **No dotloop → FUB** `Team Exit - <name>` + notes/texts/calls. 4. **Still nothing → not resolved → investigation.**
   Keep the departed agent — the relationship is the asset. **Ben Froeschl is never the agent.**

Precedence on the child `determined_agent`: **dotloop CONTRACT > SISU > MLS list_agent > recon.**

## Disposition + lost listings

- **Disposition:** live county deed PRIMARY (owner_match = owns / owner_differs = moved) > Repliers (sold date/price) >
  Ninja (agents). Deed-lag beats a lagging deed; Arboridge guard. Mover-finder (cross-county by owner name,
  deed-verified) finds the new home; else "moved — unknown, ask them."
- **A LOST LISTING is bigger than the new address** — it's a seller lead; we don't need where they moved, just a text ask.
- **Lost-listing rule (Bazalgette):** ONLY a home we BOUGHT for them that THEY later sold through a NON-SRG agent.
  **If WE sold the address, it is NEVER lost** (a same-date buyer agent is the other side of OUR sale; a years-later
  resale is a stranger's deal). Match by house# + street token, not raw string.
