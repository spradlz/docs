## 🏛️ THE 2-LAYER ARCHITECTURE (Zach LOCKED 2026-06-27 PM) — 6 source tables → resolver → 2 read tables

**Not "2 tables instead of 6" — "6 SOURCE tables FEED 2 CANONICAL read tables."** Both stay.

- **SOURCE (system of record, raw, never bulk-copied):** `sisu_transactions` · `hl_srg_mls_deals` ·
  `hl_srg_dotloop_contracts` · `Listing` (Repliers) · `fub.*` (Camp David). The resolver READS these.
- **READ MODEL (the 2 canonical tables):** PARENT (person + dossier) · CHILD (house facts + dossier). The card +
  AI clone read **ONLY** these. Resolve ONCE, materialize on the child, read forever — never re-JOIN 6 tables at read time.

**Two separate resolver processes:**
1. **PERSON → PARENT (LLM):** query FUB for all of a person's FUB IDs (one human, 2–3 dupe records) → LLM connects them
   → ONE dossier → PARENT.
2. **HOUSE → CHILD (deterministic attach):** dotloop contract + MLS closing sheet + SISU → CHILD. No LLM. Agent falls
   out of the contract/SISU; disposition = stored `sale_history` (sold-after-our-sale → moved → not-us → lost listing),
   ≤1 county lookup, NO 50-deed swarm.

<Warning>
**THE NO-BIAS RULE.** Most child rows carry a STALE `determined_agent` from prior runs **while the source columns are
EMPTY** (`sisu_agent`/`dl_loop_id`/`mls_number` = NULL) — a guessed agent on top of no evidence (the "50 times"
pattern). **The resolver runs BLIND: never shown the prior agent/disposition/dossier; it tees up ONLY raw source
evidence and resolves independently.** Showing the model the prior answer is not a test.
</Warning>

## The agent waterfall (LOCKED — "no agent = not resolved"; never SRG/placeholder/MLS-listing)

1. **SISU `agent`** (client first+last + amount). Zach → Ana. Not `closings.agent` (MLS-polluted).
2. **No real SISU agent → dotloop CONTRACT** → the signed **Licensee Assisting Seller/Buyer** = final truth.
   `dotloop-contract-agent.mjs → contractAgent(loopId, profileId)`. The participant list lies on departed-agent loops;
   the real agent survives only in the signed PDF. **`Past Agent Referrals` and `SRG` are PLACEHOLDERS that TRIGGER this
   step — never the displayed agent.** Proven: Earl → Jessica Rich, Burton → Jeremy David.
3. **No dotloop → FUB** `Team Exit - <name>` + notes/texts/calls. 4. **Still nothing → not resolved → investigation.**
   Keep the departed agent — the relationship is the asset. **Ben Froeschl is never the agent.**

Precedence on the child `determined_agent`: **dotloop CONTRACT > SISU > MLS list_agent > recon.**

## Disposition + lost listings

- **Disposition:** live county deed PRIMARY (owner_match = owns / owner_differs = moved) > Repliers (sold date/price) >
  Ninja (agents). Deed-lag beats a lagging deed; Arboridge guard. Mover-finder (cross-county by owner name,
  deed-verified) finds the new home; else "moved — unknown, ask them."
- **A LOST LISTING is bigger than the new address** — it's a seller lead; we don't need where they moved, just a text ask.
- **Lost-listing rule (Bazalgette):** ONLY a home we BOUGHT for them that THEY later sold through a NON-SRG agent.
  **If WE sold the address, it is NEVER lost** (a same-date buyer agent is the other side of OUR sale; a years-later
  resale is a stranger's deal). Match by house# + street token, not raw string.
