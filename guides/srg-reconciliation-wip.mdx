## The model (locked 2026-06-27)

**Make SISU complete.** SISU is canon for *which deals exist*; each source fills one blank:

- **SISU** — the deal spine (client, $, close date).
- **MLS** (`hl_srg_mls_deals`, 100% join by `mls_number`) — the **true address / city / zip** (SISU's address is unreliable; always cross-reference the MLS closing).
- **FUB** — notes / investigative (current home, referral, moved out of state).
- **dotloop** — the signed contract → the real agent.
- **County deed** — disposition truth: who's on title now. **The router.**
- **Google Places** — geo normalizer when MLS lacks a zip.

## The confidence-tier cost play

| Tier | Definition | Engine | Cost |
|---|---|---|---|
| 🟢 **GREEN** | 1–2 deals · real named agent · triangulated MLS home | `deterministic-resolve.mjs` | **$0** |
| 🟡 **YELLOW** | 1–2 deals · agent is Ana-only | assembly + one `dotloop_contract` probe | cheap |
| 🔴 **RED** | 3+ deals, ambiguous agents, or missing triangulation | `s6-investigator.mjs` (Sonnet) | ~$0.30–0.50 |

**Within GREEN, the county deed is the router** — and the split is FACT, not a guess: `owner_match` → lock free (still-there); `owner_differs` → moved → mover-finder; `ambiguous` (trust/LLC) → likely theirs, med; `not_found`/`out_of_coverage` → coverage pass.

## The numbers (clean dry-run, 766 BUY-LAST GREEN)

- **522 still-there + 107 moved = 82% known to hard fact**, no LLM. 86 uncheckable (52 fringe-county, 34 harder edge cases). A mover is *not* "unknown" — we know for a fact they left; only the new address is open.

## The mover-finder — free, cross-county, deed-verified

When a client's old-home deed shows a different owner, `mover-finder.mjs` searches **every covered county by owner name** and returns their new address straight off the deed (find + verify in one — the parcel is owned by a first+last-name match). No paid skip-trace.

- **Counties live:** Johnson (AIMS), Wyandotte (Tyler quick search), Clay (ArcGIS owner LIKE), **Cass (WFS deedhold)**.
- **Ranked match** (exact > nickname > prefix) — excludes "TRAN, TU" for a client named "Tuyet".
- **Multiples → drill in:** returns the candidate **addresses** (not just names) for a confident pick; a passed spouse/co-owner name auto-resolves.
- **Confidence:** high (one match, address in hand) · ambiguous (candidates + addresses) · unknown (out of covered metro → outreach asks the client).
- **Proven:** Anthony Kuchta → 19008 W 189th St, Spring Hill (deed: KUCHTA, LYNDA & ANTHONY).

Every mover's dossier carries structured move data — `moved` · `last_known_address` · `new_address` (deed-verified or "unknown — find out") · `confidence` — so the **LLM writes the re-engagement message from the data** (we provide the data, not the message).

**Gaps (deferred):** Jackson owner-name search (`owner_info LIKE` 400s on the ParcelViewer layer — needs a different endpoint); Platte/Lafayette/Douglas fringe-county deed adapters.

## The DNC roster — "ever an SG agent/employee"

**A past client who is a previous SG agent or employee is do-not-contact.** DNC is derived from *identity*, never from a tag:

- **Roster (130 names)** = SISU agent field (closers) ∪ FUB tags `Agent:` / `Team Exit -` / `Imported Agent:`. FUB is the agent/employee *history* (Team Exit = departed agents, ISA = employees).
- **DNC = an exact full-name match** to that roster → **8 real** (Mark Tallent, April Atkinson, Zak Zimmerman, Megan Young…), marked `bucket='agent-dnc'`.
- A **surname collision alone is NOT DNC** — an earlier bug benched ~100 real clients that way; fixed, they're back in GREEN.
- ⛔ We **never** use any literal "DNC" / "Do Not Contact" FUB tag — those were rogue-applied and are ignored entirely.
- dotloop's loop index is metadata-only (no stored participant roster); participants are pullable via the API if ever needed.

## The agent resolution waterfall (LOCKED — accuracy is the only benchmark)

**Every closing resolves to a REAL human agent. Never "SRG", never an MLS-listing agent, never a placeholder. "No agent = not resolved."** The agent comes from a strict waterfall:

1. **SISU `agent`** — precise match on the client's actual SISU deal (client first + last + amount). The source of truth.
2. **No SISU agent → dotloop contract** for the property → the signed **Licensee** (authoritative; settles ambiguity).
3. **No dotloop → FUB** — the `Team Exit - <name>` tag plus the notes, texts, calls, and full scraper record, to find who actually worked the client (an agent who left the team and closed them outside it).
4. **Still nothing → not resolved → investigation tier.**

**The Ben Froeschl rule:** `Agent: Ben Froeschl` is *never* the agent of record — he was the sales manager and never closed a deal; default attribution was dumped on him. Exclude him everywhere. A client tagged both `Agent: Ben Froeschl` and `Team Exit - Alexis Miller` (e.g. Jerry Hall) → the agent is **Alexis Miller**.

**Keep the departed agent — the relationship is the asset.** A past agent who left SRG is still the agent of record; the reactivation message references them ("you worked with Alexis on my team, she helped you buy 123 Main"). The client doesn't care they left — they feel a realtor who knows their history.

**Why this is locked:** the agent was pulled from `closings.agent` (MLS-polluted: `spradlinggp` vs `anasprad`) → 42 cards showed the wrong agent, Ana-Only was miscounted (260 vs the true **461** from the SISU `bucket` column), and a non-SISU `fub-address-sweep` record (Jerry Hall, MLS-listing agent "John Tramble") was wrongly resolved. Agent/bucket come from SISU; only a real human name is acceptable.

## Doctrine locked (principles, not rules)

1. **SISU's address is unreliable — cross-reference the MLS closing** for every transaction's address/geo.
2. **Never assert a current home the deed contradicts** (the Nations error). Sold/moved ≠ knowing where they live now.
3. **No SISU = not a deal** — never invent a transaction or lost listing from a Repliers/dotloop-only signal.
4. **A new address is only asserted when a county deed `owner_match` confirms it** — same bar as the still-there tier.
5. **Card format:** `CONTACTS → TRANSACTIONS WITH US → CURRENT (deed-confirmed / moved) → DOSSIER`.

## Operational lesson — assessors rate-limit

County assessors throttle under rapid sequential load. Run the resolver **solo and paced** (`--pace <ms>`, default 600ms) — never alongside concurrent county diagnostics. Throttling can only flip confirm→uncheckable, **never fake an `owner_match`**, so locked rows are always valid. Recover throttled rows with `--write --unresolved --pace 800` after a cooldown.
