## 🆕 THE RESOLVER — `triangulate-resolver.mjs` (Stage 1 built + tested 2026-07-01)

**The harvest is done (4 tables full); the resolver that combines them is now being built.** It replaces the
token-burning s6 agentic loop. Full spec: internal PRD `GT_HOUSELOOP_TRIANGULATE_RESOLVER_PRD.md`.

**Why s6 had to go:** 8-turn agentic loop on **Sonnet 4.6** that did its lookups (county/Ninja/Matrix-SSH/dotloop)
**live inside the loop** and re-fed the growing transcript every turn (a 13.8M-input-tokens/day spike) → ~$100/day, and
it **guessed** whenever a live fetch timed out. The harvest moved every live lookup OUT of the loop into a pre-built table.

**The design — deterministic-first, one-shot Haiku:**
- **Stage 1 — assembly ($0, no LLM):** per client, join brain(`hl_fub_person_facts` by fub_id) + SISU(`sisu_transactions`
  by name) + dotloop agent(`hl_srg_dotloop_contracts` by address) + MLS disposition(`hl_mls_property_history`
  rooftop — a later Sold = they moved) → tier **GREEN** (sources agree) / **YELLOW** (conflict or choice) / **RED** (gap).
- **Stage 2 — Haiku, ONE shot, YELLOW only (gated):** reconcile source conflicts per the reconciliation laws +
  synthesize the **RELATIONSHIP dossier** = the Myli 12-domain facts (spouse/kids/life-events/comms/motivation) **woven
  WITH** the transaction history. RED → `needs_review` with the exact missing source named. Never invents.

**Stage-1 test on the 12-card set (2026-07-01, first run):**
- **GREEN pc6 Purtee** — fully resolved deterministically at $0, incl. the disposition (6424 Evanston later sold
  2026-03-20 $253k by Homeric Group = they moved). Proves the $0 path.
- **YELLOW pc11/21/38/57** — correctly flagged for the Haiku reconcile (agent name-variants like Walter Hawkins /
  Walter V Hawkins Jr., an off-brain 2024 deal, duplicate address rows, SISU/brain tx-count mismatch).
- **RED pc5/22/12/17/24** — flagged "no FUB brain", but this is a **linkage gap, not missing data** — the SISU closings
  + dotloop agents are present; `past_client.fub_id` didn't hit a brain for older 2017-2020 clients. Next fix = brain
  fallback by name/phone.

**Cost:** est. **~$15-30 one-time** (Haiku, YELLOW-only) vs s6's ~$100/day. **Proof gate:** 12/12 on
`gold-convergence.mjs` before any scale. Gated by Zach: Stage 2 build → scale to 1,834 → FUB publish (separate gate).

## 🆕 PRD — HARVEST-THEN-TRIANGULATE (Zach LOCKED 2026-06-29 PM, the method that finally holds)

<Warning>
**Why every prior attempt failed:** they resolved **reactively** — went looking for a fact *during* matching, so the
facts never all existed at once and the matcher **guessed** (Billy Fischer stamped "ghost card / no SISU" when SISU
held the closing under Walter Hawkins; Collins's 16451 Blair assumed "owns" when one MLS query showed it sold 2022;
Anders's current home found via GIS but never written to the rendered field). A day of grinding burned $100+ of LLM.
</Warning>

**THE METHOD: harvest exhaustively into provenance-stamped tables FIRST → then triangulate complete data.** When every
fact already lives in a table before resolution starts, the matcher cannot guess — there is nothing missing to guess
about. **Cost = cheap Haiku only** (reading FUB text/calls/tags/stages to extract structured facts + find SISU deals in
FUB). Everything else is deterministic: DB joins, the dotloop API, the MLS scrape, county/Ninja APIs.

### The four sources → four canonical staging tables (every row stamped to its source key)
| # | Source | Access | Table | State (2026-06-29) |
|---|---|---|---|---|
| spine | **SISU** | DB | `sisu_transactions` | ✅ have it (the paid-truth: price · date · client · agent) |
| 1 | **Dotloop** (agent spine) | **Public API v2 OAuth2** (live, `document:*`) | `hl_srg_dotloop_loops` + `hl_srg_dotloop_contracts` | loops **17,455 ✅** · contracts only **202** → finish |
| 2 | **MLS** | Heartland Matrix scrape (Camp David) | **`hl_mls_property_history`** (NEW) | build + harvest full rooftop history |
| 3 | **FUB** | scraper mirror DB + **Haiku** extraction | **`hl_fub_person_facts`** (NEW) | build + harvest |

### Why dotloop is the AGENT SPINE — the departed-agent unlock
When a team agent leaves, dotloop **deletes them from `/participant`** — the loop then shows only the agent-of-record
(Ana) + clients, NOT the agent who worked the deal. That agent survives ONLY in the signed contract PDF's **"Licensee
Assisting Seller/Buyer"** field. We hold the `document:*` scope → `dotloop.downloadDocument()` → `pdftotext` → the real
human agent, **deterministically, no LLM**. This is the fix for the **535-of-2,168** missing-agent gap and the
Walter-Hawkins / Mikey-Fletcher / Team-Exit class. dotloop is "the final say on who actually transacted what with us."

### Harvest specs
- **Dotloop → `hl_srg_dotloop_contracts`** (1 row / closing loop, all via API): `/loop/:id/detail` → address · MLS# ·
  price · close date; `/participant` → client name+email+role+side; **download contract PDF → licensee/designated
  agent**. Provenance = loop id + document id. Resumable (skip pulled loops). Reuse the dotloop suite.
- **MLS → `hl_mls_property_history`** (1 row / MLS record / rooftop): dedup every address across SISU + child homes +
  dotloop detail → unique rooftops; `mls-address-lookup.mjs` → **every** record (Sold/Active/Expired/Pending) with
  mls# · status · sale price · close date · list agent · office · beds/baths/sqft/year. Provenance = mls#. **Catches
  the post-our-sale resale** (the lost-listing signal we kept missing). Cache per rooftop; the only slow scrape → runs
  all night, resumable.
- **FUB → `hl_fub_person_facts`** (1 row / FUB person): a schema-constrained **Haiku** extraction over notes/texts/
  calls/tags/stages → fields ONLY (no prose): `spouse_names[]` · `mentioned_addresses[]` · `team_exit_agent` ·
  `closing_refs[]` · `dnc` · `is_agent` · emails/phones — each carrying its **source snippet**. Also links SISU deals →
  FUB person(s) by name+phone+email. Extraction, not narration.

### Phase 2 — Triangulate (deterministic-first; Haiku only for the ambiguous tail)
Join the four tables per past client → write `hl_srg_past_client_homes` (child SSOT) + `hl_srg_past_clients.dossier_json`:
- **Agent** = dotloop contract licensee → SISU → MLS our-side → FUB Team-Exit (Ben Froeschl never; Zach→Ana).
- **Disposition** = MLS rooftop history (later Sold by a non-SRG agent after our buy = LOST listing, dated/priced/agent;
  EXIT rule — a home WE sold is never lost) + county deed for current ownership.
- **Current home** = deed owner_match, else (sold-everything) AIMS/county surname search → seat via Ninja (HLID+value),
  **materialized into `dossier_json.current_properties`** (the field the card renders).
- 80% resolve in pure code; only true ambiguity (collisions, multiple candidate homes, conflicting sources) escalates
  to a cheap Haiku that can only MATCH facts already on the table. **No-add/no-delete/match-only.**

### Cost + guardrails
Cost = cheap Haiku only (FUB extraction + ambiguous triangulation); zero Opus/Sonnet in the path. No auto-sign-off, no
FUB publish (Zach gate). Provenance on every row; a fact with no trace = `needs_review`, never asserted. The harvest
tables are canonical staging — reusable for the AI clone + website, not scratch.

## 🛑 THE LAW (locked 2026-06-27 — read this first, it overrides everything below)

**`hl_srg_past_client_homes` (the CHILD) is THE single source of truth. EVERY resolved fact is WRITTEN there as a
column. The Audit Report, the FUB note, and the `/home` page all RENDER from the child. Nothing is "resolved" until
it is written into the child table.** The parent `hl_srg_past_clients` holds the person + `bucket` + `dossier_json`
(a **derived render of the child**) — never the place truth lives.

## The agent waterfall (LOCKED — "no agent = not resolved"; never SRG/placeholder/MLS-listing)

1. **Dotloop CONTRACT** → the signed **Licensee Assisting Seller/Buyer** = final truth (recovers departed agents the
   participant list drops). `dotloop-contract-agent.mjs → contractAgent(loopId, profileId)`; `lib/dotloop/client.ts →
   downloadDocument()`. **Precedence: dotloop CONTRACT > SISU > MLS list_agent > recon.**
2. **SISU `agent`** (client first+last + amount). Zach → Ana. Not `closings.agent` (MLS-polluted).
3. **FUB** `Team Exit - <name>` + notes/texts/calls. 4. **Still nothing → not resolved → investigation.**
   Keep the departed agent — the relationship is the asset. **Ben Froeschl is never the agent.**

## Disposition + lost listings

- **Disposition:** MLS rooftop history (`hl_mls_property_history`) PRIMARY for the resale signal > live county deed
  (owner_match = owns / owner_differs = moved) > Repliers (sold date/price) > Ninja (agents). Mover-finder (cross-county
  by owner name, deed-verified) finds the new home; else "moved — unknown, ask them."
- **Lost-listing rule (Bazalgette):** ONLY a home we BOUGHT for them that THEY later sold through a NON-SRG agent.
  **If WE sold the address, it is NEVER lost.** Match by house# + street token, not raw string.
