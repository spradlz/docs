HouseLoop's organic-search engine is a programmatic web of indexable pages: every active
listing as its own readable URL, plus a directory tree of **county → city → zip → neighborhood**
landing pages. Every page is server-rendered from live inventory, carries Schema.org
structured data, and is listed in the sitemap so Google can discover it.

## What's live today

| Surface | URL shape | Status |
|---|---|---|
| Individual listing | `/listings/{address-slug}/{hlid}/` | **Live** — readable slug, JSON-LD, canonical, OG/Twitter |
| County landing | `/{state}/{county}` | **Live** |
| City landing | `/{state}/{county}/{city}` | **Live** |
| Zip landing | `/{state}/{county}/{city}/{zip}` | **Live** |
| Neighborhood landing | `/{state}/{county}/{city}/n/{slug}` | **Live** — first-class indexed page |
| Facet variants | `?oh=1` / `?pr=Single+Family` / `?pool=1` etc. | **Live** — property type, open houses, features |
| Sitemap + robots | `/sitemap.xml`, `/robots.txt` | **Live** — chunked, every active listing + the full geo tree |
| GSC data in CRM | CRM → Settings → Website → SEO | **Live** — clicks, impressions, avg position via DWD SA |

The `/n/` prefix on neighborhoods is deliberate — it avoids colliding with the `[zip]`
segment one level up.

## The Monday pipeline

Every Monday, a chain of Inngest crons builds the week's SEO content:

```
04:00 UTC  Stats engine (geo-weekly-report) → sold/flow/window/FAQ/narrative per geo
04:30 UTC  LLM article writer (seo-articles-generate) → Sonnet/Haiku narratives
05:00 UTC  County page builder → hl_geo_page (8 counties)
06:00 UTC  Zip page builder → hl_geo_page (~205 zips)
07:00 UTC  City page builder → hl_geo_page (~134 cities)
08:00 UTC  Neighborhood page builder → hl_geo_page (~5,500 neighborhoods)
```

See [SEO pipeline — every Inngest job](/guides/seo-background-jobs) for the full reference.

## Coverage

The 8 Core Heartland counties (Jackson, Johnson, Clay, Cass, Platte, Wyandotte, Leavenworth,
Miami) → every real city → neighborhoods per city → every zip → every active listing.

The directory tree comes from one taxonomy file (`lib/geo/heartland-geo.generated.json`),
rebuilt by `scripts/build-geo-taxonomy.ts`. As of June 2026: **8 counties, 134 cities,
205 zips, 5,524 neighborhoods = 5,871 geo pages** + every individual listing.

## Page anatomy (geo + neighborhood pages)

1. **H1** — `"{Place}, {ST} Real Estate & Homes For Sale"` (Zillow phrasing, one line).
2. **Result count + sort**, filter bar, listing cards, map toggle.
3. **Breadcrumb trail** — `For Sale: Kansas • Johnson County • Overland Park` at the top of the SEO block (`BreadcrumbList` JSON-LD).
4. **Market Trends hero** — 8-card grid from `hl_geo_market_stats` (homes for sale, median sale price, sold 12mo, $/sqft, sell-to-list, days to offer, coming soon, under contract). Anchored at `#market-trends`.
5. **Weekly article** — LLM-written narrative + deterministic FAQ from `hl_geo_weekly_report`. Updated every Monday.
6. **The SEO section** — rendered dynamically by `LocationSeo`: Find a Home You'll Love (amenity) · Select Property Type · Popular Searches · Nearby City Homes ($) · Neighborhood Homes ($) · Homes by Zip ($) · Estimate Sale Proceeds · More to Explore (price bands).
7. **Compliance footer** (see below).

## Compliance — non-negotiable

<Warning>
**Aggregate sold stats ARE permitted** at neighborhood / zip / city / county level per
Heartland §15. Only **per-address** sold price is restricted (§12.3b) — relevant only to
the individual listing detail page.
</Warning>

- Per-page Heartland MO/KS disclosure: "Listings courtesy of Heartland MLS, distributed by MLS GRID."
- Per-listing brokerage + MLS# + "courtesy of Heartland MLS as distributed by MLS GRID."
- No seller PII; honor address-withhold; same-brand site; no framing.
- Dynamic filtered search URLs (`/listings?minPrice=…`) are **not** indexed — only the clean
  canonical landing pages are.

## URL scheme

```
Listing detail   /listings/{address-slug}/{hlid}/        e.g. /listings/4607-olive-st-kansas-city-mo-64130/hl_heartlandmls_2614865/
County           /{state}/{county}                       e.g. /kansas/johnson-county
City             /{state}/{county}/{city}                e.g. /kansas/johnson-county/overland-park
Zip              /{state}/{county}/{city}/{zip}           e.g. /kansas/johnson-county/overland-park/66210
Neighborhood     /{state}/{county}/{city}/n/{slug}        e.g. /kansas/johnson-county/overland-park/n/wild-horse
```

Legacy `/listings/{hlid}/` is a permanent 301 to the canonical readable URL, so every link
Google already crawled keeps its equity.

## Google Search Console

GSC data is pulled via domain-wide delegation (DWD SA impersonating `admin@houseloop.ai`).
The CRM SEO dashboard shows 30-day totals (clicks, impressions, avg position) and a daily
series. The API route is `GET houseloop.ai/api/seo/overview`.

Code: `apps/houseloop-web/lib/seo/gsc.ts` + `lib/google/dwd.ts`.
Railway env: `HOUSELOOP_GCP_SA_KEY_JSON` on the `houseloop-web` service.
