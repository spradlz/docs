This is the GT backlog for the AI clone + lead-detail capabilities. **Human ↔ AI is 1:1
parity:** every gap below is ONE shared `lib/` function wired to three surfaces — the human
button on lead detail, the AI tool in `ALL_TOOLS` (`lib/ai/lead-conversation.ts`), and the
Workflow Builder step (`capabilities.ts` `built:true`). Build once, all three light up.

## ✅ BUILT (human + AI, today)

| Capability | AI tool | Handler |
|---|---|---|
| Text a lead | `text_lead` | `lib/phone/send-sms.ts` + `send-bb.ts` |
| Email a lead (free-text) | `send_email` | `/api/email/send` (`template_key:'direct'`) |
| Search homes | `search_homes`, `get_home_details`, `lookup_property` | `/api/listings/*`, `/api/search/nlp` |
| Create appointment | `schedule_appointment` | `hl_appointments` + `google-calendar.ts` + `advance-on-appointment.ts` |
| Disposition / cancel appt | `disposition_appointment` | `lib/appointments/apply-outcome.ts` (shared w/ UI) |
| Add note | `add_note` | `hl_notes` / `/api/leads/[id]/notes` |
| Create task | `create_task` | tasks API |
| **Read a text** (inbound) | trigger + thread context | `dispatchInboundToAi` |

## Build phases (recommended)
1. **Reads first** (they inform everything): update contact (10), read home activity (12), read calendar (read).
2. **Send actions:** text listing (1), email listing (2).
3. **Requests + saved search:** accept showing (13), follow up contact (14), create saved search (3).
4. **Lead ops:** mark next action (11), update stage (5), follow up task (4), tags (6).
5. **Bigger:** market report (7), inbound email→AI, calls (8), proactive cadence (9), voice answer.
