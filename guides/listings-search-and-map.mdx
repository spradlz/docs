These are the **public website** read APIs served from `https://houseloop.ai/api`
(distinct from the CRM API at `crm.houseloop.ai/api`). They are read-only `GET`
endpoints — no auth required — backed by the live Heartland MLS listing cache.

Every search surface (the `/listings` page, the map, the home shelves, this API)
shares **one query envelope**: the same URL params mean the same thing everywhere.
No presets, no magic flags — every filter is explicit in the URL.

## Base URL

```
https://houseloop.ai/api      # production
http://localhost:3001/api     # local dev
```

## The query envelope (shared by every endpoint)

| Param | Meaning | Example |
|---|---|---|
| `city` | City name (comma-list for multi) | `city=Overland Park` |
| `state` | State abbr | `state=KS` |
| `zip` | ZIP (comma-list) | `zip=66085` |
| `boardId` | MLS board (Heartland = `157`) | `boardId=157` |
| `sort` | **Sort order — a SORT, not a filter.** Persists across filter changes. | `sort=newest` |
| `minPrice` / `maxPrice` | Price band | `minPrice=400000` |
| `minBeds` / `minBaths` | Bed / bath floor | `minBeds=3` |
| `style` | Heartland style discriminator | `style=Single Family Residence` |
| `minAcres` / `maxAcres` | Lot size band | `minAcres=2` |
| `bbox` | Viewport `W,S,E,N` (map endpoints) | `bbox=-94.8,38.8,-94.5,39.1` |
| `limit` | Page size (search caps at 100) | `limit=24` |
| `page` | 1-based page | `page=1` |

**Sort values:** `newest` (MLS list date desc), `viewsDesc` (most viewed),
`priceAsc`, `priceDesc`, `bedsDesc`, `bathsDesc`, `sqftDesc`, `lotsizeDesc`,
`priceDropDesc`, `recentlySold`. Default is `newest` when no `sort` is pinned.

<Note>
**Sort vs. filter (locked 2026-05-21):** `sort` orders results top-to-bottom and
**persists** as you add or remove location filters. Location (`city`/`zip`) is a
removable filter. So "most viewed homes in Overland Park" = `?city=Overland Park&state=KS&sort=viewsDesc`;
drop the city and you get the most-viewed homes across the whole metro, same sort.
</Note>

## GET /api/listings/search

Full search. URL params → envelope → results.

```bash
curl "https://houseloop.ai/api/listings/search?city=Overland%20Park&state=KS&sort=newest&limit=24"
```

Returns an array of `Listing` objects (price, address, beds/baths/sqft, photos,
status, `mlsListedAt`) plus engagement stats (views, reactions, shares, saves)
batched per listing. Default `limit` 24, max 100.

## GET /api/listings/count

Just the total for a query — cheap, no rows.

```bash
curl "https://houseloop.ai/api/listings/count?city=Leawood&state=KS"
# → { "totalCount": 312 }
```

## GET /api/listings/map

Markers for the current map viewport. Takes the **same envelope** plus map params,
so the map mirrors the list exactly (a 194-result search plots those 194).

| Param | Meaning |
|---|---|
| `bbox` | Viewport `W,S,E,N` to restrict markers |
| `zoom` | Current map zoom (informational; passed through) |
| `bounds=1` | **Fit mode** — skip markers, return the bounding box of the whole filtered set |
| `only` / `aimatch` | CSV of AI-match `hlid`s → those pins come back flagged `aiMatch:true` |

**Marker mode** (default — `mode:"pins"`, **no clustering, ever**):

```bash
curl "https://houseloop.ai/api/listings/map?city=Leawood&state=KS&bbox=-94.7,38.9,-94.5,39.0&zoom=13"
# → { "ok": true, "mode": "pins", "total": 312,
#      "pins": [ { hlid, lat, lng, listPrice, status, isNew, aiMatch, street, city, beds, baths, sqft, photo, brokerage }, … ] }
```

<Note>
**`total` vs. `pins.length` (locked 2026-05-24):** `total` is the **full** count of
homes matching the query within the `bbox` — this is the honest number the list view
shows. `pins` is **capped at 50 markers per viewport** (`MAX_MAP_PINS`): dots are
stride-sampled across the set, AI-match pills are always kept. So the UI renders a
"{pins.length} of {total}" results pill (e.g. "46 of 312"). The map is never a 2,000-pin
blob; zoom in → the bbox shrinks → fewer homes → eventually every home is shown.
Clustering is **banned** — individual house markers at every zoom.
</Note>

**Fit mode** (`bounds=1`) — used to auto-zoom the map to encompass every result
when entering map view (Zillow behavior), so a metro-scattered search frames all
of its homes instead of only the few in the default viewport:

```bash
curl "https://houseloop.ai/api/listings/map?city=Leawood&state=KS&bounds=1"
# → { "ok": true, "total": 312, "bounds": { "north": 39.01, "south": 38.90, "east": -94.55, "west": -94.69 } }
```

<Note>
**Search-as-you-move (client behavior, 2026-05-24):** in map/split view the client pushes
the live viewport `bbox` into the URL on every pan/zoom, so the list **and** the result
count re-scope to exactly what's on screen — `bbox` → `geo.bbox` flows through the same
shared envelope into the search `WHERE`. The map renders **dots** by default and flips
markers to compact **price pills** by pixel-spacing (never stacked); at a ~5mi frame every
marker becomes a pill. AI-search matches always render as ✨ pills. Full UI spec:
`GT_HOUSELOOP_MAP_VIEW.md §0`.
</Note>

## GET /api/geo/place-boundary

The Census TIGER named-place outline drawn around a searched city (single `city` + KS/MO).
TIGERweb proxy with a 30-day cache; silently degrades to no outline if the place isn't found.

```bash
curl "https://houseloop.ai/api/geo/place-boundary?name=Overland%20Park&state=KS"
# → { "ok": true, "geometry": { "type": "MultiPolygon", "coordinates": [ … ] } }
# bogus place → { "ok": false }
```

## GET /api/search/classify

Intent classifier that routes the search bar. Legacy/deterministic search handles only the
six facets (location, home type, status, beds, baths, price); **anything beyond** (features /
amenities like "white kitchen", "pool", "open floor plan") routes to AI Smart Home Search.
No regex — an LLM (gpt-4o-mini) classifies. See the [AI Smart Home Search](/guides/ai-smart-home-search)
guide for the AI side.

`POST` with a JSON body (not GET):

```bash
curl -X POST "https://houseloop.ai/api/search/classify" \
  -H "content-type: application/json" \
  -d '{"q":"homes with a white kitchen in Overland Park"}'
# → { "intent": "ai" }      # plain "overland park" → { "intent": "normal" }
```

`/api/search/nlp` is also `POST {"q":"…"}` — run it after classify returns `intent:"ai"`.

## GET /api/home

The composed home page: the ordered shelves + inspiration links. Cached 5 min.

```bash
curl "https://houseloop.ai/api/home"
# → { "shelves": [ { slug, title, href, items: Listing[], display }, … ], "inspiration": {…} }
```

**Shelf model (locked 2026-05-21):** each shelf is the **12 newest-listed homes**
in its filter, in order — new listings push in at the front, the oldest drops off.
Shelves: `all-kc-metro` ("All Homes In KC Metro"), per-city rows
(`overland-park`, `leawood`, …), and `acreage`. The hero carousel has its own
composer. Shelves are DB-config (`hl_home_shelf` table, seeded from
`prisma/seed/home-shelves.ts`) — not hardcoded.

## GET /api/locations/autocomplete

Typeahead for the search bar. `q` must be ≥ 3 chars.

```bash
curl "https://houseloop.ai/api/locations/autocomplete?q=overland"
# → { addresses, cities, counties, neighborhoods, schoolDistricts, schools }
```

## GET /api/listings/{hlid}

A single listing by HouseLoop id, including the full `photos.all` gallery and
price `history`.

```bash
curl "https://houseloop.ai/api/listings/HL-123456"
```

## IDX attribution (MLS GRID / Heartland)

HouseLoop displays Heartland MLS listings via the MLS GRID IDX feed, so the
[MLS GRID IDX Rules](https://www.mlsgrid.com) govern where broker attribution and the
source disclaimer must appear. The key lever is the **200-character minimal-info exemption**.

> **§13/§23/§24/§25 (verbatim):** *"Displays of minimal information (e.g. a one-line or 'thumbnail,'
> text messages, 'tweets', etc. of two hundred (200) characters or less) are exempt from this
> requirement but only when linked directly to a display that includes all required disclosures."*

### Where attribution goes

| Surface | Component | Attribution |
|---|---|---|
| Gallery / list card | `components/listing/ListingCard.tsx` | **None** — minimal-info ≤200 chars linking to detail (§13/§23). §10: thumbnails carry no brokerage branding. |
| Map pin preview ("Tap to open") | `components/listings/ListingsMap.tsx` | **None** — same exemption (§24/§26). |
| Listing **detail** page | `components/listing/ListingDetail.tsx` | **Required** — brokerage name (§23) + `Source: Heartland MLS as distributed by MLS GRID · MLS #{n}` (§24) + Heartland logo. |
| Site **footer** (every page) | `components/site/Footer.tsx` | **Required** — §24 source + logo, §25 full disclaimer + consumer-use, §32 DMCA. Site-wide carrier that keeps the cards/pins exempt. |

### Reusable logo

`components/listing/HeartlandMlsLogo.tsx` wraps the transparent logo at `/public/heartland-mls.png`.
Always use this component — never scatter raw `<img src="/heartland-mls.png">`, and **never render it on a
minimal-info surface** (cards, map pins, thumbnails).

### Footer disclaimer (required text, §25)

> Based on information submitted to the MLS GRID as of {date} CT. All data is obtained from various sources and
> may not have been verified by broker or MLS GRID. Supplied Open House information is subject to change without
> notice. All information should be independently reviewed and verified for accuracy. Properties may or may not be
> listed by the office/agent presenting the information. Some IDX listings have been excluded from this website.
> Sold prices are not displayed.

### Other hard rules

No seller name/phone/email (§30) · no listing-agreement type (§21) · refresh ≤12h (§11) · ≤500/2,500 listings per
search (§27) · per-address **sold price never displayed** (Heartland §12.3(b)). Source of truth:
`products/houseloop/Drop/IDX Rules.pdf` + `GT/GT - ACTIVE/GT_HOUSELOOP_SEO.md` §15.3.
