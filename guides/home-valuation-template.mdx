The Home Valuation Report is a monthly personalized email sent to past clients from their assigned agent (Ana Spradling, Zach Spradling, or Myli). It combines a personal note with a full property valuation report.

## Email Structure

The email has two distinct parts:

### 1. Personal Message (above the report)

A 2-3 sentence casual note from the agent. AI-generated using merge fields but reads like a human wrote it. No branding above it — reads like a plain email.

**Example:**
> Hey John,
>
> Just wanted to send over your June home value update — looks like your home went up another **$13,000** this month! The Blackthorne Estates neighborhood is continuing to do really well, and your home is actually outperforming the broader KC market. Here's your full report below.
>
> Hope you and the family are doing great!
>
> **Ana Spradling**
> Spradling Group | eXp Realty

### 2. Report Card (the embedded report)

A single white card with rounded corners containing:

| Section | Description |
|---------|-------------|
| **Header** | Lockup A (stacked: Spradling Group eXp / Powered By / HouseLoop) + "Home Valuation Report" title + month pill |
| **Property photo** | Hero image scraped from Redfin, stored in R2 |
| **Address card** | Street, city/state/zip, beds/baths/sqft/year/subdivision |
| **Estimated value** | Big bold number on white card, 56px DM Sans |
| **Valuation range** | Low / Estimate / High with House Canary-style bar chart distribution (blue→green→yellow) |
| **Value history chart** | SVG line chart from purchase date to current, with gradient fill |
| **Market temperature** | Green-to-red gauge bar with plain English verdict ("Balanced & Stable") + DOM / list-to-sale / inventory |
| **Beating the market** | Three stat boxes: Your Home vs ZIP vs KC Metro appreciation |
| **Appreciation cards** | +X% / +$X green card alongside purchase price/date card |
| **Comparable sales** | Horizontal scroll of comp cards with photos, address, price, sold date, bed/bath/sqft, distance, similarity % |
| **Agent + CTA** | Agent headshot + contact + "Schedule a Call" full-width button |
| **Brand band** | Dark blue footer with white Spradling Group / eXp / HouseLoop logos + disclaimer |

## The data contract (HvrData)

The report's field shape is the canonical **`HvrData`** interface in `apps/houseloop-web/components/listing/HomeValuationReport.tsx`: `estimatedValue`, `valueLow/valueHigh`, `psf*`, `purchasePrice/Date`, `appreciationPct`, `equityGain`, market temp (`avgDom`/`listToSalePct`/`inventoryMonths`), `zip/metroAppreciation`, `comps[]`, `valueHistory[]`, `beds/baths/sqft/yearBuilt/neighborhood`, `photoUrl`. Every surface renders this same shape.

## Data Storage — the ONE table

Every valuation snapshot lives in a **single table: `hl_srg_home_valuations`** (CRM DB, migration `0136`), one row per past client (`past_client_id` → `hl_srg_past_clients.id`). The full `HvrData` snapshot sits in a `data` JSONB column; hot fields (`estimated_value`, `equity`, `appreciation_pct`, …) are promoted to scalar columns for sorting.

**There is exactly one valuation table — no parallel valuation tables anywhere.** Every surface reads it: the [Dashboard](/guides/home-valuation-dashboard), the CRM Home Report modal, the web `/home/[slug]/report`, and this email. The only writer is the monthly Inngest cron. An absent / NULL row renders "—". SSOT: `GT_HOUSELOOP_MASTER.md` §12.

## Data Requirements

### Available now:
- Purchase price + date (SISU import)
- Comps + neighborhood / market temperature (Repliers sold listings + `hl_geo_market_stats`)
- Photos (Redfin scraper → R2)

### Needs building (writes into `hl_srg_home_valuations`):
- **Realie AVM call** — `modelValue` (estimate), `modelValueMin/Max` (range), `equityCurrentEstBal` (equity), `totalLienBalance` (mortgage owed), `LTVCurrentEstCombined` (LTV), `lenderName`
- **Comps engine** — blended 50/50 with the Realie AVM when 3+ comps
- **Value history chart data** — monthly estimates (interpolated from zip trends + purchase/current anchors)
- **AI contextual message** — LLM call at send time from merge field data
