## The model (locked 2026-07-13): services are JOBS, equipment is configured once

**Agents are the workers · services are the JOBS they hold · voice, inboxes, numbers, and the pond pool are
the EQUIPMENT.** The six jobs: **Receptionist · Inside Sales · AI Clones · Database Manager · Transaction
Coordinator · Content Manager.** The one-question law decides where every configuration lives: *will more
than one job ever touch this thing?* **Yes** → it's equipment with its own config page (Voice · Inboxes ·
Numbers). **No** → the config lives on the job's page (pond rules → Inside Sales · audit rules → Database
Manager · writing → Content Manager).

## The four blocks (top to bottom)

1. **Your AI services** — the six job boxes, one line, **these ARE the master switches** (no separate
   masters row). Each box: icon / full name (never abbreviated, never wrapped) / centered toggle / prose /
   Configure. Real wires only: **Inside Sales** = `hl_settings.ai_conversations.enabled` (the follow-up
   engine, fail-open) · **Receptionist** = `voice_enabled` (fail-closed). Clones/DBM/TC/CM show disabled
   toggles with honest copy until their flags exist — no orphan flags, ever. The page header carries the ONE
   global **All AI** switch: OFF kills `enabled` + `voice_enabled` together; ON restores the engine only
   (voice returns on its own switch — fail-closed).
2. **Your AI team — on duty** — one BUSINESS CARD per AI agent, atomic single-value rows so nothing wraps:
   Status · Duties (icons from REAL seats — inbox manager count + the calls channel, never hardcoded) ·
   Leads · Inboxes · Actions today · Appts 7d · Spend month, plus the Add Agent ghost. Data:
   `/api/ai-follow-up/agent-cards`.
3. **Team pulse** — THE PULSE DECK (`components/team/PulseDeck.tsx`): the ten-card member instrument at
   AI-team scope — Total leads · Phone calls · Texts · Emails · Tasks · Homes searched · Facts captured
   [Notes · Dossiers] · Appointments set · **Income contributed** (AI-attribution closed GCI, see the
   2026-07-22 note above) · **Spend** [Conversation · Extraction] (per-actor MODEL_PRICING). Pills INSIDE
   each hero, 2×5 grid, Today/**7d**(default)/MTD/YTD, month-vs-cap capline below. Data:
   `/api/team/pulse?scope=ai-team`. The SAME deck renders on every member page at one-person scope (humans
   get the 8-card 2×4 without Facts/Spend). EXTENSION LAW: each future service ships its win-card into the deck.
4. **Live AI Activity** — the compact table **When · Agent · Action · Lead · Result · Cost** under a
   `MenuTable` header (agent tabs · lens filters). The **Agent** cell shows a robot-head icon + the real actor
   name (clones marked "(clone)"). Cost = the row's stamped tokens priced at the actor's model rate, computed
   by `/api/ai-follow-up/feed`. The table streams the recent tail; the full [Run Log](/guides/run-log) stays
   the forensic timeline.
