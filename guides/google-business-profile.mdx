## What it is
**CRM → Website → Analytics → Google Business.** A dashboard of how people find and act on the Spradling Realty
Group listing on **Google Search + Maps**: total impressions (split search vs maps, desktop vs mobile), website
clicks, calls, and direction requests. This is our **local-SEO** surface. It reads **live** from Google's
Business Profile Performance API on every load — no stored data. Until API access is granted it renders a
"pending access" card instead of the tiles.

## The architecture — three identities that must line up
This integration keeps confusing people because **three separate things** each carry an identity, and they are
**not** the same identity today. All three must point at the same business for data to flow.

| Thing | What it is | Identity today |
|---|---|---|
| **The Profile** | The Google Business listing itself ("The Spradling Group – EXP Realty") | Owned by **zach@spradlingrealtygroup.com** + **ana@spradlingrealtygroup.com**; `admin@houseloop.ai` was invited as an owner (2026-05-27) |
| **The API quota grant** | GBP API access is granted **per Google Cloud project**, not per profile | Project **`houseloop-496200`** (the HouseLoop project). API **Enabled**, but **quota = 0** → not granted |
| **The Reader** | The code that fetches the data | `lib/analytics/gmb.ts` — a **domain-wide-delegation (DWD)** service account (`houseloop-master@houseloop-496200.iam.gserviceaccount.com`) **impersonating `admin@houseloop.ai`**, scope `business.manage` |

The reader walks three Google APIs (all on `houseloop-496200`): `mybusinessaccountmanagement` (list accounts) →
`mybusinessbusinessinformation` (list locations, auto-pick the one whose title matches "Spradling") →
`businessprofileperformance` (fetch the daily metrics). `GMB_LOCATION_ID` (`locations/{id}`) is an optional env
fast-path; if unset it resolves the location at runtime.

## Why the reader MUST be admin@houseloop.ai (and not Spradling)
Domain-wide delegation is a **Google Workspace mechanism scoped to the `houseloop.ai` workspace**. The service
account can impersonate `admin@houseloop.ai`, but it **cannot** impersonate `zach@` or `ana@`
(`spradlingrealtygroup.com` is a different workspace). So:

- The reader identity is locked to **`admin@houseloop.ai`**.
- For it to see the Spradling listing, **`admin@houseloop.ai` must be a manager/owner of the Spradling profile.**

That constraint is why the fix is **Path A** (keep the HouseLoop plumbing, make the *request* match Spradling) —
going "pure Spradling" would mean rewriting the reader to OAuth as zach@/ana@, for zero functional gain.

## Access model + the June 2026 history
Google's Business Profile APIs ship with **quota 0** after you enable them. You then submit a one-time
**GBP API access request** describing your business + website; Google reviews whether the **applicant matches a
business that legitimately manages the profile**, and grants quota to the **project**.

| Date | Event |
|---|---|
| 2026-05-27 | Ana invited `admin@houseloop.ai` as an owner of "The Spradling Group – EXP Realty". |
| 2026-06-15 | **Access request `0-2380000041528` REJECTED** — *"did not pass our internal quality checks… ensure your Business Profile and official website are up to date."* Root cause: filed under **HouseLoop** identity, but the verified profile is **The Spradling Group** (mismatch). |
| 2026-06-15 | `zach@spradlingrealtygroup.com` became a verified **Owner** of the profile. |

## Path A — the fix (runbook)
Keep the existing HouseLoop project/reader; make the access request describe the business Google wants to see.

1. **Make `admin@houseloop.ai` an Owner of the Spradling profile.** It was invited 2026-05-27 — confirm it's
   still an active Owner/Manager on "The Spradling Group – EXP Realty" (Business Profile → Settings → People &
   access). This is what lets the DWD reader see the location, and what ties the HouseLoop project's identity to
   the verified profile.
2. **Re-file the GBP API access request for project `houseloop-496200`** ([request form](https://developers.google.com/my-business/content/prereqs#request-access)),
   describing the business as **The Spradling Group** with website **spradlingrealtygroup.com** — matching the
   verified profile. That match is exactly what the prior request failed.
3. **No code change.** The CRM already reads `houseloop-496200`. When Google grants access, the quota flips above
   0 and the tab **auto-fills** on next load (`configured:false` → live tiles).

Do **not** file this on a Spradling-owned project — the reader can only use `houseloop-496200`'s service account.

## How to check status
- **Quota (the source of truth):** [Cloud Console → Business Profile Performance API → Quotas](https://console.cloud.google.com/apis/api/businessprofileperformance.googleapis.com/quotas?project=houseloop-496200)
  (sign in as `admin@houseloop.ai`). Watch **"Requests per minute"** — `0` = not granted; **> 0 = granted**.
- **Decision email:** arrives from `googlebusinessprofile-support@google.com` (subject carries the case number)
  to the applicant + `admin@houseloop.ai`.

## Troubleshooting
| Symptom | Meaning |
|---|---|
| Tab shows "Check back / pending access" | The live API returned a quota/permission error — access not granted. Check the quota page. |
| Quota "Requests per minute" = `0` | Access **not granted** yet — run the Path A runbook. |
| `403` / `PERMISSION_DENIED` / `accessNotConfigured` / `SERVICE_DISABLED` | API access not configured/granted on `houseloop-496200`. |
| `configured:false` / "no location" (API works but no data) | `admin@houseloop.ai` isn't a manager on the profile, **or** no location titled "Spradling" resolves — fix Path A step 1 (or set `GMB_LOCATION_ID`). |
