## What it is
**CRM → Website → Analytics → Google Business.** A dashboard of how people find and act on the Spradling Realty
Group listing on **Google Search + Maps**: impressions (search vs maps, desktop vs mobile), website clicks,
calls, and direction requests. This is our **local-SEO** surface. It reads **live** from Google's Business
Profile Performance API on every load — no stored data. Until API access is granted it renders a "pending
access" card instead of the tiles.

## The architecture — three identities that must line up
This integration confuses people because **three separate things** each carry an identity.

| Thing | What it is | Identity |
|---|---|---|
| **The Profile** | The Google Business listing ("The Spradling Group – EXP Realty", 7900 College Blvd, Overland Park KS, Verified) | Owners: **zach@** + **ana@spradlingrealtygroup.com**; **admin@houseloop.ai** is a manager (invited 2026-05-27, accepted) |
| **The API quota grant** | GBP API access is granted **per Google Cloud project**, not per profile | Project **`houseloop-496200`** (HouseLoop). All 3 APIs Enabled; **quota = 0** until Google grants |
| **The Reader** | The code that fetches the data | `lib/analytics/gmb.ts` — a **domain-wide-delegation (DWD)** SA (`houseloop-master@houseloop-496200.iam.gserviceaccount.com`) **impersonating `admin@houseloop.ai`**, scope `business.manage` |

The reader walks three APIs (all on `houseloop-496200`): `mybusinessaccountmanagement` → `mybusinessbusinessinformation`
(auto-picks the location titled "Spradling") → `businessprofileperformance`. `GMB_LOCATION_ID` (`locations/{id}`)
is an optional env fast-path; unset by default (auto-resolves).

## Why the reader MUST be admin@houseloop.ai
Domain-wide delegation is a **`houseloop.ai`-workspace mechanism**. The SA can impersonate `admin@houseloop.ai`
but **not** `zach@`/`ana@` (different workspace). So the reader identity is locked to `admin@houseloop.ai`, and
**admin@houseloop.ai must be a manager/owner of the Spradling profile** (it is). That's why the fix is **Path A**
(keep the HouseLoop plumbing, make the *request* match Spradling) — "pure Spradling" would mean rewriting the
reader to OAuth for zero functional gain.

## How to check back
- **Quota (the source of truth):** [Cloud Console → Business Profile Performance API → Quotas](https://console.cloud.google.com/apis/api/businessprofileperformance.googleapis.com/quotas?project=houseloop-496200)
  (sign in `admin@houseloop.ai`). **"Requests per minute" > 0 = GRANTED.**
- **Decision email:** from `googlebusinessprofile-support@google.com`, to the filer **ana@spradlingrealtygroup.com**
  (and possibly admin@houseloop.ai as a profile manager).
- **The tab itself:** once quota > 0, CRM → Analytics → Google Business populates the tiles automatically.
- **Reminder:** a daily check-back is scheduled internally; the durable record is the GT tracker
  `GT/GT - ACTIVE/GT_HOUSELOOP_GMB_API_ACCESS_WIP.md`.

<Note>
**On the "verify it's you" phone prompts (why forwarding was dropped):** Ana's 2-Step Verification is OFF (login
needs no prompt). But Google fires a separate **step-up challenge on *sensitive changes*** (like editing Gmail
forwarding) from an unrecognized browser — its only option is "tap Yes on Ana's phone," and **no admin setting
disables it**. It's per-device-trust: approve once on a browser and it stops re-challenging. We therefore
**dropped the email-forwarding plan** (it was only a convenience) and monitor Ana's inbox + the quota directly.
</Note>

## Troubleshooting
| Symptom | Meaning |
|---|---|
| Tab shows "Check back / pending access" | Live API returned quota/permission error — access not granted yet. Check the quota page. |
| Quota "Requests per minute" = `0` | Access **not granted** — case `8-0566000041156` still under review. |
| `403` / `PERMISSION_DENIED` / `accessNotConfigured` / `SERVICE_DISABLED` | API access not granted on `houseloop-496200`. |
| `configured:false` / "no location" (API works, no data) | admin@houseloop.ai lost manager access, **or** no "Spradling" location resolves — re-confirm profile access (or set `GMB_LOCATION_ID`). |
