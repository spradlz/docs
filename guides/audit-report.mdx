The **Audit Report** is the human-readable proof surface for the [SRG Past-Client Reconciliation](/guides/srg-reconciliation-engine) pipeline. It's where you (or Zach) check the resolver's work on every past client — the transactions it found, who it decided each house belongs to, and the exact dossier note that landed in Follow Up Boss.

It lives in the CRM under **Seller Report → Audit Report**.

## What it is

A full-width CRM page (`apps/houseloop-crm/components/seller-report/AuditReport.tsx`) backed by `/api/settings/seller-report/audit`, which reads the resolver's output (`resolver_block` jsonb on `hl_srg_past_clients`) **live** — no copy, no cache. What the resolver wrote is what the Audit Report shows.

It is intentionally rendered with **gold tables** (separate from the tokenized CRM `DataTable` — it carries an `eslint-disable hl/no-raw-table`), with tokenized fonts/buttons and a navy header, so it reads as a report rather than a data grid.

## The jump bar

Four anchored sections:

| Section | What it shows |
|---|---|
| **All** | Every resolved client, one row each — name, bucket, current-home disposition, transaction count. |
| **Transaction History** | Per client: every deal we did with them — date, BOUGHT/SOLD, address, price, and the **SG agent who repped that deal** (chain of custody). |
| **Resolution Matrix** | How each house was resolved — the determined agent, the disposition verdict (deed-confirmed / sold / unverified), and the **`*_source` provenance** for each fact (dotloop contract vs SISU vs MLS vs recon). This is the "show your work" view. |
| **Follow-Up Boss Notes** | The exact dossier note (`note_markdown`) the resolver produced and the transport wrote to FUB — the [card format](/guides/srg-reconciliation-engine#the-card-the-follow-up-boss-note-format-locked). |

## Deep link to Follow Up Boss

Each client name links straight to their **FUB profile**:

```
https://spradlinggroup.followupboss.com/2/people/view/<fub_id>
```

So you can jump from the resolver's verdict to the live person record in one click — confirm the note landed, cross-check the contact, see the real timeline.

## What it reads

| Field on `resolver_block` | Surfaces as |
|---|---|
| `contacts[]` | the contact line(s) |
| `transactions[]` (each carries `agent`) | Transaction History + the per-line agent column |
| `disposition` / `dispositions[]` | current-home verdict (deed-confirmed / sold / unverified) |
| `bucket` + `bucket_basis` | the bucket pill + why it resolved that way |
| `dnc` | DNC / Internal suppression state |
| `tags[]` | the FUB tag taxonomy applied |
| `note_markdown` | the Follow-Up Boss Notes section |

Because it reads `resolver_block` directly, the Audit Report is always in sync with the latest resolver run — re-run the resolver on a client and refresh the page to see the new verdict.
