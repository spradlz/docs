---
title: "Website Pixel Tracking"
description: "HouseLoop's first-party tracking pixel — capture visitor behavior on your site and link anonymous activity to leads. No third party."
---

The HouseLoop pixel is **first-party**: it tracks page views, property views, and
high-intent actions on houseloop.ai, then POSTs them to **your own** CRM ingest
(`/api/pixel/track`) — the data lands in your database, never a third party. When a
visitor identifies (sign-up / login), all of their prior anonymous activity links to
their lead automatically. Manage it in the CRM under **Settings → Website → Pixel
Tracking**, and view per-lead activity on the lead-detail **Homes** and **Requests** tabs.

## How it flows

```
houseloop.ai            CRM ingest                       CRM lead detail
hl-pixel  ──fires──▶  POST /api/pixel/track  ──▶  hl_pixel_event  ──▶  Homes / Requests feed
hlIdentify(email)        (identity-stitch by email)
```

## 1. Install

On houseloop.ai the pixel is **built in** (`HlPixelInit` in the app layout) — it mints
an anonymous `visitor_id`, fires `page_view` on every route, and POSTs to the CRM. There
is no snippet to paste. To fire events from your own code, import the helpers:

```ts
import { hlTrack, hlIdentify } from "@/lib/pixel/hl-pixel";

hlTrack("page_view", { payload: { path: location.pathname } });
```

## 2. Identify logged-in users

After login/signup, call `hlIdentify` with the user's email. This links every prior
anonymous event from that browser to the lead's record (matched by email).

```ts
hlIdentify({ email: user.email, name: user.name });
```

<Note>
Call this once after authentication with the **actual** logged-in user's email. On
houseloop.ai it fires automatically via `HlPixelIdentify` after NextAuth hydrates.
</Note>

## 3. Track property views

Fired automatically on every listing detail page (`HlPropertyTrack`). To fire manually:

```ts
hlTrack("property_view", {
  mls_number: "12345",
  property: {
    street: "123 Main St", city: "Olathe", state: "KS", zip: "66061",
    price: 765000, beds: 4, baths: 4, sqft: 2920,
    propertyType: "Single Family Residence", photo: "https://…",
  },
});
```

The CRM derives a **repeat-view count** per lead + home (the "viewed N×" badge in the feed).

## 4. Event catalog

| Event | Fires when |
| --- | --- |
| `page_view` | Any page / route change (auto) |
| `property_view` | Listing detail page (auto) |
| `save_home` / `unsave_home` | Visitor saves / unsaves a listing |
| `like_home` | Visitor likes a listing |
| `share_home` | Visitor shares a listing |
| `tour_request` | Visitor requests a tour (carries email → identifies the lead) |
| `saved_search` | Visitor saves a search |
| `account_created` / `login` | Sign-up / login |
| `cta_click` | Visitor clicks a tracked CTA |

New event types are just new strings passed to `hlTrack` — no schema change required.

## 5. View the activity

- **CRM lead detail → Homes tab:** the lead's property events (viewed / saved / liked /
  shared) — each card shows photo, price, address, beds/baths, timestamp, a repeat-view
  badge, and a **View Property Details** link.
- **CRM lead detail → Requests tab:** high-intent actions (tour requests, saved searches).
- **Settings → Website → Pixel Tracking:** live event counts + recent activity across all visitors.

<Note>
All behavioral data stays in your HouseLoop database (`hl_pixel_event`). The pixel does
not send anything to any third-party tracking service.
</Note>

## Analytics dashboard & bot classification

**Settings → Website → Analytics** (`GET /api/pixel/analytics?days=`) turns the raw
events into a SureSend-style dashboard: six human metric tiles (Page Views, Unique
Visitors, Identified Leads, Form Submissions, Property Views, Active Sessions), each
driving a full-width time-series, plus Top Pages.

<Warning>
**Bots are split out, not counted as visitors.** Raw pixel events count every crawler
hit — early on, "1,002 unique visitors" was actually **991 Googlebot/GoogleOther + 11
humans**. At ingest (`/api/pixel/track`) each visitor's user-agent is classified
(`hl_pixel_visitor.is_bot` / `bot_name`). The dashboard's **human metrics exclude bots**;
a dedicated **SEO Crawlers** tab reports crawl volume, by-crawler (Googlebot, GoogleOther,
GPTBot, ClaudeBot, …) and top crawled pages as an SEO-health signal.
</Warning>

Property-scoped events also carry the **street address** (built from the pixel's
`street`/`city`/`state`), so the CRM **Run Log → Website Activity** lane reads
`Viewed · 12315 W 183rd Terrace, Overland Park, KS` rather than a bare MLS number.
