---
title: "Website Pixel Tracking"
description: "Install the tracking pixel on your website to capture visitors and link anonymous activity to leads."
---

The HouseLoop pixel tracks page views, property views, and form activity on
your public website, then links anonymous sessions to a lead the moment the
visitor identifies (sign-up / login). Manage it in the CRM under
**Settings → Website → Pixel Tracking**.

## 1. Install the pixel

Paste this into the `<head>` of every page on your site. The tracking code is
unique to your account — copy yours from the Pixel Tracking page (the snippet
there is pre-filled with your real ID).

```html
<!-- Sure Send CRM Pixel -->
<script>
(function(w,t,c){w['SureSendPixel']=c;(w[c]=w[c]||function(){
(w[c].q=w[c].q||[]).push(arguments);}),(w[c].t=1*new Date());
var s=document.createElement('script');
s.async=1;s.src=t;
document.head.appendChild(s);
})(window,"https://suresend.ai/pixel/SS-XXXXXXXXXX.js","ssPixel");
window.ssPixel("init", "SS-XXXXXXXXXX");
window.ssPixel("track", "pageview");
</script>
<!-- End Sure Send CRM Pixel -->
```

Once installed, every page load fires a `pageview` event. You'll see them
appear in **Recent Activity** within seconds.

## 2. Identify logged-in users

After a visitor logs in or signs up, call `identify` with their email. This
links every anonymous page view from that browser session to the lead's
contact record.

```html
<script>
window.ssPixel("identify", {
  email: "user@example.com",   // Required
  name:  "John Doe",           // Optional
  phone: "+1-555-123-4567"     // Optional
});
</script>
```

<Note>
Place the `identify` call on pages that render only after authentication, and
pass the **actual** logged-in user's email from your system — not the example.
</Note>

## 3. Track property views

On property detail pages, fire a `property_view` event with the listing
details. `mlsNumber` is required for the CRM to match the view to a property.

```js
ssPixel('track', 'property_view', {
  mlsNumber:    '12345',
  street:       '123 Main St',
  city:         'San Francisco',
  state:        'CA',
  zipCode:      '94102',
  price:        850000,
  propertyType: 'Condo',
  bedrooms:     '2',
  bathrooms:    '2',
  squareFeet:   '1200',
  lotSize:      '0.25',
  forRent:      false
});
```

### Available fields

| Field | Type | Notes |
|---|---|---|
| `mlsNumber` | string | **Required** for property matching |
| `street`, `city`, `state`, `zipCode` | string | Property address |
| `price` | number | Asking price |
| `propertyType` | string | e.g. "House", "Condo", "Townhouse" |
| `bedrooms`, `bathrooms` | string | Counts |
| `squareFeet` | string | Property square footage |
| `lotSize` | string | Lot size in acres |
| `forRent` | boolean | `true` for rentals |

## 4. Verify it's working

1. Turn on **Debug Mode** in Pixel Tracking → Tracking Settings.
2. Visit a property page on your site.
3. Open the browser console (F12) and look for `Sure Send Pixel` log lines.
4. Confirm the view shows up under **Recent Activity** in the CRM.

## Tracking settings

| Setting | What it does |
|---|---|
| **Enable Pixel Tracking** | Master switch — page views and visitor activity |
| **Form Capture** | Auto-captures form submissions |
| **Call to Action** | Shows a CTA on property pages |
| **Debug Mode** | Console logging for testing |
