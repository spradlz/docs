## Session Cookie (Browser / CRM Users)

The CRM UI authenticates via a session cookie set by `POST /api/auth/login`.
All browser requests include this cookie automatically.

```bash
# Log in and save the session cookie
curl -c cookies.txt -X POST https://crm.houseloop.ai/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "you@houseloop.ai", "password": "yourpassword"}'

# Use it in subsequent requests
curl -b cookies.txt https://crm.houseloop.ai/api/leads
```

## Google OAuth (SSO)

Users can also log in via Google. Redirect to `GET /api/auth/google` and the
callback at `GET /api/auth/google/callback` sets the session cookie.

## Machine Auth — `x-hl-internal-secret` (Myli, AI agents, automation)

**Anything a human can do in the CRM, an AI or backend job can do too.** Trusted
server-to-server callers — Myli, the scheduled-action executor, internal jobs,
and AI agents — authenticate by sending the `x-hl-internal-secret` header instead
of a browser session cookie. It opens **every** endpoint a logged-in human can hit.

```bash
# The secret lives in the credential vault as HL_INTERNAL_SECRET (server-only,
# never exposed to a browser). Read it from .env / the vault, then:
curl https://crm.houseloop.ai/api/leads \
  -H "x-hl-internal-secret: $HL_INTERNAL_SECRET"

# Writes work identically — e.g. recolor an appointment type:
curl -X PATCH https://crm.houseloop.ai/api/appointment-types/$ID \
  -H "x-hl-internal-secret: $HL_INTERNAL_SECRET" \
  -H "Content-Type: application/json" \
  -d '{"color": "#3B82F6"}'
```

The middleware (`middleware.ts`) checks `req.headers['x-hl-internal-secret']`
against the `HL_INTERNAL_SECRET` env var. If it matches, the session-cookie
redirect is bypassed and the request proceeds. This is the canonical path for
agents — do **not** look for an `Authorization: Bearer` scheme; the CRM does not
use one.

<Warning>
`HL_INTERNAL_SECRET` is a master key — one header opens write access to every
endpoint. Keep it server-side only. Never ship it to a browser or commit it.
</Warning>

## Errors

All 401 responses follow this format:

```json
{ "ok": false, "error": "unauthorized" }
```
