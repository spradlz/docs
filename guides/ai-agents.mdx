## The roster (`/ai/agents`)

**THE FIXED CARD LAW:** every agent card is the same 300px box, forever. A blue-tint top band (64px
headshot + status dot · name with ellipsis · role uppercase), then **four fixed stat slots — always the
same four, "—" when empty**: Leads assigned · Actions 7d · Capabilities n/26 · Spend MTD. One action:
**Open Agent** — the card IS the door. No pill, long name, or extra capability can ever reflow a card.
Data: `/api/ai-follow-up/agent-cards` (leads, 7-day actions, MTD tokens priced at the agent's model rate).

**Roles ride as the second tab** of the same blue header (no second blue header): the `hl_role` table
straight up — persona · `tools` capability gates · docs · model, with "worn by" from `hl_member_role`.
Reuses `RolesTable` + `AddRoleModal` verbatim.

## Agent Detail (`/ai/agents/[id]`)

Header = identity (name · role · line · email) + the agent's **own on/off switch** (`hl_users.is_active`)
— pausing one agent never touches the masters. Five tabs:

1. **Overview** — the week at a glance (`StatStrip`: Actions 7d · Texts · Calls answered · Appts set · Spend MTD).
2. **Persona** — the composed prompt the agent actually hears; each layer edits at its source
   (role shell → per-agent override). Reuses `AgentRolePanel`.
3. **Capabilities** — the 26 gates from `CAPABILITY_TREE`, role-inherited with per-agent overrides
   highlighted (`AgentRolePanel` permissions tab).
4. **Connections** — the accounts and lines the agent operates (`MemberConnections`, reused verbatim).
5. **Assignments** — ONE answer to "what is this agent responsible for": Leads (count + link) ·
   Voice agent (from real inbox seats — the calls channel) · Inbox manager (per-inbox seats, managed on
   each inbox) · Pond and Database Manager (honest "Not set up" until C5/C6 ship). Flip a seat here or on
   the service's page — SAME seat, one truth.

## Roles & team-member identity (the model — folded from the retired doc)

**Roles are reusable system-level shells; members hold MULTIPLE roles.** A role (`hl_role`) carries a
`persona`, a `tools` capability JSON, a Claude `model`, `workflow_ids`, reference `docs`, and flags
(`is_system`, `grants_tc`). Seeded: **AI Clone** (ai, system) · **AI ISA** (ai, system — Myli) ·
**Real Estate Agent** (human) · **Transaction Coordinator** (human, `grants_tc`).

Roles attach many-to-many via **`hl_member_role`**, and configuration **overrides per member on the
join row** (`persona`, `voice_persona`, `tools`, `docs`, `workflow_ids`) — the shell is the reusable
default; the join is where one member diverges. The **AI Clone role is configured at assign-time**: the
clone's tuning lives on that member's override, so every human's clone is independently tuned off one shell.

**Clone vs Myli — the clone is a STATE of the employee, Myli is her own actor:**

| | **AI Clone** | **Myli** |
|---|---|---|
| What it is | the **same human, in an AI state** — not a user row (mode-on-human, mig `0392`; 0 clone rows) | a standalone `type='ai_agent'` user |
| Comms identity | none of its own — the human's number/email/calendar, always | own email / phone / iMessage |
| Lane | **LEFT** (it IS the human) | **RIGHT** |
| Enabled via | `hl_agent_clone_config.enabled` + per-lead `ai_mode_enabled` | `assigned_ai_agent_id` |
| Attribution | `actor` = the human's id, `actor_role='ai_agent'`, **`actor_kind='clone'`** (mig `0413`) | `actor` = Myli's id, `actor_kind='myli'` |
| Performance card | **yes — an attribution card** ("Ana's Clone", next to Myli on `/ai`) counting only the AI-state rows (Zach 2026-07-16, reversing the earlier no-card rule) | yes |
| Service seats | **never** (a card ≠ a seat — clones work their human's leads + inbox backup only) | can hold seats |

A clone acts **as** its human — a human's work never routes to Myli. Its card is a **tracking
surface, not an actor**: nothing sends or books "as the card", and the same person's manual work
(`actor_role='human'`) is never counted as the clone's.

## Task & appointment assignment (folded from the retired doc)

Leads, tasks, and appointments share **one grouped polymorphic "Assign to"** control — three sections
with hard dividers: **Agents** (humans holding Real Estate Agent) · **AI Agents** (Myli + each human's
clone) · **Transaction Coordinators**. The assignee is ONE `hl_users` id; the **lane is derived** from
`type` (+ whether the human has clone config): human → LEFT lane; clone → acts as its human, LEFT lane
+ mirrors into Next Actions; Myli → RIGHT lane + Next Actions; TC → the TC section.

**The AI-Clone task-level override:** a task assigned directly to a clone fires the clone **without**
the global AI toggle — two independent paths (global AI mode = everything eligible; a single
clone-assigned task = just that item). Only AI-capable items are eligible (**text · email ·
follow-up**); a physical showing can never be handed to AI. Clone-owned items render faint in Tasks &
Appointments and lit in Next Actions.
