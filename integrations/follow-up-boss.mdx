## The law: FAIL CLOSED

**No FUB signal creates a HouseLoop lead unless its trigger event's `Creates Lead` toggle is explicitly ON ‚Äî
and the `_global` master switch is ON.** A signal with no config row does NOTHING (logged skip). This is
enforced in code with fail-closed defaults (`fub-lead-import.ts`) after the 2026-07-09 incident where fail-open
defaults minted 422 unauthorized leads with every toggle off ‚Äî all purged, gate live-fire proven (111 backlog
imports vs the closed gate ‚Üí 0 leads).

Three belts keep it that way:
1. **Import-layer door hygiene** ‚Äî existing-person re-imports and behavioral/comms webhook types never fire
   native registration doors.
2. **Fail-closed config gate** ‚Äî no row / toggle off / `_global` off ‚Üí no lead, no workflow, receipt logged.
3. **Dispatch registration guard** ‚Äî a `*_registration` door can never route via a fallback tier
   (`runtime.ts`), so first-touch outreach to imports is structurally impossible.

## How a FUB signal flows

```
Zillow / Realtor / PPC ‚Üí Follow Up Boss
  ‚Üí native webhook ‚Üí fub.* mirror (raw truth, always captured)
  ‚Üí classification (classifyFubEvent ‚Üí the atomic trigger event)
  ‚Üí hl_fub_webhook_settings row = THE decision:
       Sync (ingest to history) ¬∑ Creates Lead ¬∑ Workflow ¬∑ Notify ‚Äî four independent toggles
  ‚Üí creates_lead ON ‚Üí hl_leads (platform='follow_up_boss') ‚Üí the FUB door fires ‚Üí hl_trigger_fire ledger
```

FUB-platform fires show on the **Follow Up Boss** section of [Trigger Events](/guides/trigger-events) ‚Äî never
on the native HouseLoop.ai board (ledger scoped by platform).

<Note>
**Behavioral signals honor Creates Lead too (Zach 2026-07-11).** A `property_saved` / `property_viewed` /
`in_app_message` from a not-yet-lead now **mints a new lead when its `creates_lead` is ON** ‚Äî a save fires the
real `saved_home_registration` door, a view/in-app the generic `direct_registration`. OFF ‚Üí lead-less notify.
No hardcoded never-mint list; the toggle governs per event. Detail:
[FUB Trigger Events ‚Üí Behavioral signals mint](/guides/trigger-events#behavioral-signals-mint-when-creates-lead-is-on).
</Note>

## What a FUB signal becomes in HouseLoop ‚Äî the 2026-07-10/11 hardening

Once captured, a lead's activity is mirrored into HouseLoop the right way ‚Äî governed by **one master**, **forward-only**,
with **real timestamps**. What shipped this pass (deep detail: [FUB Lead Import](/fub/master-scraper-ssot)):

- **ONE FUB master switch.** There were secretly **two** ‚Äî the UI toggle wrote `hl_settings.fub_sync_master` while
  the fail-closed notify/process gate read `hl_fub_webhook_settings._global.enabled`. They diverged, `_global` was
  orphaned OFF, and it **silently killed every FUB notification for hours**. Collapsed: `getFubSyncMaster` /
  `setFubSyncMaster` now read/write `_global.enabled` ‚Äî the flag the UI shows and the code obeys (notify + process +
  forward-sync). It can't orphan again.
- **The lead-detail feed is FORWARD-ONLY.** A pulled-in lead starts CLEAN ‚Äî pre-import texts/calls/emails stay
  context-only under the FUB tab; only new comms flow into the feed + Texts/Calls/Emails sub-tabs going forward
  (`backfillFubComms` watermark floored at the lead's `created_at`).
- **Real timestamps, not "22 hours ago."** Property view/save activity is stamped from `scraped_at` (the real
  trigger time, matches FUB to the minute) instead of a date-only midnight, and the feed renders the actual date +
  time (Central), not a relative "Xh ago."
- **ONE home-activity writer.** The dashboard's `parseZillowViewNote` ghost writer ‚Äî which hardcoded every Zillow
  home note as `activity_type='viewed'` (so **saved** homes recorded as **viewed**) ‚Äî was removed;
  `lib/fub/sync-lead-record.ts` is the sole, correct writer (derives saved-vs-viewed, stamps `fub_event_id`). 1,318
  legacy mislabeled rows were cleaned (565 dup deletes + 753 corrected ‚Üí `saved`).
- **Inquiries/tours flow like texts.** `fub.events` is now stored **unconditionally** in the webhook router, so a
  new inquiry/tour always lands and renders forward (the July-7 ingest ceiling is lifted).
- **Myli's facts are distilled, not dumped.** The extractor locks the slot vocabulary (`canonicalSlot`) so facts
  can't bloat, and raw house-view lists collapse into ONE synthesized signal ("viewed 32 homes in 2 days ‚Üí high
  urgency"). A bloated lead went **55 junk facts ‚Üí 13** meaningful ones, stable on re-run.
- **The import wizard is a filter + assign + tag builder.** Select who comes in by FUB **stage** and **tag** (a
  searchable modal over all ~1,276 live FUB tags, AND'd with stages, with a live match count); **assign** every
  imported lead to a person (or Myli-AI-only, never ownerless); **apply HouseLoop tags** (pick existing or create
  new) on top of the origin tag + the lead's own imported FUB tags. One-way, stage-mapped, Rule #0-gated on the full run.

## Lead-source canonicalization ‚Äî the six buckets

Follow Up Boss carries **hundreds of messy source strings** ("Zillow Preferred", "Premier Agent", "Trulia.com",
"Agent COI", "&lt;unspecified&gt;", "FiveStreet", "Opcity"‚Ä¶). Importing those verbatim would give HouseLoop an
un-filterable soup. So every FUB lead's source is **folded into exactly SIX canonical buckets** ‚Äî the normalized
list you import into and report on. Locked by Zach 2026-07-04: *"clean lead sources ‚Äî six buckets, everything
else is noise."*

| Canonical bucket | `hl_lead_sources` key | What folds in |
|---|---|---|
| **Zillow** | `fub_zillow` | Zillow, Zillow Preferred, Premier Agent, Zillow Flex, **Trulia** (Zillow-owned) |
| **Google PPC** | `fub_google_ppc` | anything carrying the `Google Adwords PPC` tag (tag wins, even over a Zillow source string) |
| **Google Organic** | `fub_google_organic` | `google` / `google organic` source text |
| **Realtor.com** | `fub_realtor` | `realtor.com` |
| **PC SOI** | `fub_pc_soi` | Agent COI, SOI, **Referral** (a referral is relationship-based), SRG past client |
| **Other** | `fub_other` | everything else ‚Äî FiveStreet, Opcity, Homes.com, HomeLight, unknown/new strings |

**The fold happens in ONE place** ‚Äî `lib/fub-import/source-canonicalize.ts` (`canonicalizeFubSource`), the single
map. Three rules, in order: ‚ëÝ the **`Google Adwords PPC` tag wins** ‚Üí Google PPC (it beats the source field);
‚ë° the **fold map** matches the source text ‚Üí its bucket; ‚ë¢ **default = Other** ‚Äî an unknown or brand-new string
NEVER invents a bucket, it lands in Other. Nothing is lost: the fine original origin is preserved on the lead as
a **`Lead Source - <original>` tag** (e.g. `Lead Source - Opcity`). The buckets are seeded by **migration 0315**;
the old hardcoded `Zillow.com` import label was killed so imports stamp the *real* resolved bucket.

### We do NOT rewrite FUB's Source field ‚Äî we tag (it's immutable)

**Follow Up Boss's `source` field cannot be changed via the API** (probed 2026-07-04 ‚Üí "No valid fields").
So inside FUB, a lead's Source stays exactly what it always was ‚Äî "Zillow Preferred", "Premier Agent",
"zillow.com" ‚Äî **forever**. We never touch it. The normalized bucket lives in FUB as a **tag** instead.

The one-time backfill ‚Äî a registered Inngest function (`fub-source-sweep`, event `fub/source-sweep.run`,
**manual, no cron**; Rule #0-gated, reversal file saved first) ‚Äî stamped the entire book:
- **`HL Source: <bucket>`** ‚Äî the canonical bucket (e.g. `HL Source: Zillow`), for filtering inside FUB.
- **`Lead Source - <origin>`** ‚Äî the exact original string preserved (e.g. `Lead Source - Zillow Preferred`).

Tag writes are GET ‚Üí merge ‚Üí PUT off the **live** FUB record (never the mirror), so a stale array can't delete a
tag a lead just earned. **Verified live (mirror, this session):** **48,920 of 49,234** people carry an
`HL Source:` tag (**28,072 = Zillow**); only **64** Zillow-family stragglers still need it.

<Warning>
**The `HL Source:` tag is the normalized signal in FUB ‚Äî the Source field is NOT.** Inside FUB, "Zillow
Preferred" (and the other raw strings) will always show as the Source; that's expected and unavoidable (FUB owns
that field). The `HL Source: Zillow` tag is where the bucket lives. Likewise, **do not read
`fub.people.source` in our mirror as the canonical bucket** ‚Äî the mirror re-syncs FUB's raw source over any fold,
so it also shows the raw strings. The canonical bucket is derived by the resolver / carried by the tag, never
stored durably in a source column.
</Warning>

### How HouseLoop gets the bucket (it derives, not reads the tag)

When a FUB lead **imports into HouseLoop**, the import runs the raw Source string through the fold map
(`canonicalizeFubSource`) ‚Üí resolves the bucket ‚Üí stores it on the HouseLoop lead (`hl_leads.source_id`) and
adds the `Lead Source - <origin>` tag. So the HouseLoop lead's Source = **Zillow**, even though FUB's field says
"Zillow Preferred". HouseLoop derives the bucket from the raw string ‚Äî it does **not** depend on the
`HL Source:` tag (the only tag it reads as an override is `Google Adwords PPC` ‚Üí Google PPC).

**Where you see it:** the **Platform / Source** page ‚Äî a platform card + the six-bucket KPI rows (click a bucket
to filter its leads); each imported lead shows its bucket as its Source, plus the `Lead Source - <origin>` chip
on lead detail. New inbound is canonicalized the same way at import time. Live bucket spread across 49,234
`fub.people` (by `HL Source:` tag): **Zillow ‚âà 28,072**, the rest split across Google PPC ¬∑ PC SOI ¬∑ Realtor.com ¬∑
Google Organic ¬∑ Other.

## Boundaries that never move

- **Deleting a HouseLoop lead NEVER touches Follow Up Boss** ‚Äî the delete path has zero FUB references
  ([Lead Deletion](/guides/lead-deletion)); FUB people stay intact no matter what's purged here.
- **The `fub.*` mirror is read-only truth** ‚Äî capture everything, act on nothing without a toggle.
- Notifications ship from the System line/address ‚Äî never a person's, never Myli's.
