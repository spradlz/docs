**In the app:** CRM sidebar → **Team Settings** → **My Team** — [`/settings/team`](https://crm.houseloop.ai/settings/team)

<Info>
**Rebuilt 2026-07-13 (C7, Zach-locked wireframe `my-team-v1-LOCKED.html`).** FUB-simple: there is no
"Users" concept — it's My Team, and everyone works here, humans AND AI. My Team is the **HR view**
(who they are, access, connections); the [AI section](/guides/ai-agents) is the **ops view** (what the
AI ones are doing). Same people, two lenses, zero duplicate config.
</Info>

## The roster — business cards in department bands

ONE blue header drives the page: **Members | Roles** tabs. The Members tab renders every non-clone member
as a **fixed business card** (the same `FixedTeamCard` the AI roster uses — 300px, 64px headshot + status
dot, four fixed slots: **Leads · Connections (the channel icons ✉📅📞💬, bright=connected) · AI Clone (a LIVE toggle on the card — same wire as the tab) · Access**, one button).
Cards group into **department bands** derived from the roles people wear (Real Estate Agents · Showing
Partners · AI Agents · …); the "Group by" control flips to a flat Everyone view. Human cards open the
member page; AI cards open the agent's own `/ai/agents/[id]` page. The Add Member ghost opens the one
create modal (`InviteMemberModal` — human or AI agent). Global AI policy (`AiFailoverPolicyCard`,
`AiModelsTable`) stays stacked below — policy, not roster.

## The member page — one flat template (the nesting kill)

`/settings/team/agents/[id]` — humans only (an AI agent's id redirects to its `/ai/agents/[id]` page:
one surface per actor). ONE flat row of tabs, deep-linkable with `?tab=`:

0. **Overview** — the PulseDeck (`/api/team/pulse?scope=member:<id>`): the member's own ten/eight-card KPI
   deck with the Today/7d/MTD/YTD picker. The page is ONE infinite scroll — the pill row at the top is pure
   JUMP navigation (nothing hides); every section below is always expanded.
1. **Profile** — name, email, phone, time zone (debounced auto-save). `/settings/profile` lands here.
2. **Permissions** — the Access select (owner/admin/member) with the can't-edit-your-own-access guard.
3. **Connections** — the star: **Email · Google Calendar · Phone · Text**, four flat sections
   (`MemberConnections`; Phone = the Twilio line + AI-voice line policy; Text = default channel +
   iMessage handle). "Configure email, text, phone" answered on one screen — for AI members too.
4. **Roles** — the member's human **job hats** (chips + one add-role picker). AI roles never appear on
   a human here.
5. **AI Clone** — see below.

The header carries the Active toggle and the **Actions** kebab (Emulate User · Delete Member — the
dependency-aware delete with the reassign flow, now shared via `MemberDeleteFlow`).

## The AI clone — built in, not a role

**Every human member has an AI clone built in** — off by default, starting from the shared default
persona. You never "add" a clone and it never appears in a role list. The member page's AI Clone tab
(`CloneSection`) is the entire surface: **one master toggle** (writes `hl_agent_clone_config.enabled` —
no user row is ever minted, the clone is a MODE on the human, mig `0392`; OFF = disarm only — nothing is
ever deleted), the **takeover minutes**, and **"Configure the clone →"** which opens the clone's agent
page (the human's id, in clone view).

<Note>
**One truth under the hood:** a clone's persona/capabilities live on its HUMAN's own `hl_member_role`
rows — the exact rows the runtime reads (`loadCloneContext`). The `/ai/agents/[id]` clone view edits
those same rows directly, and the roster card API sources clone cards from
`hl_agent_clone_config.enabled`, so what you see is always what runs. Every enabled clone also gets a
**performance card** on `/ai` next to Myli — attribution of the member's AI-state work
(`actor_kind='clone'`), never a separate identity.
</Note>

## Roles — job hats, one library

The **Roles tab** is the one role library (AI and human kinds in one table). Roles are JOB HATS — one
member can wear several (Myli: AI ISA today, Receptionist and Content Manager as those services ship).
"AI Clone" in the library is the built-in default template every member's clone starts from — edit it to
change the default; it is not assignable. Role shells (persona · capability gates · docs · model) edit at
`/settings/team/roles/[id]`; per-member overrides edit on the member/agent pages.
