<Note>Not built yet. This settings panel currently renders a placeholder ("coming soon") card, not a working forms manager. There is no `hl_forms` table, API route, or UI behind it today. This page documents what the surface is intended to be, so the docs nav matches the app 1:1.</Note>

**In the app:** CRM sidebar → **Lead Capture** → **Forms / Websites** — [`/settings/forms`](https://crm.houseloop.ai/settings/forms)

## What it does

Today, nothing functional. Visiting `/settings/forms` shows the standard CRM page shell (title, subtitle, and a "this surface is stubbed" card) via the shared `<ComingSoon>` renderer — the same placeholder every not-yet-built settings panel uses.

When built, this is intended to be the **Lead Capture** surface for managing lead-capture forms — the apply form, the openings form, and future custom forms — that feed leads into the CRM.

## What you configure

Nothing is configurable yet. There are no controls, inputs, or saved settings on this page in its current stubbed state.

## Data source

- **Route:** `/settings/forms` is served by the settings catch-all route `app/(crm)/settings/[section]/page.tsx`. There is no dedicated `app/(crm)/settings/forms/page.tsx` — `forms` is one entry in that route's `META` map, which drives the placeholder title/blurb.
- **Renderer:** `components/shell/ComingSoon.tsx` (wraps `PageShell` / `PageHeader` / `PageContent`).
- **Database:** none today. The stub's blurb references a future `hl_forms` table, but that table does not exist in the codebase yet.
- **API:** none — there is no API route backing this surface.

## How it fits

`/settings/forms` is a placeholder in the CRM's Lead Capture area, present so the settings navigation is complete. A real page added at `app/(crm)/settings/forms/page.tsx` would shadow the catch-all route and replace the stub when the feature is implemented. Until then, treat this as planned scope, not shipped functionality.
