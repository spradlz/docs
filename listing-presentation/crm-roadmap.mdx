<Note>
**The goal, in Zach's words.** In the HouseLoop CRM, a **Listing Presentations** section sits alongside
the Seller Report. You type in an address, hit enter, and it runs through and outputs the pages per our
template. A dialogue box lets you talk to the AI — "re-look at this price," "add a column here" — and edit
in real time. This page is the plan to get there.
</Note>

## Where it lives in the CRM

A new left-nav section, **Listing Presentations**, parallel to the existing **Seller Report** under the
seller/CRM surfaces. One row per listing you're preparing; open one to get the address bar, the rendered
pages, and the chat box.

## The three build phases

<Steps>
<Step title="Phase 1 — the skill (done)">
Talk to Benji, he runs the deterministic engine, you iterate in chat. This is live today and is what the
other two pages in this section document. It proves the template and the build engine.
</Step>

<Step title="Phase 2 — the build engine as a service">
Port `build-lp-v3.py` + `rebrand-cma.py` into a server-side module the CRM can call. It already takes
structured input (subject facts + CMA comp data) and returns HTML/PDF — so this is a lift-and-shape, not
a rewrite. Output stored per listing; re-runnable on every edit.
</Step>

<Step title="Phase 3 — the MLS data service + the UI">
Automate the one manual half (the MLS pull) as a background job, and build the CRM section: address bar,
live page preview, and the chat box wired to Myli for real-time edits.
</Step>
</Steps>

## Architecture recommendation

**It does not need to be a standalone MCP server.** It's a CRM feature backed by our normal stack. The
honest shape:

| Piece | What it is | Notes |
| --- | --- | --- |
| **MLS data job** | A registered **Inngest** function that logs into Heartland MLS Matrix with our stored creds and pulls subdivision-anchored comps + photos | The hard part. Auth + anti-bot risk; must use a real browser session, obey MLS terms, and **fail loudly** (never fabricate) — per the CMA data-pull skill. "Inngest or it doesn't exist." |
| **Build module** | The two Python scripts, ported to a server function that takes `{subject, comps}` and returns HTML + PDF | Deterministic; already parameter-shaped. The reproducible core. |
| **CRM section** | A `Listing Presentations` left-nav surface: address bar, page preview, version history | Built from the existing CRM component set (no new lookalikes). |
| **The chat box** | Myli (our AI) with a small set of tools that re-run the build with edits and re-render | "Change the price," "add a column" become tool calls that re-run the module — the same edits Benji makes today, in the UI. |

**Why not an MCP server:** MCP exposes tools to an external AI client. Here the AI (Myli) already lives
inside our stack, so the cleaner design is CRM UI + Inngest job + build module + Myli tools. An MCP layer
would be extra surface with no payoff.

## Honest dependencies and risks

- **The MLS pull is the gating risk.** It's auth-gated, browser-based, and subject to MLS terms and
  anti-bot defenses. Automating it reliably is the real project; until it's solid, the pull stays
  semi-manual (Phase 1/2 with a human-authenticated session).
- **Pricing stays a judgment step.** The engine can lay out the comps and compute the deltas, but the
  recommended number and the narrative are a comparative read. The CRM version should *surface* the
  analysis and let a human confirm it — not silently auto-price.
- **Data honesty is a hard rule, not a nicety.** No fabricated photos, no padded galleries, no number the
  data doesn't support. The Lawrence Gunther build caught all three (a faked 19-photo gallery, a price
  that contradicted a pending comp); the CRM version has to enforce the same, structurally.

## Definition of done (Phase 3)

Zach opens **Listing Presentations**, types an address, hits enter, and within a few minutes has a
letter-size PDF matching this template — comps, difference columns, hero photos, the price graphic — with
a chat box he can use to take a second pass at the price or add a column, and it re-renders live.
