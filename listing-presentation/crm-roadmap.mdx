**The goal, in Zach's words.** A new **Listings** left-nav item with a **CMA Tool**. It works like creating a TC
loop: you type an **address**, it makes a **card**, and the card runs the whole process end to end. A big **chat

## The end-state template — the GT canon (LOCKED)

Every CMA renders into the exact book proven on the Kenney Hunt / 9928 W 52nd St build (the **gold-star reference**):

**Cover → By the Numbers → Listing Presentation → CMA cover → Closest Comparables (section) → Additional Area
Comparables (section) → per-comp agent analysis (lead sentence + 4–5 bullets + difference column + all photos) →
"The Bottom Line" conclusion page (read + As-Is / After-Work range boxes) → Suggested Price Range (white write-in
box) → Thank You.**

The reference PDF + all source data/photos are locked at
`.claude/reports/listing-presentation-work/merriam-9928-w-52nd/` and `.../deliverables/`. The build engine + the
step-by-step recipe live in [The Reproducible Build Engine](/listing-presentation/build-engine) and
`viktor-okoth/BUILD-HANDOFF.md`.

## The autonomous pipeline (address in → book out)

1. **Matrix (Camp David persistent Chrome):** log in, search the subject, **select the closest comparables**.
2. **Cloud CMA handoff:** Select All → click the Matrix **Cloud CMA** button → the comps push into a Cloud CMA report.
3. **Export** the non-branded Cloud CMA PDF.
4. **Extract:** `pdfimages` (all photos) + `pdftotext` (comp data) → `data.json` + `photos.json`.
5. **Rebuild + rebrand** into the HouseLoop template with `build-cma.py` (2-section, per-comp analysis, conclusion).
6. **Assemble** deck + By-the-Numbers + CMA in the locked order → the final PDF + live HTML.

Steps 3–6 are **proven twice** (Viktor + Merriam) and fully generalized. Step 1–2 — the **Matrix comp-selection +
Cloud CMA handoff** — is the hard, fragile piece that needs to be **locked and mapped** on Camp David.

## Architecture (per the constitutional rules)

| Piece | What it is | Rule it honors |
| --- | --- | --- |
| **MLS data job** | Inngest event → Camp David CDP consumer drives Matrix + Cloud CMA on the persistent Chrome, exports the PDF, hands off to the build module | Inngest-or-it-doesn't-exist · no-headless-browsers · locked/mapped steps |
| **Build module** | `build-cma.py` + deck/track-record builders, server-callable, env-driven, renders the gold-star template | Reproducible, data-driven |
| **CMA Tool UI** | `Listings` → `CMA Tool`: address → card → live HTML preview → edit, built from the existing CRM component set + the TC-loop create pattern | Reuse-not-recreate · UI doctrine |
| **The chat bar** | Myli with the pipeline tools — re-run, re-extract, re-price, edit a comp's analysis, correct in real time | The AI already lives in our stack (no MCP server) |

## Honest dependencies & risks
- **Phase 1 is the gate.** The Matrix comp-selection + Cloud CMA handoff is auth-gated, anti-bot, browser-based.
  Until it's rock-solid it stays semi-manual (agent selects comps + exports; the tool takes it from the export).
- **Pricing stays judgment.** The recommended list is left blank (white write-in box) for the agent; the tool
  surfaces the range and the read, it does not auto-price.
- **Data honesty is structural.** Real photos only, the seller name comes from the call sheet (never the Cloud CMA),
  thin markets are labeled thin, additional comps are context not equals.
