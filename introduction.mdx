## Overview

HouseLoop CRM is built API-first. Every action you can take in the UI has a corresponding REST endpoint. This means Myli (and any integration) can do everything a human operator can.

**Base URL**
```
http://localhost:3020/api       (local dev)
https://crm.houseloop.ai/api    (production)
```

**Authentication:** All endpoints require a valid session cookie (from `/api/auth/login`) or a service API key in the `Authorization: Bearer <key>` header.

## Quick Start

<CodeGroup>
```bash Create a Lead
curl -X POST http://localhost:3020/api/leads \
  -H "Content-Type: application/json" \
  -d '{"first_name":"Jane","last_name":"Smith","email":"jane@example.com","phone":"5555550100","source":"Website"}'
```

```bash List Leads
curl http://localhost:3020/api/leads
```

```bash Update a Stage
curl -X PATCH http://localhost:3020/api/leads/{id} \
  -H "Content-Type: application/json" \
  -d '{"stage_id":"<stage-uuid>"}'
```
</CodeGroup>

## Design Principles

- **REST + JSON** — all requests/responses are JSON
- **UUIDs** — all IDs are UUID v4
- **Timestamps** — ISO 8601 (`2026-05-16T23:45:00Z`)
- **Soft deletes** — leads are never hard-deleted from the DB; they get `deleted_at`
- **Pagination** — list endpoints support `?limit=` and `?offset=` (default limit: 500)
- **Error format** — `{ ok: false, error: "reason" }` with appropriate HTTP status
