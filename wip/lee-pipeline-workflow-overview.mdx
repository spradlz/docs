##### 1. THE PROBLEM
HouseLoop is the website AND the CRM in one product ‚Äî and Follow Up Boss is a legacy CRM piping in leads with
history. "Lead source" alone can't express that. We spent a day circling because two things were tangled:
**where a lead came from** vs **what it does on our site**. This PRD untangles it.

##### 2. WHAT THE RESEARCH SAID (5 CRMs ‚Äî HubSpot, Salesforce, UTM/GA4, FUB/kvCORE/Sierra, Pipedrive/GHL/AC)
Unanimous findings (full briefs in session history):
- **Two fields, never one:** a TRUE ORIGIN (Zillow) kept separate from the CONNECTOR (Follow Up Boss). Origin
  is first-touch + immutable. (HubSpot Original vs Record Source; Salesforce LeadSource vs Source System.)
- **Events ‚âÝ source.** Activity is a separate growing stream; source is a stamp set once. (We already do this:
  `hl_lead_activity`.)
- **Trigger on the EVENT, filter on SOURCE/PLATFORM.** Never one workflow per source. Merge fields fill the
  words. (Every CRM. Sierra/GHL/AC explicit.)
- **No "type" field** for origin ‚Äî "type" = the event (already our catalog) or the pipeline role (already our
  Buyer/Seller pipelines). Don't add it.
- **Don't prefix `fub-zillow.com`** ‚Äî keep source = the real origin (Zillow), connector separate (live FUB data
  with 48k people confirmed this). Platform = the connector/world.

##### 3. THE MODEL (LOCKED)

###### 3.1 The keystone ‚Äî TWO buckets, TWO routing keys
| Bucket | Keyed on | Meaning |
|---|---|---|
| **New Lead Registration** | the lead's **ORIGIN** (source + platform) | "how they came in." realtor.com/Zillow via Follow Up Boss ‚Üí that source's registration (the reactivation). Origin is **set once, permanent** ‚Äî a merge field forever, but stops STEERING after registration. |
| **(HouseLoop) System Events** | the **SURFACE** (where the action fired) | "what they do on HouseLoop." ANY lead ‚Äî regardless of origin ‚Äî who likes a home / requests a tour on HouseLoop.ai fires the **HouseLoop** workflow. |
| **Follow-Up** | schedule | the nurtures |

a home **on HouseLoop.ai** ‚Üí SURFACE = HouseLoop ‚Üí runs the **HouseLoop liked-home workflow**, not the FUB one.

###### 3.2 The cascade (first match wins)
```

  1. SPECIFIC workflow for (this source + this event)?    ‚Üí run it
```

Already engine-true: `dispatchDoor` routes existing-lead events by event SURFACE and keeps registration

One event ‚Üí one workflow ‚Üí `{{first_name}}` `{{home_address}}` `{{lead_source}}` `{{platform}}` fill the rest.
Never a workflow per source. Adding a source = a new ROW, not a rebuild.
