### Background & model (rolled in ‚Äî the original reconciliation)

**North star (Zach-signed 2026-07-01):** ONE canonical DB table `hl_trigger_events` is the single source of truth for
every trigger. Everything derives from it (Lead Sources page, Trigger Events page, workflow builder palette, Workflows
list) ‚Äî no second list. A "Trigger Event" = a trigger (starts a workflow) that fires on an event a lead did.

**Why:** the same trigger info was duplicated across 4 code lists (`WF_TRIGGERS`, `CANONICAL_LEAD_EVENTS`,
`EVENT_TITLES`, `EVENT_TRIGGER`) + 3 DB tables, hand-synced ‚Äî so a trigger added under Lead Sources never flowed into
the builder. Full map: `.claude/checkpoints/workflow-trigger-catalog.md`.

**Attribution (PRD ¬ß3):** every lead = **Platform** (the system it arrived through ‚Äî Follow Up Boss / houseloop.ai / any)
‚Ä∫ **Lead Source** (the finer origin within the platform ‚Äî Zillow Preferred / houseloop.ai Organic) ‚Ä∫ **Tags** (page /
campaign). Shown on lead detail; grouped under platform on the scoreboard. `platform=any` is a first-class scope.

**Config (PRD ¬ß9):** two axes ‚Äî Platform = the tab (`houseloop`/`fub`/`any`); Category = the group
(registration/system/follow-up). Per-platform config = two toggles (Creates-Lead, Workflow-Active) + FUB's Process.
Workflows bind to **(trigger √ó platform-scope)**; `any` = agnostic. Notification = read-only `notify_class` rolling up
to the Notifications section. **The event always fires ‚Äî toggles gate the lead + the workflow, never the firing.**
