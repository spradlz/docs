### THE CORE INSIGHT — MLS# is the ground-truth anchor
You can't *search* Zillow by MLS#, but Zillow's **property-detail carries it** (`attributionInfo.mlsId`). So:
resolve the address → get the Zillow `zpid` → **validate `mlsId` against our known MLS#(s)** for the house =
100% certainty it's the right rooftop (not "probably"). MLS# is unambiguous where addresses garble.

### THE RESOLVER (per house, runs on all ~1,936)
1. **CASS/USPS clean** the stored address (Google Address Validation, HouseLoop key `GOOGLE_GEO_SERVER_KEY`,
   project houseloop-496200) — fixes wrong zip + garble. **99.9% validate** (1,935/1,936).
2. **Find the Zillow record** — OpenWeb Ninja search, multi-form: CASS address → street-type variants
   (St/Ter/Dr/Ct/Ln) → range-preserved (`10700-10702`) → original. → candidate `zpid`.
3. **LOCK via MLS#** — `property-details(zpid).attributionInfo.mlsId` must match a known MLS# for the house →
   CONFIRMED canonical. No match after all forms → FLAGGED for review (never silently wrong).
4. Output per house: `HLID · website_hlid · canonical address · slug · /home URL · zpid · zestimate · status`.

### VALUE — every house gets one, correct tier
- **sold + Zestimate** → Zestimate × 1.005 (`ninja_monthly` bucket)
- **active / pending / coming-soon** → list price × 1.005 (`active` — flips to Zestimate on close)
- **sold, no Zestimate** → last sold price × **ZHVI area appreciation** (`zhvi_monthly` bucket) — proven:
  1219 Helen Ct → $197,152 (Redfin $195–235k)
- + a monthly value-history line for all. Two monthly Inngest cohorts: Ninja (Zestimate homes) + ZHVI (no-Zestimate).
