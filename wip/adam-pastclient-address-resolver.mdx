### THE CORE INSIGHT ‚Äî MLS# is the ground-truth anchor
You can't *search* Zillow by MLS#, but Zillow's **property-detail carries it** (`attributionInfo.mlsId`). So:
resolve the address ‚Üí get the Zillow `zpid` ‚Üí **validate `mlsId` against our known MLS#(s)** for the house =
100% certainty it's the right rooftop (not "probably"). MLS# is unambiguous where addresses garble.

### THE RESOLVER (per house, runs on all ~1,936)
1. **CASS/USPS clean** the stored address (Google Address Validation, HouseLoop key `GOOGLE_GEO_SERVER_KEY`,
   project houseloop-496200) ‚Äî fixes wrong zip + garble. **99.9% validate** (1,935/1,936).
2. **Find the Zillow record** ‚Äî OpenWeb Ninja search, multi-form: CASS address ‚Üí street-type variants
   (St/Ter/Dr/Ct/Ln) ‚Üí range-preserved (`10700-10702`) ‚Üí original. ‚Üí candidate `zpid`.
3. **LOCK via MLS#** ‚Äî `property-details(zpid).attributionInfo.mlsId` must match a known MLS# for the house ‚Üí
   CONFIRMED canonical. No match after all forms ‚Üí FLAGGED for review (never silently wrong).
4. Output per house: `HLID ¬∑ website_hlid ¬∑ canonical address ¬∑ slug ¬∑ /home URL ¬∑ zpid ¬∑ zestimate ¬∑ status`.
