## 4 · Notifications (the delivery layer)

Notifications ride **under** the spine — a door fires, the event routes to its notification class, and delivery is
governed in **exactly one place** (`/settings/notifications`), source- and integration-agnostic, owner-only. Deep
catalog: [Lee Messages & Notifications Catalog](/platform/message-catalog).

### 4a · The One-Card System — ONE shell, payload-driven (LOCKED 2026-07-06)

Agent notifications are ONE shell + a payload catalog. **`hl_notify_classes`** is the payload matrix (**40 rows**:
`heading_tpl` / `subline_tpl` / `with_message`) → two locked shells `notify_card` / `notify_card_sms`. Editing a
heading edits one row. 66 legacy `notify_*` rows quarantined (reversible). Migrations 0320 / 0321.

### 4b · The Blocks Engine — templates own every pixel (LOCKED 2026-07-05)

Templates own EVERY pixel of markup; code supplies ONLY data. Repeating structures are BLOCK rows
(`{{block:listing_card list=items as=listing limit=12}}` + `{{#ns.field}}…{{/ns.field}}`). `match-cards.ts` is
data-only. Migration `0319_blocks_engine`.

### 4c · Three families that must NOT collapse

**1. Agent notifications** collapse into the One-Card System. **2. Lead-facing intent templates** (door welcomes,
saved-search digests, tour/contact confirms, Myli `{{ai.*}}`) stay individual — one per intent, by design.
**3. Shared blocks** are the reused markup both embed. One engine, three shapes, never merged.

### 4d · Delivery — owner-only, provenance in the copy

- **Owner-only fan-out (LOCKED 2026-07-07)** — a lead notification goes to the lead's **owner only**; never
  collaborators, never AI agents. (Killed the Msale flood: 28 pings/6 people → 1.) `lib/notifications/create.ts`.
- **Provenance (LOCKED 2026-07-07)** — every card names **source** + **integration**: *"…requested a tour — 123
  Main St · from Zillow via Follow Up Boss."* From `hl_leads.source` + `hl_leads.platform` → `{{notify.source}}` /
  `{{notify.integration}}`, auto-appended to every card subline in one place. `lib/notifications/dispatch.ts`.
- **No rogue notifiers (LOCKED 2026-07-07)** — the tour/contact agent SMS that fired on `if(agent?.phone)` (ignoring
  the toggle + owner) is dead; routes through pref-gated `notifyLeadEvent`. Codebase swept clean.
- **Two-beat standard** — Beat 1 branded SYSTEM email instant; Beat 2 Myli ~2 min later (email + text). Every text
  is Myli (ONE NUMBER). **FUB self-heal** for call/email detail. **Voicemail + forwarding** under Settings → Phone.
