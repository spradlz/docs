<Note>
**This is a build-out / system reference, not a CRM screen.** It answers "what does a lead's activity
**look like**, what **code** renders it, and where does it **come from**" — the counterpart to
[Pipeline & Workflow Master Overview](/guides/pipeline-workflow-overview), which answers "what **happens**
to a lead." Cards render through `components/leads/detail/ActivityHeroCard.tsx` (hero) + the `FeedRow`
compact body, tokens from `lib/leads/card-tokens.ts`. Each card is worked **design → verify live → lock →
ship**; a row is `🔒 locked` only once it's verified on a real lead and the code is committed.
</Note>

## The two shapes

Every activity renders as one of two cards, decided by **category** (see
[Trigger Events](/crm/pipeline-workflow-system-events)):

- **Hero card** — a **Registration** event (the lead's first action *creates* the lead: New Lead — Liked
  Home / Saved Home / Tour Request / …). Big photo + headline + address + facts + message.
- **Compact card** — a **System** event (an *existing* lead does the same thing again: Property Viewed /
  Liked / Saved / …). A tight icon-tile row.

The *same* action renders both ways: a tour that creates a lead is a hero card; an existing lead's tour is
a compact row.

## The 3 source variants (every card)

A card's **picture tile + "via" badge + link row** are driven by `lib/leads/card-source.ts` from the
lead's source — the only thing that changes across variants:

**Link order is a template rule (never hand-set): the HouseLoop link is ALWAYS on the LEFT, the
lead-source/portal link ALWAYS on the RIGHT** (`HomeLinks.tsx`).

| Variant | Picture tile | "via" badge | Links (HouseLoop left · source right) |
|---|---|---|---|
| **HouseLoop.ai** (native) | the real house photo | `via HouseLoop.ai` | View on HouseLoop │ View on Zillow |
| **Follow Up Boss · Zillow** | the **Zillow logo** (`/portals/zillow.png`) | `via Zillow.com` | Look up on HouseLoop │ View on Zillow |
| **Follow Up Boss · other portal** | the **source name as text** ("Realtor.com") | `via Realtor.com` | Look up on HouseLoop │ View on Realtor.com |

### The code

The source drives exactly 3 lines (`source`, `picture`, `homeLinks`) — everything else is shared:

```tsx
<ActivityHeroCard
  accentColor="#059669"                        // tour_request badge color
  actor={{ label: 'Lead', color: 'var(--hl-blue)' }}
  title="Requested a Tour"
  pill={{ label: 'Tour Request', bg: '#0596691A', fg: '#059669' }}
  timeStr="Jun 26, 6:12 PM"
  calendarChip="Jul 10, 6:30 pm"               // dynamic: from event_payload.(payload.)requestedDate/Time
  address="14019 W 149th St, Olathe, KS 66062"
  subline="$525,000 · 4 bd · 3 ba · 2,920 sqft"
  quote="Hi — is this home still available? ..." // event_payload.message

  // ↓↓ the only 3 lines the SOURCE changes (cardSource() derives them from lead.source) ↓↓
  source={{ label: sourceLabel }}              // 'HouseLoop.ai' | 'Zillow.com' | 'Realtor.com'
  picture={cardPicture({ native, source, photo })}
    // native          → { kind:'photo', src: home.photo }      → real house photo
    // source ~ zillow → { kind:'zillow' }                      → /portals/zillow.png
    // other portal    → { kind:'text', text: portalLabel(s) }  → "Realtor.com"
  homeLinks={native
    ? { houseLoop: '/home/…', portal: 'https://zillow.com/…' }  // View on HouseLoop │ View on Zillow
    : { portal: portalUrl, lookupUrl: '/home/…' }}              // View on <portal> │ Look up on HouseLoop
/>
```

### What fires it & where the data comes from

| | |
|---|---|
| **Trigger event** | `tour_request` (website) · `fub_tour_request` (FUB import) · `virtually_toured` shares this card |
| **HouseLoop.ai** | the site's Request-a-Tour form → `/api/pixel/track` → `hl_lead_activity`. `native = true`, real photo, links HouseLoop │ Zillow. |
| **Follow Up Boss** | FUB webhook (tour intent) → the Camp David scraper backfills the lead's house history into `hl_lead_activity` at import. `native = false`; the **lead source** rides along (Zillow / Realtor.com) and paints the tile + the "via" badge + the portal link. |
| **Source → picture** | Zillow → the Zillow logo tile · other portal → the source name as text ("Realtor.com") · native → the home photo (or the green Home icon if none). |
| **Renders via** | `components/leads/detail/ActivityHeroCard.tsx`; picture/link logic in `lib/leads/card-source.ts` (`cardPicture` / `portalLabel`); tokens in `lib/leads/card-tokens.ts`. |
| **Data fields** | `property_payload`: street/city/state/zip · price · beds · baths · sqft · mls · photo · hlid/slug · url. `event_payload`: requestedDate · requestedTime · message. `lead.source` drives the variant. |
