<Note>Part of [Messages & Notifications](/platform/message-catalog). **This is DATA injected INTO a template —
not a template.** Screen: **Settings → AI Context Blocks**.</Note>

## What it is

A `{{ai.<key>}}` token in an email or text template marks a section that **Myli composes per-lead at send time**,
grounded in the lead's real behavior (the home they liked, their saved search) — the personal, non-canned part
of the message. Each block is a real `hl_templates` row (kind `context_block`) carrying:

- an **LLM compose prompt** (how Myli writes it), and
- a **static fallback** (fail-open: if the compose fails, the fallback ships so a message never goes blank).

This is the accepted AI-voice layer — real catalog rows, never inline backend copy. The deterministic tail
(sign-off, disarmer) stays in fixed template copy; only the `{{ai.*}}` slice is composed.

## How it renders

`renderAiContextBlocks(leadId, body, …)` resolves every `{{ai.*}}` in a template body into composed text, then
`applyMerge` drops it into the message. Same one-brain mechanism for email and text (text flattens any HTML the
block emits to plain SMS). Fail-open to the block's fallback.
