<Warning>Part of [Messages & Notifications](/platform/message-catalog). **Audit finding: there is NO unified
in-app display-message system yet.** This page documents the current ad-hoc state + flags it as a gap so it's
on the map, not lost.</Warning>

## What it is

The messages HouseLoop shows *on screen* (as opposed to email/text it sends out): success/error toasts, page
banners, empty-states, inline form confirmations. Today these are built per-component — there is no shared
Toast / Banner / EmptyState primitive and no toast library.

## Current state (ad-hoc)

| Pattern | Today | Status |
|---|---|---|
| Banners | one-off `components/leads/detail/DuplicateLeadsBanner.tsx`; no shared `Banner` | ⚠️ ad-hoc |
| Modals | `components/ui/IpadModal.tsx` · `HlFormModal.tsx` (shared shells — good) | ✅ shared |
| Toasts / snackbars | none — no toast library (react-hot-toast / sonner absent) | ❌ missing |
| Empty-states | inline "no … yet" per page (TriggerEventsBoard, StagesPage, …); no shared `EmptyState` | ⚠️ ad-hoc |
| Inline confirm / error | scattered inline strings per form | ⚠️ ad-hoc |

## The gap / opportunity

A unified display-message system — shared, tokenized `Banner` + `Toast` + `EmptyState` components — would make
on-screen messaging catalog-driven the way email/text templates already are (one place, consistent brand, no
per-component drift). **Not built yet.** Captured here so it's a tracked item, not a surprise.
