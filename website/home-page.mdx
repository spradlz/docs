## The one idea: shelves are DATA, not code

The home page is a stack of **shelves** — each a row of listing cards ("Homes in Overland Park", "Homes on
land"). Every shelf is a **row in `hl_home_shelf`** (web DB), not a component: its query (`queryEnvelope`),
its display knobs (`displayOptions`), its order (`sortOrder`), and its on/off (`isActive`) all live in the
row. **Adding a shelf = INSERT a row** (edit `prisma/seed/home-shelves.ts`, `npm run db:seed`) — you never
edit `HomeRows.tsx`.

Current live set (13 rows, all active): `hero` · `all-kc-metro` · 10 city shelves (Overland Park, Leawood,
Olathe, Lenexa, Shawnee, Kansas City, Lee's Summit, Liberty, Independence, Blue Springs) · `acreage`.
(The old `featured` shelf is retired.)

## Composition pipeline

| Piece | File | What it does |
|---|---|---|
| Shelf composer | `lib/home/composer.ts` | Reads active `hl_home_shelf` rows (sorted), runs each `queryEnvelope` through the ONE search engine (`searchListings`), slices to the card count. Newest-first, no shuffle (Zach 2026-05-21: "the N newest — that's how they filter in and out"). |
| Card count | `displayOptions.maxCards` → `queryEnvelope.limit` → 15 | `maxCards` is honored as the per-shelf cap (wired 2026-07-06 — it was decorative before). |
| Hero composer | `lib/home/hero-composer.ts` | Its own path because the hero has 3 extra concerns: curated `FeaturedListing` admin overrides lead the rotation · cookie-aware rotation (`hl_seen_hero` — returning visitors see fresh slides) · a stricter photo bar (`photos.min` ≥ 25 in the hero row's envelope). |
| Hero slide count | `displayOptions.slideCount` on the `hero` row | DB-first (wired 2026-07-06): explicit caller override (mobile passes 1) > the hero row's `slideCount` > 5. No more hardcoded constant. |
| Inspiration links | `components/communities/InspirationLinks.tsx` ← `hl_inspiration_city` (24 rows) + `hl_inspiration_category` (5 rows), seed `prisma/seed/inspiration.ts` | City + category tabs. Category hrefs MUST use real `url-to-query` params — an unknown param silently drops and the tab renders the whole grid (fixed 2026-07-06: `minYearBuilt`, `oh=1`, `sort=viewsDesc`). |
| Market band | `components/market/LiveHousingDataSection.tsx` ← `MarketSnapshot` | County market snapshots. |
| Header / Footer | `components/site/Header.tsx` / `Footer.tsx` | 3-region desktop header + mobile variants; footer carries the MLS GRID / Heartland IDX disclosure + DMCA. |

## Global constants — `hl_app_config`

Product constants a non-engineer might change live in `hl_app_config` via `lib/config/config-service.ts`
(typed accessors, 5-min cache). **Every key has a real consumer** (dead keys purged 2026-07-06):

| Key | Consumer |
|---|---|
| `lot.sqft_per_acre` / `lot.max_sane_acres` | acreage math + outlier defense |
| `cache.active_ttl_seconds` (6h) | `getListing` freshness for active listings |
| `cache.pending_ttl_seconds` (1h) | per-status TTL — pending/under-contract refreshes faster (about to close) |
| `sync.*` | runtime watermarks (not user-facing) |

## Drift defense (why this stays clean)

- ESLint `no-magic-numbers` + `no-restricted-imports` (provider wire types die in `lib/listings/providers/`).
- `npm run drift-check` full-tree grep audit + `tools/houseloop-prisma-check.sh` — both run on pre-commit for HouseLoop files.
- The 5-question decision tree in `apps/houseloop-web/AGENTS.md` — read it before adding anything here.

## Known absent (flagged, not built)

- **Geo personalization** — every visitor sees the identical shelf order; `sortOrder` was built to support
  per-visitor ordering when we want it.
