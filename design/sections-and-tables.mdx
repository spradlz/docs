title: "Design Standards — Sections, Tables, Menus & KPIs"
description: "The four block components that carry a body: SettingSection (rows), SettingsTable (a table), MenuTable (a table with a menu), and StatStrip (KPIs). Plus NavCard for launcher pages. Each shares the one brand-blue header."
---

All four block components render the **same brand-blue header** (via `SectionBand`). The only thing that changes is the body.

## SettingSection — rows

`components/settings/SettingsSection.tsx`. A brand-blue header (icon + title + description + one optional action) welded to a body of rows.

- Sub-divide with `SettingGroup` — an uppercase eyebrow label + a hairline divider. **Never a nested card, never a colored band.**
- Each setting is a `SettingRow` — label left, control right, hairline divider between.
- Use it for: Notifications channels, Profile, Permissions, Connections, any "label → toggle / value / button" surface.

## SettingsTable — a table

`components/settings/SettingsSection.tsx`. The same brand-blue header welded to a full-width `DataTable`. The header owns the rounded top; the table owns the rounded bottom; they read as **one connected unit**.

- **The count is derived** — `count={rows.length}`. A "Showing 2 of 2" that disagrees with the table is structurally impossible because the number and the table read the same array.
- **Grouping is a TYPE column**, never a nested band. "AI" vs "Agent Team" is a type cell, not a sub-header.
- The table is a `DataTable flushTop hideToolbar`. Use it for: Members, Roles, dialer numbers, any row-list.

## MenuTable — a table with a menu

`components/ui/MenuTable.tsx`. One brand-blue header holds the title + count + a right slot + the menu tabs + an optional filter sub-row — all white-on-blue, wrapping — welded to the table below. The header owns the rounded top, the table the rounded bottom: **one unit that opens from the header.**

- A menu is **always** this. Never a navy band, never a floating tab strip, never floating pills.
- The active tab is a **soft** translucent-white pill (`rgba(255,255,255,0.18)`), never bright solid white.
- Filters and a date range live **in the header**, never floating above the table.
- Use it for: Run Log, Leads, Tasks, Workflows, Calendar, AI Cost — any table driven by tabs / filters.

## StatStrip — KPIs

`components/ui/StatStrip.tsx`. A row of KPI cards, rendered **inside** a blue-header section — never floating on the page.

- **One normalized color scheme.** Default: brand-blue accent (value + icon in `--hl-blue`, a plain border). The only allowed variation is a small **locked** semantic set — `neutral` (blue), `positive` (green, only when the metric is genuinely good/complete), `attention` (amber) — chosen by what the metric **is**, never as decoration.
- **Never a per-card rainbow.** A different accent per card is the bug this component exists to kill.

## NavCard — launcher pages only

`components/ui/NavCard.tsx`. The **one** sanctioned floating tile. A `NavCardGrid` of `NavCard`s (icon + title + sub + optional status).

- Use it **only** for navigation launchers — home pages, the integrations marketplace. **Never for config** — a settings surface is a Section or a Table, never a grid of cards.

## The consolidation lens

Before adding a block, ask: *can this fold into an existing one?* Default to fewer, fuller blocks. If three surfaces describe the same thing, they should be one. A new block type is justified only when it maps to a distinct body type — never as an overflow. A **sixth** body type is a Zach-approved event, never invented to "fill a gap."
