---
title: "Design Standards — Buttons, Pills & Toggles"
description: "The button, pill, tag, and toggle vocabulary. Six button variants, a fixed set of status and type pills, one toggle — all tokenized utility classes. No raw button styles, no hand-rolled pills, no per-item rainbow."
---

Controls are the part of the CRM that drifted the most: 134 raw `<button style>` bypassing the button class, ~20 files re-implementing the filter pill, and 9 hand-rolling the toggle. The fix is the same as everywhere — **reuse the class, never hand-roll.**

## Buttons — `.hl-btn`

There is **one** button, with six variants and two sizes. **Never write `<button style={{...}}>`.**

| Variant | Class | Looks like |
|---|---|---|
| Primary | `.hl-btn .hl-btn--primary` | Blue fill, white text — the main action |
| Secondary | `.hl-btn .hl-btn--secondary` | White with a border |
| Ghost | `.hl-btn .hl-btn--ghost` | Text only, no fill |
| Danger | `.hl-btn .hl-btn--danger` | White with a red border |
| On-navy | `.hl-btn .hl-btn--on-navy` | Translucent-white — for the navy `PageHeader` |
| Icon | `.hl-btn .hl-btn--icon` | 36 × 36 square icon button |

Sizes: add `.hl-btn--sm` or `.hl-btn--lg`.

## Pills

Each pill has a canonical home. Do not hand-roll one, and **never give each item a different color** (a per-item rainbow is banned).

| Pill | Class / component | It is |
|---|---|---|
| Status pill | `.hl-status-pill--active/--paused/--inactive/--draft/--new` | A dot-prefixed status pill |
| Generic pill | `.hl-pill--success/--warn/--danger/--neutral/--info` | A semantic pill |
| Type tag | `.hl-type-tag--outbound/--receipt/--standby` | An uppercase dot-prefixed type tag |
| Status dot | `.hl-status-dot--active/--warn/--danger` | A bare dot |
| Tag pill | `TagPill` | The one blue tag pill (blue-tint fill, blue text) |
| Merge chip | `.hl-merge-chip` | The merge-token chip — always blue |
| Filter pills | `HlFilterPills` | The single-select filter bar (do not re-implement) |
| Count chip | `CountChip` | The white-on-blue count pill (header count) |
| Color / channel pill | `ColorPill` / `ChannelPill` | Token-driven category / channel pill — channel brand colors live here once |

<Warning>
**Status is a pill or a dot — never a colored border around a box.** And **green is SMS / success only** — a green pill for anything else, or a red / amber / green rainbow across a pill row, is a bug.
</Warning>

## Toggles — `.hl-toggle`

There is **one** toggle: `.hl-toggle` (with `.hl-toggle--sm`). Its dimensions are tokens (width 40, height 24, thumb 20, travel 18). **Never hand-roll an absolute-positioned thumb switch** — nine files did, and each one drifted.

## The channel-color exception

The one place brand colors are allowed is **channel identity** — the iMessage blue (`#007AFF`), the SMS green (`#34C759`), the social-platform brand colors. These live in **one** place — `ColorPill` / `ChannelPill` — as tokens, never scattered as raw hex maps across files. Myli / AI is **not** a channel color: it uses brand blue.
