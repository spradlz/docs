## 🔨 TC Loop MVP — Running Build Log

<Info>**Status snapshot — 2026-07-07, ~8:15 PM CT.** Design **LOCKED** by Zach ("that's the spec!"). Now building end-to-end. This is the live running list: the plan, what was signed off, and what I'm doing next.</Info>

### What we're building (plain English)

The **TC Loop** is how a lead's transaction gets coordinated, contract-to-close. Today it's confusing — it assumes one deal per client and acts like everyone's already under contract. We're rebuilding it around one idea: **a loop is a container for a pursuit**, and every client has two kinds.

- **🗂 Agency Loop** — ONE per client. It holds all the agency paperwork (buyer-broker / showing agreements) and represents the whole **shopping** phase. No transaction coordinator, no prices, no closing date — none of that applies yet.
- **🏠 Property Loop** — one per house that actually goes **under contract**. The full TC file: coordinator assigned, status, list + contract price, key dates, closing.

It's **one dashboard with a loop selector**. Pick the Agency Loop → the band is the light "shopping" view (transaction chrome dimmed). Pick a property loop → the full transaction chrome lights up. Nothing gets redesigned — same navy band, we just add the selector row and dim what doesn't apply yet. The stepper splits into an **Agency chapter** (prep → sent → signed) and a **Contract chapter** (under contract → … → possession).
