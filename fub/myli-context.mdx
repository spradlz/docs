## The model: one lead, three layers

```
FUB lead (fub.* on Railway)
   │
   ├─ Layer 1  REAL-TIME SYNC ........ live counts + activity on the lead card (see Lead Import)
   ├─ Layer 2  FACTS (deterministic) . the 12-domain fact store — structured, no LLM
   └─ Layer 3  DOSSIER (on-demand LLM) . one Lead Context = [FUB section] + [HouseLoop section]
```

Layer 1 (real-time sync) is documented under [Lead Import](/fub/lead-import). This page is Layers 2–3 —
Myli's brain on the lead.

## The Myli-context UI (LIVE)

The lead-detail **`[Lead Context] / [Follow Up Boss Activity]`** toggle + the **FUB Activity panel**
(`LeadContextPanel.tsx`). The panel reads the lead's full FUB history live from `fub.*` by `fub_person_id` —
this is **Pipe A** (see the [two-pipe rule](/fub/lead-import)). Verified on **James Ross 50308**.

## Layer 2 — The Facts (deterministic extractor)

On first pull, the extractor structures every FUB fact into HouseLoop's **12-domain fact store** — name,
source, timeframe, price range, home views/saves/searches, Zillow Insights. **No LLM, fully structured.**

## Layer 3 — One Lead Context + the dossier (on-demand LLM)

| Section | Source | When it generates |
|---|---|---|
| **Follow Up Boss** | imported FUB history (notes, texts, calls, views, saved searches, insights) via `loadFubBrief` | **on-demand** — `POST /api/leads/[id]/context` (Sonnet, rich) |
| **HouseLoop** | everything happening in HouseLoop going forward | live |

There is **one** Lead Context (FUB section + HouseLoop section), **not two dossiers**. Myli reads the **union**
(FUB facts + FUB dossier + live HouseLoop) for any action. Regeneration is **on-demand** — Sonnet once on
import · Haiku delta on update · **never** auto-per-fact.

## ⭐ The Watermark Law — read-once, merge-forward (universal)

Governs **every** LLM rollup (dossier, facts, Hot Buyers, any summary): **never re-process data already
consumed.** The dossier stores its text + a **`source_cursor`** (high-water mark of everything already baked
in). Each regeneration:

<Steps>
  <Step title="Read only the delta">`SELECT … WHERE synced_at > source_cursor` — only rows we haven't summarized yet.</Step>
  <Step title="Merge, don't re-summarize">Prompt = `[existing dossier] + [only the new rows]` → *"merge the new events in; leave the old as-is."*</Step>
  <Step title="Advance the watermark">Write the updated dossier + `source_cursor = max(synced_at)` consumed.</Step>
</Steps>

First pull = full history (Sonnet, cursor = now). Every pull after = only the delta, merged with Haiku — cheap.
Tokens paid **once per row, ever.**

<Warning>
**The gotcha that makes it correct:** watermark on **`synced_at` (when WE ingested the row), NOT the event's
own date.** A home view from 2 years ago the scraper *just* backfilled is **new to us** — `synced_at > cursor`
catches it. Watermark on the event date and you'd silently skip every freshly-backfilled historical row.
</Warning>
