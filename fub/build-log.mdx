## What we're building (per the PRD)

Every behavioral event тАФ FUB/Zillow today, native HouseLoop tomorrow тАФ is a **first-class `hl_lead_activity`
event** rendered through **ONE catalog-driven card**, identical on **All ┬╖ Website Activity ┬╖
Follow Up Boss Activity**, each carrying **house (address + photo + link) ┬╖ what it is ┬╖ source
(`Zillow.com` | `HouseLoop.ai`)**. One-to-one ZillowтЖФHouseLoop parity (lone asymmetry: `liked_home` is
HouseLoop-only; pre-qual = ZHL тЖФ Lending Loop). This is the HouseLoop 2 event model; FUB is the test data.

## The card system (shipped on localhost)

- **One renderer** тАФ `FeedRow` (exported from `PixelFeedStream.tsx`, consumed by `CenterLane.tsx`) so the All feed
  and the Website Activity tab draw the identical card. Two tiers: **hero** (registration + tour/contact/inquiry/seller
  requests) and **compact** (viewed/saved/liked/shared/repeat/area).
- **Hero** (`ActivityHeroCard.tsx`) тАФ calendar chip ABOVE the address (FUB-style), static black address, bold facts
  ($price ┬╖ bd ┬╖ ba ┬╖ MLS#) matching the viewed-home card, big photo with a big icon fallback, source badge.
- **Dual links** (`HomeLinks.tsx`) тАФ `View on HouseLoop тФВ View on Zillow`. An out-of-area home grays "View on HouseLoop"
  and opens a coverage modal centered on the lead-detail panel (the "iPad"), with a circular X and a single
  "View on Zillow". One component, used by both tiers.
- **Tokens** (`card-tokens.ts` `CARD` + `actor-lanes.ts` `ACTOR_LANE_COLOR`) тАФ every font size/weight/color and photo
  dimension comes from one source, enforced by the ESLint rule `hl/no-raw-card-token`. Zero vibe code.
