## What we're building

Every FUB signal (tour, contact, saved search, view/save/share, Zillow intent tags, insight notes) becomes a
first-class `hl_lead_activity` event. **A request creates the lead; the backfill completes its entire behavioral
history on pull-in, and forward webhooks keep it current.** Behavior-Based Follow-Up, not "hot." We document
Zillow's playbooks; we don't replicate them.

## The two-writer split (why there are no double tours)

The import function owns the **request spine** — the inquiry that created the lead is logged once, keyed
`fub_request` / `fubreq|<event-id>`. The **backfill owns behavior only**: views/saves
(`fub.zillow_home_activity`), Zillow-Insight notes, saved searches, and signal tags. The backfill deliberately
does **not** re-mint request events — doing so logged each tour twice under a second source key. One concept,
one writer, one source key. (`logActivity` dedupes on `(source_table, source_id)`, so each writer's re-runs add
zero rows.)

## Changelog

### 2026-06-26 (PM, late) — the full backfill pipeline
- **Batched the home-activity save** — 500 one-by-one inserts/lead → a single multi-row INSERT. James lands 418 rows in one instant pass (no more per-row stall).
- **Added the `photo` column** to `fub.zillow_home_activity` and wired `images[0].uri` through the scraper → 272 of James's homes carry their photo, so the hero cards are right the first time.
- **Minted P5 into the Inngest import** — `backfillFubBehavior(fubPersonId, leadId)` runs as an idempotent `backfill-behavior` step inside `fub-lead-import`. On pull-in, the lead's complete FUB behavior history becomes first-class events through the real pipeline (Run Log visible). Dry-run on James: **349 events** (274 views, 41 saves, 15 area-interest, 8 repeat-views, 5 saved-searches, 2 tours, inquiry, rental, behavior tags).
- **Split the writers** — pulled `convertEvent` out of the backfill so request events (tour/inquiry/contact) are owned solely by the import's inquiry spine. The backfill now mints behavior only; this kills the double-logged tour cards.

`deploy.sh`; workers reloaded; verified live. Checkpoint: `.claude/checkpoints/fub-behavior-events-scraper-wip-2026-06-26.md`.
