## Image pull-in â€” why photos are blank + the fix (Issue B, diagnosed 2026-06-28)

**Confirmed in data:** every off-market home card has an EMPTY `meta.photo`, while `fub.zillow_home_activity`
HAS photos. Three gaps, and **no Street-View fallback exists anywhere** (`grep streetview` = 0):
1. `resolveHouseloopUrls` (route.ts) injects a photo ONLY when the home matches a HouseLoop `Listing`. Off-market
   â†’ no Listing â†’ no photo, **no fallback**.
2. `behavior_repeat_views` / `behavior_area_interest` are minted with **no photo** (`convertNote`). The request
   spine has a `fub.zillow_home_activity` photo fallback; the behavior cards don't.
3. **Junk URL addresses** (`realscout.com/agent/...`) are minted unfiltered (`convertHomeActivity` + the note regex).

**The fix â€” ONE photo-resolution chain for every home card:** Listing photo â†’ the home's **Zillow photo** from
`fub.zillow_home_activity` (match by zpid, else address â€” the pattern the request spine already uses) â†’ **Google
Street View by address** (new tokenized `photoForAddress()` util, reads the existing Google Maps key). Plus a
**junk filter** (drop an "address" that's a URL / doesn't start with a house number) at the mint layer + the
backfill SQL. Net: **every home card shows a real picture**; URL-junk never becomes a card.
