## What we're building (per the PRD)

Every behavioral event — FUB/Zillow today, native HouseLoop tomorrow — is a **first-class `hl_lead_activity`
event** rendered through **ONE catalog-driven card**, identical on **All · Website Activity · Follow Up Boss
Activity**, each carrying **house (address + photo + link) · what it is · source (`Zillow.com` | `HouseLoop.ai`)**.
One-to-one Zillow↔HouseLoop parity. FUB is the test data; this is the HouseLoop event model.

## The card system — TWO render tiers (no new components, catalog-driven)

`FeedRow` (exported from `PixelFeedStream.tsx`, reused by `CenterLane.tsx`) draws every card from the one
`activity-badges.ts` catalog. Exactly two shapes:

- **HERO** — `ActivityHeroCard.tsx`. Big card: accent border · actor pill · title · "via" source badge · time ·
  photo/icon tile · address + facts ($price · bd · ba · MLS#) · `View on HouseLoop │ View on Zillow` ·
  quote/message · calendar chip. Fired by `request:true`, `registration`, and `hero:true`.
- **COMPACT** — the row card. Thumbnail (80×80) · actor pill · label+icon · "via" badge · time · address ·
  facts · View links. Fired by everything else.

**Zach 2026-06-28: every BEHAVIOR TRIGGER renders as a HERO card** (see the matrix). Home-bearing triggers fill
the hero with the home + photo; flag-only triggers fill the hero body with the "what it means" note instead of
an empty home block.

## Image pull-in — why photos are blank + the fix (Issue B, diagnosed 2026-06-28)

**Confirmed in data:** every off-market home card has an EMPTY `meta.photo`, while `fub.zillow_home_activity`
HAS photos. Three gaps, and **no Street-View fallback exists anywhere** (`grep streetview` = 0):
1. `resolveHouseloopUrls` (route.ts) injects a photo ONLY when the home matches a HouseLoop `Listing`. Off-market
   → no Listing → no photo, **no fallback**.
2. `behavior_repeat_views` / `behavior_area_interest` are minted with **no photo** (`convertNote`). The request
   spine has a `fub.zillow_home_activity` photo fallback; the behavior cards don't.
3. **Junk URL addresses** (`realscout.com/agent/...`) are minted unfiltered (`convertHomeActivity` + the note regex).

**The fix — ONE photo-resolution chain for every home card:** Listing photo → the home's **Zillow photo** from
`fub.zillow_home_activity` (match by zpid, else address — the pattern the request spine already uses) → **Google
Street View by address** (new tokenized `photoForAddress()` util, reads the existing Google Maps key). Plus a
**junk filter** (drop an "address" that's a URL / doesn't start with a house number) at the mint layer + the
backfill SQL. Net: **every home card shows a real picture**; URL-junk never becomes a card.
