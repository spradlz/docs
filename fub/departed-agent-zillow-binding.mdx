<Info>
**Why this exists.** Follow Up Boss is a Zillow walled garden — our **Zillow Tech Connect** lead feed is bound
to the FUB *login email* of a specific seat (**Ben Froeschl**, a departed employee). Deleting that seat or
changing its login email risks the Zillow sync. So we **keep the seat and take over the login** instead. This
page is the runbook for that takeover + the `info@houseloop.ai` capture inbox it feeds. Set up 2026-07-09 (Adam).
</Info>

## The problem

- **Ben Froeschl** left, but his FUB seat (`bfroeschl@spradlingrealtygroup.com`) is the identity **Zillow Tech
  Connect** is bound to. Kill it and the Zillow feed can break / demand re-verification.
- His login email — `bfroeschl@spradlingrealtygroup.com` — was a **deleted Google Workspace user**, so we
  couldn't receive his password-reset mail, and Zillow notifications addressed to him were going nowhere.
- His FUB profile still carried his **old personal cell** (702-686-5697).

## The fix — three moves

### 1. Resurrect the address as an ALIAS (not a user, not a group)
`bfroeschl@spradlingrealtygroup.com` is now an **email alias on the `support@spradlingrealtygroup.com` Google
user** (Admin Console → Users → support → *Add alternate emails* → `bfroeschl`). All external mail to Ben's
address now lands in the **support@** inbox.

<Note>
**Why an alias, not a Google Group.** A Group was the first attempt, but org policy locks *Allow external
members* to **No**, so the Group rejected the external member `fub-notifications@houseloop.ai` and (being a
Group) applied a posting policy that blocked external robot senders. An **alias on a user** has no posting
policy — it accepts all external mail. The Group was deleted; the alias is the keeper.
</Note>

### 2. Take over the FUB login (never change the login email)
1. FUB → *Forgot password* for `bfroeschl@spradlingrealtygroup.com`.
2. The reset email lands in **support@** (via the alias) — **proven**: it arrived seconds after the request.
3. Set a new password → **`spradling2026`** (stored: Master Cred Sheet row *Follow Up Boss -- Ben Froeschl*,
   `.env` `CRED_FUB_BEN_FROESCHL_*`). Verified with a clean login.

<Warning>
**Never change Ben's FUB *login email*.** That is the string Zillow Tech Connect is bound to. Change the
password, the phone, the name — never the login email.
</Warning>

### 3. Phone → the Legacy Agents Twilio line
Ben's FUB profile **Phone** was set to **(913) 675-9877** — the **Legacy Agents Twilio line** that every
departed-agent number forwards to (Caleb's past-agent pool). Old cell 702-686-5697 removed. His old agent number
**913-379-3429** was added as a `Ben Froeschl` row in the **Legacy Agents routing group** (HouseLoop → Settings →
Phone Numbers), matching the other past-agent rows (`platform=follow_up_boss`, forwards to the same line).

<Note>
FUB's Settings → Phone field does **not** persist a `fill()`+blur — it needs a slow character `type()` **and** the
**Save** button at the bottom of the page. (Field mask formats `9136759877` → `(913) 675-9877`.)
</Note>

## `info@houseloop.ai` — the capture + contact inbox

A HouseLoop mailbox, **`info@houseloop.ai`** (Google user; `.env` `CRED_GOOGLE_WORKSPACE_INFO_HOUSELOOP_AI_*`),
is the single landing spot for:

- **Vendor lead notifications** — point Zillow, Follow Up Boss, Google My Business, Realtor.com email alerts here
  so an AI layer can parse them into one stream.
- **Consumer "contact HouseLoop"** — usable directly on the houseloop.ai website contact form ("notifications@"
  read too technical for the consumer side, so the mailbox was renamed **notifications@ → info@**; the old
  `notifications@houseloop.ai` is retained as an **alias** so nothing already pointed at it breaks). The
  `fub-notifications@houseloop.ai` alias is also on this mailbox.
